Aplica-se às novas propriedades de configuração do Spring criadas ao invocar Novo | Componente Spring | Propriedades de Configuração na janela de ferramentas Projeto.
Este template integrado é editável. Além de expressões e comentários Kotlin, você também pode usar variáveis predefinidas (listadas abaixo), que serão expandidas como macros para os valores correspondentes.

Você também pode especificar variáveis personalizadas. Variáveis personalizadas usam o seguinte formato: ${VARIABLE_NAME}, onde VARIABLE_NAME é o nome da variável (por exemplo, ${MY_CUSTOM_FUNCTION_NAME}). Antes de a IDE criar um novo arquivo usando variáveis personalizadas, você verá um diálogo onde poderá definir os valores para as variáveis personalizadas no template.

Ao usar a diretiva #parse, você pode incluir templates da aba Incluir. Para incluir um template, especifique o nome completo do template como um parâmetro entre aspas (por exemplo, #parse("File Header.java")).
Variáveis predefinidas aceitam os seguintes valores:
${PACKAGE_NAME} O nome do pacote onde a nova classe é criada
${NAME} O nome da nova classe que você especificou no diálogo Criar Nova Classe
${USER} O nome de login de sistema do usuário atual
${DATE} A data atual do sistema
${TIME} A hora atual do sistema
${YEAR} O ano atual
${MONTH} O mês atual
${MONTH_NAME_SHORT} As primeiras 3 letras do nome do mês atual (Jan, Fev, etc.)
${MONTH_NAME_FULL} O nome completo do mês atual (Janeiro, Fevereiro, etc.)
${DAY} O dia atual do mês
${HOUR} A hora atual
${MINUTE} O minuto atual
${PROJECT_NAME} O nome do projeto atual
