Go Application é um template integrado usado sempre que um novo arquivo Go é criado. Diferente do template Go File, o Go Application possui uma função main predefinida. Para usar este template, vá para Arquivo | Novo, selecione Go File na categoria Go e escolha Aplicativo Simples.

Este template é editável. No template, você pode usar as variáveis predefinidas listadas abaixo. Essas variáveis serão expandidas posteriormente para seus valores correspondentes no editor.

Você também pode especificar variáveis personalizadas. Variáveis personalizadas usam o seguinte formato: ${VARIABLE_NAME}, onde VARIABLE_NAME é o nome da variável (por exemplo, ${MY_CUSTOM_FUNCTION_NAME}). Antes que a IDE crie um novo arquivo usando uma variável personalizada, você verá uma caixa de diálogo onde poderá definir o valor da variável personalizada no template.

Ao usar a diretiva #parse, você pode incluir templates da aba Include. Para incluir um template, especifique o nome completo do template entre aspas como parâmetro (por exemplo, #parse("File Header")).
Lista de variáveis predefinidas
${GO_PACKAGE_NAME_WITH_TEST_SUFFIX} O nome do pacote no qual o novo arquivo de teste é criado. O arquivo deve ter o sufixo _test. O nome usa o seguinte formato: package_test.
${GO_PACKAGE_NAME} O nome do pacote no qual o novo arquivo é criado.
${DATE} Data atual do sistema.
${DAY} Dia atual do mês.
${DAY_NAME_SHORT} As primeiras 3 letras do nome do dia atual (por exemplo, Mon, Tue, etc.).
${DAY_NAME_FULL} O nome completo do dia atual (por exemplo, Monday, Tuesday, etc.).
${DIR_PATH} O caminho do diretório do novo arquivo. Este caminho é relativo à raiz do projeto.
${DS} Símbolo de cifrão ($). Esta variável é usada para escapar o caractere de cifrão, para que ele não seja tratado como um prefixo de variável de template.
${FILE_NAME} O nome do novo arquivo.
${HOUR} Hora atual.
${MINUTE} Minuto atual.
${SECOND} Segundo atual.
${MONTH} Mês atual.
${MONTH_NAME_SHORT} As primeiras 3 letras do nome do mês atual (por exemplo, Jan, Feb, etc.).
${MONTH_NAME_FULL} O nome completo do mês atual (por exemplo, January, February, etc.).
${NAME} O nome da nova entidade: arquivo, tipo, interface, etc.
${PRODUCT_NAME} O nome da IDE.
${PROJECT_NAME} O nome do projeto atual.
${TIME} Hora atual do sistema.
${USER} O nome de login do sistema do usuário atual.
${YEAR} Ano atual.
