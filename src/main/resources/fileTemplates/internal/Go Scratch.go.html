Go Scratch é o modelo integrado usado sempre que você cria um novo arquivo de rascunho (scratch file) do Go. Arquivos de rascunho são arquivos totalmente funcionais, executáveis e depuráveis que você pode usar para esboçar suas ideias de codificação. Para criar um arquivo de rascunho, clique em Arquivo | Novo | Arquivo de Rascunho. Na caixa de diálogo Novo Arquivo de Rascunho, localize e selecione Go.

O modelo é editável. No modelo, você pode usar as variáveis predefinidas listadas abaixo. Essas variáveis serão expandidas posteriormente para os valores correspondentes no editor.

Você também pode especificar variáveis personalizadas. As variáveis personalizadas usam o seguinte formato: ${VARIABLE_NAME}, onde VARIABLE_NAME é o nome da variável (por exemplo, ${MY_CUSTOM_FUNCTION_NAME}). Antes que a IDE crie um novo arquivo usando uma variável personalizada, você verá uma caixa de diálogo onde poderá definir o valor para a variável personalizada no modelo.

Ao usar a diretriz #parse, você pode incluir modelos da aba Incluir. Para incluir um modelo, especifique o nome completo do modelo como um parâmetro entre aspas (por exemplo, #parse("File Header")).
Lista de variáveis predefinidas
${GO_PACKAGE_NAME_WITH_TEST_SUFFIX} O nome do pacote no qual o novo arquivo de teste é criado. O arquivo deve ter o sufixo _test. O nome terá o formato: package_test.
${GO_PACKAGE_NAME} O nome do pacote no qual o novo arquivo é criado.
${DATE} Data atual do sistema.
${DAY} Dia atual do mês.
${DAY_NAME_SHORT} As primeiras 3 letras do nome do dia atual (por exemplo, Mon, Tue, etc.).
${DAY_NAME_FULL} O nome completo do dia atual (por exemplo, Monday, Tuesday, etc.).
${DIR_PATH} O caminho do diretório para o novo arquivo. Este caminho é relativo à raiz do projeto.
${DS} Símbolo de cifrão ($). Esta variável é usada para escapar o caractere de cifrão, para que ele não seja tratado como um prefixo de uma variável de modelo.
${FILE_NAME} O nome do novo arquivo.
${HOUR} Hora atual.
${MINUTE} Minuto atual.
${SECOND} Segundo atual.
${MONTH} Mês atual.
${MONTH_NAME_SHORT} As primeiras 3 letras do nome do mês atual (por exemplo, Jan, Feb, etc.).
${MONTH_NAME_FULL} O nome completo do mês atual (por exemplo, January, February, etc.).
${NAME} O nome da nova entidade: arquivo, tipo, interface, etc.
${PRODUCT_NAME} O nome da IDE.
${PROJECT_NAME} O nome do projeto atual.
${TIME} Hora atual do sistema.
${USER} Nome de login do sistema do usuário atual.
${YEAR} Ano atual.
