Go File é um modelo integrado usado toda vez que você cria um novo arquivo Go. Para usar este modelo, vá para Arquivo | Novo, selecione Go File na categoria Go e clique em Arquivo vazio.

Este modelo é editável. No modelo, você pode usar as variáveis predefinidas listadas abaixo. Essas variáveis serão expandidas para seus valores correspondentes posteriormente no editor.

Você também pode especificar variáveis personalizadas. As variáveis personalizadas usam o seguinte formato: ${VARIABLE_NAME}, onde VARIABLE_NAME é o nome da variável (por exemplo, ${MY_CUSTOM_FUNCTION_NAME}). Antes que o IDE crie um novo arquivo usando variáveis personalizadas, você verá uma caixa de diálogo onde poderá definir os valores das variáveis personalizadas no modelo.

Ao usar a diretiva #parse, você pode incluir modelos da guia Incluir. Para incluir um modelo, especifique o nome completo do modelo como um parâmetro entre aspas (por exemplo, #parse("File Header")).
Lista de variáveis predefinidas
${GO_PACKAGE_NAME_WITH_TEST_SUFFIX} O nome do pacote no qual o novo arquivo de teste é criado. O arquivo deve ter o sufixo _test. O nome usa o formato: package_test.
${GO_PACKAGE_NAME} O nome do pacote no qual o novo arquivo é criado.
${DATE} A data atual do sistema.
${DAY} O dia atual do mês.
${DAY_NAME_SHORT} As primeiras 3 letras do nome do dia atual (ex: Mon, Tue, etc.).
${DAY_NAME_FULL} O nome completo do dia atual (ex: Monday, Tuesday, etc.).
${DIR_PATH} O caminho do diretório para o novo arquivo. Este caminho é relativo à raiz do projeto.
${DS} O símbolo de cifrão ($). Esta variável é usada para escapar o caractere de cifrão, para que não seja tratado como um prefixo de variável de modelo.
${FILE_NAME} O nome do novo arquivo.
${HOUR} A hora atual.
${MINUTE} O minuto atual.
${SECOND} O segundo atual.
${MONTH} O mês atual.
${MONTH_NAME_SHORT} As primeiras 3 letras do nome do mês atual (ex: Jan, Feb, etc.).
${MONTH_NAME_FULL} O nome completo do mês atual (ex: January, February, etc.).
${NAME} O nome da nova entidade: arquivo, tipo, interface, etc.
${PRODUCT_NAME} O nome do IDE.
${PROJECT_NAME} O nome do projeto atual.
${TIME} A hora atual do sistema.
${USER} O nome de login do sistema do usuário atual.
${YEAR} O ano atual.
