Define o modelo usado para gerar JavaDoc para classes, interfaces ou registros, por exemplo, ao aplicar a ação de intenção Adicionar Javadoc.

Além de expressões e comentários Java, você pode usar variáveis predefinidas, que serão expandidas para seus valores correspondentes como macros.

Ao usar a diretiva #parse, é possível incluir modelos da guia Includes. Para incluir um modelo, especifique o nome completo do modelo como um parâmetro entre aspas (por exemplo, #parse("File Header.java")).
As variáveis predefinidas aceitam os seguintes valores:
${RECORD_COMPONENTS} Lista de parâmetros do registro
${TYPE_PARAMS} Lista de nomes de parâmetros de tipo
${CONTAINING_CLASS} Nome da classe pai
${INTERFACE} Retorna verdadeiro se o proprietário do JavaDoc for uma interface
${ENUM} Retorna verdadeiro se o proprietário do JavaDoc for um enum
${RECORD} Retorna verdadeiro se o proprietário do JavaDoc for um registro
${PACKAGE_NAME} O nome do pacote no qual o novo enum será criado
${NAME} O nome do novo enum que você especificou na caixa de diálogo Criar Nova Classe
${USER} Nome de login de sistema do usuário atual
${DATE} Data atual do sistema
${TIME} Hora atual do sistema
${YEAR} Ano atual
${MONTH} Mês atual
${MONTH_NAME_SHORT} As primeiras 3 letras do nome do mês atual (Jan, Fev, etc.)
${MONTH_NAME_FULL} Nome completo do mês atual (Janeiro, Fevereiro, etc.)
${DAY} Dia atual do mês
${HOUR} Hora atual
${MINUTE} Minuto atual
${PROJECT_NAME} Nome do projeto atual
