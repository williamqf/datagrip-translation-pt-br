Este é um template para criar métodos de teste WebSocket para Ktor em uma classe de teste JUnit 4.
As variáveis pré-definidas recebem os seguintes valores:
${NAME} O nome do método criado
${ROUTE} A rota relativa ao serviço atual que precisa ser testada
${CONFIGURE_SERVER} Um método de extensão de rota que configura a árvore de rotas para a parte do servidor que precisa ser testada
${BODY} O corpo do método gerado
