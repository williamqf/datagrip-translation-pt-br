Este modelo interno é usado para preencher o corpo de um método sempre que ele é gerado, por exemplo, através da ação de intenção Criar método do uso.
O modelo é editável. Junto com expressões e comentários de JavaScript, você também pode usar variáveis predefinidas. Essas variáveis serão substituídas com os valores correspondentes quando o modelo for aplicado.
Variáveis predefinidas tomarão os seguintes valores:
${IS_ASSIGNMENT_TARGET} retorna 'true' se o método criado é um destino de atribuição
${IS_NEW_CALL} retorna 'true' se o método criado está em chamada 'new'
${HAS_EXPECTED_TYPE} retorna 'true' se o método criado está em local com o tipo esperado
${DEFAULT_RETURN_VALUE} valor padrão do tipo esperado para o método criado ('indefinido' se nenhum tipo esperado)
${METHOD_NAME} nome do método criado
${SEMICOLON} ponto e vírgula ou vazio, com base nas opções de estilo do código
Todas as variáveis predefinidas do modelo Cabeçalho do Arquivo (guia Incluir) também estão disponíveis (ex.: ${USER}, ${DATE}, etc.)
