Este modelo integrado é usado para preencher o corpo do método sempre que um método for gerado (por exemplo, por meio da ação de intenção Criar método a partir do uso).
O modelo é editável. Além de expressões JavaScript e comentários, você também pode usar variáveis predefinidas. Ao aplicar o modelo, essas variáveis serão substituídas pelos valores correspondentes.
As variáveis predefinidas aceitarão os seguintes valores:
${IS_ASSIGNMENT_TARGET} Retorna 'true' se o método criado for um alvo de atribuição.
${IS_NEW_CALL} Retorna 'true' se o método criado estiver sob uma chamada 'new'.
${HAS_EXPECTED_TYPE} Retorna 'true' se o método criado estiver em uma posição onde um tipo é esperado.
${DEFAULT_RETURN_VALUE} O valor padrão do tipo esperado do método criado ('undefined' se não houver um tipo esperado).
${METHOD_NAME} O nome do método criado.
${SEMICOLON} Ponto e vírgula ou vazio, com base nas opções de estilo de código.
Todas as variáveis predefinidas do modelo de Cabeçalho de arquivo (guia Inclusões) também estão disponíveis. (Por exemplo: ${USER}, ${DATE}, etc.)
