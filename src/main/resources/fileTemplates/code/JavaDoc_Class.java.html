Define um modelo usado para gerar JavaDoc para as classes, interfaces ou registros, por exemplo quando a ação de intenção Adicionar Javadoc é aplicada.

Junto com expressões e comentários Java, você também pode usar as variáveis predefinidas que serão então expandidas como macros para os valores correspondentes.

Usando a diretiva #parse, você pode incluir modelos da guia Inclusões. Para incluir um modelo, especifique o nome completo do modelo como um parâmetro entre aspas (por exemplo, #parse("File Header.java").
As variáveis predefinidas assumem os seguintes valores:
${RECORD_COMPONENTS} Lista dos parâmetros do registro
${TYPE_PARAMS} Lista dos nomes dos parâmetros de tipo
${CONTAINING_CLASS} Nome da classe pai
${INTERFACE} Retorna true quando o proprietário do JavaDoc é uma interface
${ENUM} Retorna true quando o proprietário do JavaDoc é um enum
${RECORD} Retorna true quando o proprietário do JavaDoc é um registro
${PACKAGE_NAME} Nome do pacote no qual um novo enum é criado
${NAME} Nome do novo enum especificado por você na caixa de diálogo Criar nova classe
${USER} Nome do sistema de login do usuário atual
${DATE} Data atual do sistema
${TIME} Hora atual do sistema
${YEAR} Ano atual
${MONTH} Mês atual
${MONTH_NAME_SHORT} As primeiras 3 letras do nome do mês atual (Jan, Fev, e assim por diante)
${MONTH_NAME_FULL} Nome completo do mês atual (Janeiro, Fevereiro, e assim por diante)
${DAY} Dia atual do mês
${HOUR} Hora atual
${MINUTE} Minuto atual
${PROJECT_NAME} Nome do projeto atual
