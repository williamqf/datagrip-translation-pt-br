Esta intenção altera o contrato de um método de biblioteca. A sintaxe do contrato do método é a seguinte: 

 contract ::= (clause ';')* clause
 clause ::= args '->' effect
 args ::= ((arg ',')* arg )?
 arg ::= value-constraint
 value-constraint ::= '_' | 'null' | '!null' | 'false' | 'true'
 effect ::= value-constraint | 'fail' | 'new' | 'this' | 'param' number
 number ::= [1-9] [0-9]*
As restrições significam:
  • _: qualquer valor
  • null: valor nulo
  • !null: um valor estaticamente comprovado como não nulo
  • true: valor booleano true
  • false: valor booleano false
  • fail: o método lançará uma exceção se os argumentos satisfizerem as restrições de argumento
  • new: toda vez que o método é executado, ele retorna um novo objeto não nulo, diferente de outros objetos que existiam no heap antes da execução do método. Se o método for puro, o novo objeto não será armazenado em campos ou arrays e será perdido se o valor de retorno do método não for utilizado.
  • this: o método retorna uma referência 'this' não nula
  • param1 (param2, param3, etc.): o método retorna o seu primeiro (segundo, terceiro, etc.) argumento
Exemplos:
  • @Contract("_, null -> null"): se o segundo argumento do método for nulo, ele retornará nulo.
  • @Contract("_, null -> null; _, !null -> !null") - se o segundo argumento do método for nulo, ele retornará nulo; caso contrário, retornará um valor não nulo.
  • @Contract("true -> fail") - um método assertFalse típico que lançará uma exceção se true for passado a ele.
