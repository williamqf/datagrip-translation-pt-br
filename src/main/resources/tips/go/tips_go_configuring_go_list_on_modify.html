Configuração do comportamento de 'go list'

Se você não quiser executar go list toda vez que go.mod for alterado, pressione &shortcut:ShowSettings; para abrir as configurações e navegue até Build, Execution, Deployment | Build Tools. Desmarque a caixa de seleção Reload project on build script change.

Por padrão, o GoLand recarrega o projeto após a modificação do arquivo go.mod. Se você tiver a opção External changes selecionada, o IDE não executará o go list automaticamente quando você editar arquivos. Nesse caso, o ícone "Load Go Modules changes" será exibido. Depois de terminar de editar, clique no ícone para carregar as alterações.
