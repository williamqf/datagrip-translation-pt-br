Relata blocos catch vazios ou que podem ignorar exceções.

Embora ocasionalmente seja intencional, blocos catch vazios podem complicar a depuração. Além disso, ignorar o parâmetro do catch pode ser um erro. Finalmente, analisadores de código estático relatarão se for detectado que um bloco catch pode ignorar silenciosamente exceções importantes da VM, como NullPointerException. Ignorar tais exceções (sem registrá-las em log ou relançá-las) pode ocultar bugs.

Esta inspeção não relata nenhum parâmetro de catch nomeado como ignore ou ignored. Pelo contrário, ela emitirá um aviso para parâmetros de catch nomeados como ignore ou ignored que estejam realmente sendo usados. Além disso, esta inspeção não relata parâmetros de catch em fontes de teste nomeados como expected ou ok.

Você pode usar uma correção rápida para alterar o nome da exceção para ignored. Para blocos catch vazios, uma correção rápida adicional é sugerida para gerar o corpo do catch. Você pode modificar o modelo "Catch Statement Body" na aba "Código" em Configurações | Editor | Templates de arquivos e código.

Exemplo:


  try {
    throwingMethod();
  } catch (IOException ex) {

  }

Após aplicar a correção rápida:


  try {
    System.out.println(System.in.read());
  } catch (IOException ignored) {

  }

Configurar inspeção:

  • Use a opção Não avisar quando o bloco 'catch' contiver comentários para ignorar blocos catch que contenham comentários.
  • Use a opção Não avisar quando o bloco 'catch' não estiver vazio para ignorar blocos catch que contenham instruções ou comentários internamente, mas não usem a variável em si.
  • Use a opção Não avisar quando uma exceção chamada 'ignore(d)' não for ignorada de fato para ignorar variáveis nomeadas ignored que estejam em uso.
