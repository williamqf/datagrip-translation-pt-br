Relata problemas relacionados a anotações de nulidade.

Exemplos:

  • Método de sobreposição não anotado: 
    
abstract class A {
  @NotNull abstract String m();
}
class B extends A {
    String m() { return "empty string"; }
}
  • Tipo primitivo anotado: @NotNull int myFoo;
  • @Nullable e @NotNull no mesmo membro: @Nullable @NotNull String myFooString;
  • Coleção de elementos anuláveis atribuída a uma coleção de elementos não nulos: 
    
void testList(List<@Nullable String> nullableList) {
   List<@NotNull String> list2 = nullableList;
}

Use o botão Configurar anotações para especificar anotações de nulidade e as caixas de seleção para ajustar onde a inspeção deve exibir avisos.

Esta inspeção relata apenas se o nível de linguagem do projeto ou módulo for 5 ou superior e se as anotações de nulidade estiverem disponíveis no classpath.
