Relata problemas no contexto baseado em XML do Spring Security:

  • Referência de bean não resolvida
  • Tag ou atributo obrigatório ausente
  • Tipo de propriedade incorreto
  • Propriedade enum inconsistente
  • Tipo incorreto de bean referenciado

Também relata os seguintes problemas:

  • Erro de configuração de <https>
    • <http> com o padrão '/**' deve ser o último
    • Múltiplos elementos <http> sem 'pattern' não são permitidos
  • Erro de configuração de <http>
    • Tag vazia se security='none'
  • <user-service>
    • Deve possuir o atributo 'properties' ou uma lista de elementos <user>
  • <ldap-server>
    • 'manager-password' é obrigatório ao usar 'manager-dn'
    • Deve possuir 'user-context-mapper-ref' ou 'user-details-class'
  • <authentication-provider>
    • Apenas um tipo de serviço de usuário é permitido
  • <global-method-security>
    • O valor de 'pre-post-annotations' deve ser 'enabled' para usar <pre-post-annotation-handling>
    • 'mode' com valor 'aspectj' não pode ser usado para <protect-pointcut> e <intercept-url>
    • O atributo 'requires-channel' é permitido apenas para o <http> pai
  • <protect>
    • Misturar <protect> com <global-method-security> não é recomendado
