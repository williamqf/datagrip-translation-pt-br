Relata alocações de objetos ou arrays dentro de loops. Embora não seja necessariamente um problema, alocações de objetos dentro de loops são um bom lugar para encontrar vazamentos de memória e problemas de desempenho.

Esta inspeção relata as seguintes construções:

  • Alocação explícita através do operador new
  • Métodos conhecidos por retornar novos objetos
  • Referências de métodos vinculadas a instâncias
  • Lambdas que capturam variáveis ou referências a this

Exemplo:


  // Alocação explícita
  for (Status status : Status.values()) {
    declarationsMap.put(status, new ArrayList<>());
  }

  // Lambda capturando variável
  String message = "Engine running.";
  for (Engine engine : engines) {
      if (!isRunning(engine)) {
          logger.warn(() -> {
              return String.format(message);
          });
      }
  }

  // Referência de método vinculada à instância
  for(Node node : nodes) {
    descriptor = node.getDescription();
    descriptor.ifPresent(dynamicTestExecutor::execute);
  }
