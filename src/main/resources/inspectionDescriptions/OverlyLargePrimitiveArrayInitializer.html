Relata expressões de inicialização de array para arrays primitivos que contêm elementos demais. Tais inicializadores podem resultar em arquivos de classe excessivamente grandes, pois o código deve ser gerado para inicializar cada elemento do array. Em ambientes com restrição de memória ou largura de banda, carregar grandes arrays primitivos a partir de arquivos de recurso pode ser mais eficiente.

Esta inspeção destina-se ao Java ME e a outros ambientes altamente restritos em termos de recursos. Aplicar os resultados desta inspeção sem critério pode afetar negativamente a clareza e o design do código.

Use a opção para especificar o número máximo de elementos permitido em um inicializador de array primitivo.
