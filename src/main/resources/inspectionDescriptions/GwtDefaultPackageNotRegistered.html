Relata um descritor de módulo GWT que não registra as pastas padrão client ou public, mas registra pastas de código-fonte ou públicas personalizadas.

O GWT trata implicitamente a pasta client como um caminho de código-fonte do lado do cliente apenas quando não há tags source e super-source no arquivo gwt.xml. Portanto, se a pasta client existir, mas não estiver registrada no arquivo gwt.xml, e o gwt.xml declarar tags source ou super-source personalizadas, isso provavelmente indica um erro.

Da mesma forma, se o gwt.xml declarar uma pasta public personalizada, mas não registrar uma pasta existente com o nome padrão, isso também pode indicar um erro.

A correção rápida adiciona o registro explícito das pastas padrão ao arquivo gwt.xml.
