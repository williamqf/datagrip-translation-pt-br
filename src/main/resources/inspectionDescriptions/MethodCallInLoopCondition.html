Relata chamadas de método na parte condicional de instruções de loop. Em ambientes com recursos altamente limitados, tais chamadas podem afetar negativamente o desempenho.

A aplicação indiscriminada dos resultados desta inspeção pode afetar negativamente a clareza e o design do código. Esta inspeção destina-se ao Java ME e a outros ambientes com recursos altamente limitados.

Exemplo:


  String s = "example";
  for (int i = 0; i < s.length(); i++) {
    System.out.println(s.charAt(i));
  }

Após a aplicação da correção rápida:


  String s = "example";
  int length = s.length();
  for (int i = 0; i < length; i++) {
    System.out.println(s.charAt(i));
  }

Use a opção para ignorar chamadas a métodos de iteração comuns do Java (como Iterator.hasNext()) e métodos conhecidos com efeitos colaterais (como Atomic*.compareAndSet).
