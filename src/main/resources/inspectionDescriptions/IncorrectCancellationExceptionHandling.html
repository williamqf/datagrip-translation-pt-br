Relata casos em que ProcessCanceledException e CancellationException são tratados incorretamente.

ProcessCanceledException e seus sucessores não devem ser capturados, omitidos, registrados em log ou tratados de forma alguma. Em vez disso, devem ser relançados para que a infraestrutura do IntelliJ Platform possa tratá-los corretamente.

No contexto de corrotinas, CancellationException não deve ser capturada, omitida, registrada em log ou tratada de forma alguma. Em vez disso, deve ser relançada para que a infraestrutura de corrotinas possa tratá-la corretamente.

A inspeção relata capturas explícitas de ProcessCanceledException (incluindo sucessores) e CancellationException, bem como capturas de RuntimeException, Exception e Throwable que abrangem exceções de cancelamento.

Exemplos:

ProcessCanceledException:


// ruim:
try {
  // ...
} catch (ProcessCanceledException e) { // a exceção não deve ser omitida
}

// incorreto:
try {
  // ...
} catch (ProcessCanceledException e) {
  LOG.error("Error occurred", e); // a exceção não deve ser registrada no log
  throw e;
}


// correto:
try {
  // ...
} catch (ProcessCanceledException e) {
  // ações adicionais
  throw e;
}

CancellationException:


// ruim:
cs.launch {
  try {
    // ...
  } catch (e: CancellationException) { // a exceção não deve ser omitida
  }
}

// incorreto:
suspend fun suspendingFunction() {
  try {
    // ...
  } catch (ProcessCanceledException e) {
    LOG.error("Error occurred", e); // a exceção não deve ser registrada no log
    throw e;
  }
}


// correto:
cs.launch {
  try {
    // ...
  } catch (ProcessCanceledException e) {
    // ações adicionais
    throw e;
  }
}

Observação: a inspeção está desativada em fontes de teste.
