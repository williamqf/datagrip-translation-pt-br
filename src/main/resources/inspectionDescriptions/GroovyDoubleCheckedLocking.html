Relata o bloqueio de verificação dupla (double-checked locking).

O bloqueio de verificação dupla tenta inicializar um campo sob demanda de maneira segura para threads (thread-safe), evitando ao mesmo tempo o custo de sincronização. Infelizmente, ele não é seguro para threads quando usado em um campo que não foi declarado como volatile. Ao usar o Java 1.4 ou anterior, o bloqueio de verificação dupla não funciona nem mesmo para campos volatile. Leia o artigo vinculado acima para obter uma explicação detalhada do problema.

Exemplo:


  class Foo {
      private Helper helper = null

      Helper getHelper() {
          if (helper == null)
              synchronized(this) {
                  if (helper == null) {
                      helper = new Helper()
                  }
              }
          }
          return helper;
      }
  }
