Relata classes que podem ser convertidas em classes record.

Classes record focam na modelagem de dados imutáveis em vez de comportamento extensível. Implementações implícitas automáticas de métodos orientados a dados (como equals() e acessores) ajudam a reduzir o código boilerplate.

Observe que nem toda classe pode ser uma classe record. Aqui estão algumas restrições:

  • A classe não pode ter subclasses.
  • Todos os campos não static na classe devem ser final.
  • Não deve haver inicializadores, construtores genéricos ou métodos nativos.

Para uma descrição completa das classes record, consulte a Especificação da Linguagem Java.

Exemplo:


  class Point {
    private final double x;
    private final double y;

    Point(double x, double y) {
      this.x = x;
      this.y = y;
    }

    double getX() {
      return x;
    }

    double getY() {
      return y;
    }
  }

Após a aplicação da correção rápida:


  record Point(int x, int y) {
  }

Ativar a opção Sugerir a renomeação de métodos acessores permite renomear automaticamente os acessores getX()/isX() para x().

Use a opção Não relatar se os membros se tornarem mais acessíveis para excluir classes cujos membros teriam sua acessibilidade enfraquecida devido à conversão. A correção rápida continuará disponível como uma intenção, e acioná-la exibirá os membros afetados e solicitará confirmação. No modo em lote, a conversão não será sugerida.

Use a lista Suprimir conversão se a classe estiver anotada por para excluir das conversões as classes cujas anotações correspondam aos padrões especificados.
