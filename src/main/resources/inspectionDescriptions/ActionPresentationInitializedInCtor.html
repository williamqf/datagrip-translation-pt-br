Relata quaisquer ações registradas no arquivo plugin.xml e inicializa suas representações (texto, descrição ou ícone) dentro de seus construtores.

Carregar representações dentro de construtores pode levar à alocação de recursos, o que pode ser desnecessário. Uma abordagem mais eficiente é aproveitar os construtores sem argumentos de AnAction e outras classes base e seguir as convenções de configuração de texto, descrição e ícone no plugin.xml. O IDE só carregará o texto, a descrição e o ícone quando a ação for realmente exibida na IU.

As convenções para configurar texto, descrição e ícone são as seguintes:

  • Definir o atributo id para a ação no arquivo plugin.xml.
  • Definir também o atributo icon, se necessário.
  • Definir texto e descrição no bundle de mensagens associado (pode ser substituído em <actions>):
    • action.<action-id>.text=Texto da ação traduzido
    • action.<action-id>.description=Descrição da ação traduzida

Exemplo ruim:


  // NewKotlinFileAction.kt
  internal class NewKotlinFileAction : AnAction(
    KotlinBundle.message("action.new.file.text"),
    KotlinBundle.message("action.new.file.description"),
    KotlinIcons.FILE
  )


  
  <action
    class="org.jetbrains.kotlin.idea.actions.NewKotlinFileAction"
  </action>

Exemplo correto:


  // NewKotlinFileAction.kt
  internal class NewKotlinFileAction : AnAction()


  
  <action
    id="Kotlin.NewFile"
    class="org.jetbrains.kotlin.idea.actions.NewKotlinFileAction"
    icon="org.jetbrains.kotlin.idea.KotlinIcons.FILE">
  </action>


  # KotlinBundle.properties
  action.Kotlin.NewFile.text=Classe/arquivo Kotlin
  action.Kotlin.NewFile.description=Cria uma nova classe ou arquivo Kotlin
