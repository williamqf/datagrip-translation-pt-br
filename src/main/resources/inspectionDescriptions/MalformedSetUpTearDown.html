Relata métodos JUnit 3 setUp() ou tearDown() que não são declarados como public, não retornam void ou aceitam argumentos. Tais métodos setUp() ou tearDown() provavelmente foram criados sem querer e não serão executados pelo executor de teste do JUnit.

Exemplo:


  public class PesoTeste extends TestCase {
    private Arquivo arquivo;
    private int setUp(String prefix) throws IOException {
      arquivo = Arquivo.createTempFile(prefix, ".tmp");
      return 0;
    }

    // ... testes vão aqui
  }
