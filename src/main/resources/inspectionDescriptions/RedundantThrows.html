Relata exceções declaradas na assinatura de um método que nunca são lançadas pelo próprio método ou por suas implementações e métodos que o sobrescrevem.

Esta inspeção ignora métodos relacionados à serialização, como os métodos readObject() e writeObject().

Exemplo:


  void method() throws InterruptedException {
    System.out.println();
  }

A correção rápida remove exceções desnecessárias da declaração e normaliza instruções try-catch redundantes:


  void method() {
    System.out.println();
  }

Nota: Por motivos de desempenho, algumas exceções podem não ser relatadas ao realçar no editor. Para ver todos os resultados, execute a inspeção selecionando Código | Inspecionar código ou Código | Analisar código | Executar inspeção por nome no menu principal.

Use a opção Ignorar exceções lançadas por métodos de ponto de entrada para não relatar exceções lançadas por métodos como main(). Os métodos de ponto de entrada podem ser configurados nas configurações da inspeção Java | Redundância de declaração | Declaração não utilizada.
