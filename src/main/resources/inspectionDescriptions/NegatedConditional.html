Relata expressões condicionais onde a condição está negada.

Inverter a ordem dos ramos da expressão condicional geralmente aumenta a clareza de tais instruções.

Use as opções Ignorar comparações '!= null' e Ignorar comparações '!= 0' para ignorar comparações na forma obj != null ou num != 0. Como obj != null efetivamente significa "o objeto existe", o sentido da expressão completa não envolve nenhuma negação, sendo assim fácil de entender.

O mesmo raciocínio se aplica a expressões num != 0, especialmente ao usar máscaras de bits.

Essas formas também têm a vantagem adicional de mencionar os casos de interesse primeiro. Na maioria dos casos, o valor do ramo == null é o próprio null, como mostrado no exemplo a seguir: 


    static String getName(Person p) {
        return p != null ? p.getName() : null;
    }

    static String getExecutableString(int fileMode) {
        return (fileMode & 0b001001001) != 0 ? "executable" : "non-executable";
    }
