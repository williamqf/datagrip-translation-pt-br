Relata chamadas typing.cast em que os tipos de origem e de destino não estão relacionados.

Um erro é relatado quando o tipo de origem não é um subtipo do tipo de destino, nem o tipo de destino é um subtipo do tipo de origem. Tais casts geralmente indicam um erro lógico, pois não se pode assumir que uma instância de um tipo seja uma instância de outro, e o typing.cast não verifica tipos dinamicamente.

Esta inspeção aplica-se mesmo a tipos que poderiam, teoricamente, ter um descendente comum. Por exemplo, ela sinalizará um cast entre duas classes irmãs Left e Right, pois ambas herdam de Top, mas não possuem uma relação de herança direta entre si.

Exemplo:


from typing import cast

# Tipos que não se sobrepõem — provavelmente um erro
cast(int, "a")          # 'str' -> 'int'
cast(list[int], ["a"])  # 'list[str]' -> 'list[int]'

# O escape explícito recomendado é usar um "cast duplo"
cast(int, cast(object, "a"))  # ok

# Casos de sobreposição legítimos
cast(int, object())    # downcast válido
cast(object, 1)        # upcast válido

# No entanto, a conversão a seguir é inválida porque list é invariante.
# Atualmente não é suportado por esta inspeção
int_list = [1, 2, 3]
cast(list[object], int_list)

A inspeção depende de informações de tipo estáticas; nenhum aviso será relatado quando o tipo for desconhecido. A variância de tipos genéricos ainda não é considerada.
