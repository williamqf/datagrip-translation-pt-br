Relata um tipo de retorno incorreto para métodos com a anotação @Deployment.

Conforme a documentação do Arquillian, métodos que utilizam a anotação @Deployment devem retornar um tipo que herda do ShrinkWrap Archive.

Exemplo:


  // Este caso de teste não pode ser iniciado pelo Arquillian
  @RunWith(Arquillian.class)
  public class SomeTest {
    @Deployment
    public static ZipFile createDeployment() {
      throw new UnsupportedOperationException("Implement me");
    }

    @Test
    public void testSomething() {
      Assert.fail("To be implemented");
    }
  }

Após aplicar o quick fix:


  import org.jboss.shrinkwrap.api.Archive;

  @RunWith(Arquillian.class)
  public class SomeTest {
    @Deployment
    public static Archive createDeployment() {
      throw new UnsupportedOperationException("Implement me");
    }

    @Test
    public void testSomething() {
      Assert.fail("To be implemented");
    }
  }
