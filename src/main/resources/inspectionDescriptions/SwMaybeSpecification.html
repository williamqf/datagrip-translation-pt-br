Detecta arquivos YAML/JSON que podem ser interpretados como parte de uma especificação OpenAPI/Swagger.

Esses arquivos não contêm características de especificação explícitas, mas são referenciados por arquivos de especificação primários próximos.

O exemplo a seguir contém dois arquivos localizados no mesmo diretório. O primeiro é um arquivo de especificação primária simples. O segundo é referenciado pelo primeiro e, portanto, é sugerido que seja considerado como uma especificação.

Arquivo de especificação primária `openapi.yaml`:


openapi: 3.1.0
components:
  schemas:
    CustomSchema:
      description: Custom schema object
      properties:
        foo:
          $ref: 'common.components.yaml#/components/schemas/CommonSchema'

Candidato a arquivo de especificação `common.components.yaml`:


components: # 'Mark file as OpenAPI specification' highlighting
  schemas:
    CommonSchema:
      description: Common schema object reused in several specifications
