Relata instâncias de Logger inicializadas com um literal de class pertencente a uma classe diferente da classe que contém o Logger. Isso acontece frequentemente ao copiar e colar código de outra classe e pode resultar no registro de eventos em categorias inesperadas e na aplicação incorreta de filtros.

Uma correção rápida é fornecida para substituir o literal da classe externa pelo literal da classe envolvente.

Exemplo:


  public class Paramount {
    protected static final Logger LOG = Logger.getLogger(Critical.class);

    // ... outros campos e métodos
  }

Após a aplicação da correção rápida:


  public class Paramount {
    protected static final Logger LOG = Logger.getLogger(Paramount.class);

    // ... outros campos e métodos
  }

Configuração da inspeção:

  • Use esta tabela para especificar as classes factory de logger e os métodos factory de logger reconhecidos por esta inspeção.
  • Use a opção Ignorar loggers inicializados com a superclasse para ignorar loggers inicializados com uma superclasse da classe que contém o logger.
  • Use Ignorar loggers em classes não public para emitir avisos apenas para loggers em classes public.
  • Use Ignorar loggers não inicializados em campos final para relatar apenas loggers inicializados em campos final, ignorando outros casos.
