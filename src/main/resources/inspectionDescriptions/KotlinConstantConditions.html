Relata condições e valores não triviais conhecidos estaticamente como sendo sempre true, false, null ou zero. Embora às vezes seja intencional, isso geralmente é um sinal de erros de lógica no programa. Além disso, relata ramos when inacessíveis e certas expressões conhecidas estaticamente por sempre falharem.

Exemplo:


fun process(x: Int?) {
  val isNull = x == null
  if (!isNull) {
    if (x != null) {} // a condição é sempre true
    require(x!! < 0 && x > 10) // a condição é sempre false
  } else {
    println(x!!) // o operador !! sempre falhará
  }
}
fun process(v: Any) {
  when(v) {
    is CharSequence -> println(v as Int) // o cast sempre falhará
    is String -> println(v) // o ramo é inacessível
  }
}

Desmarque a opção "Avisar quando constantes forem armazenadas em variáveis" para evitar o relato de variáveis com valores constantes que não estejam em condições.
