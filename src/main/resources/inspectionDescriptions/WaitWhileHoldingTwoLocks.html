Relata chamadas ao método wait() que podem ocorrer enquanto a thread atual detém dois locks.

Chamar wait() libera apenas um lock no alvo, portanto, aguardar enquanto se detém dois locks pode facilmente levar a um deadlock.

Exemplo:


  synchronized (lockA) {
    synchronized (lockB) {
      lockB.wait(); // aviso
      // a thread A fica travada aqui, mantendo o lockA
    }
  }

  synchronized (lockA) { // a thread B não consegue entrar neste bloco e liberar a thread A
    lockB.notify();
  }
