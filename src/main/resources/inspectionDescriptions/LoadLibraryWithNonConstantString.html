Relata chamadas para java.lang.System.loadLibrary(), java.lang.System.load(), java.lang.Runtime.loadLibrary() e java.lang.Runtime.load() que usam uma string construída dinamicamente como o nome da biblioteca.

Strings de nomes de biblioteca construídas são uma fonte comum de vulnerabilidades de segurança. Por padrão, esta inspeção ignora constantes de tempo de compilação.

Exemplo:


  void test(int i) {
    System.loadLibrary("foo" + i);
  }

Use as configurações da inspeção para tratar qualquer campo static final como uma constante. Tenha cuidado, pois quando essa opção estiver ativada, strings como a seguinte serão ignoradas: 


  private static final String LIBRARY = getUserInput();
