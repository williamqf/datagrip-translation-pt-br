Relata múltiplas verificações consecutivas de propriedades de java.io.File, como:
  • isDirectory()
  • isFile()
  • lastModified()
  • length()
Tais chamadas podem ser substituídas por uma chamada em lote a Files.readAttributes(). Isso geralmente oferece um desempenho melhor do que múltiplas verificações de propriedades individuais.

Exemplo:


  boolean isNewFile(File file, long lastModified) throws IOException {
    return file.isFile() && file.lastModified() > lastModified;
  }

Após a aplicação da correção rápida:


  boolean isNewFile(File file, long lastModified) throws IOException {
    var fileAttributes = Files.readAttributes(file.toPath(), BasicFileAttributes.class);
    return fileAttributes.isRegularFile() && fileAttributes.lastModifiedTime().toMillis() > lastModified;
  }

Esta inspeção não exibirá um aviso se IOException não for tratada no contexto atual, mas a correção rápida ainda estará disponível.

Observe que as substituições geralmente não são exatamente idênticas e devem ser aplicadas com cautela. Em particular, o comportamento pode ser diferente se o arquivo não existir.

Esta inspeção relata apenas se o nível de linguagem do projeto ou módulo for 7 ou superior.
