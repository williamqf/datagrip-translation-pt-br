Relata problemas com campos AtomicLongFieldUpdater, AtomicIntegerFieldUpdater ou AtomicReferenceFieldUpdater (pacote java.util.concurrent.atomic).

Os problemas relatados são os mesmos problemas de tempo de execução que podem ocorrer com atualizadores de campos atômicos: o campo especificado não foi encontrado, o campo especificado não está acessível, o tipo do campo especificado está incorreto, etc.

Exemplos:

  • 
  class A {
    private static volatile int value = 0;
    private static final AtomicIntegerFieldUpdater updater =
    AtomicIntegerFieldUpdater.newUpdater((A.class), "value"); // Alerta: o campo 'value' possui o modificador 'static'
  }
  
  • 
  class B {
    private static final AtomicIntegerFieldUpdater updater =
      AtomicIntegerFieldUpdater.newUpdater(B.class, "value"); // Alerta: o campo chamado 'value' não foi encontrado na classe 'B'
  }
