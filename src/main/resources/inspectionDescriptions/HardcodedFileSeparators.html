Relata barras (/) ou barras invertidas (\) em literais de string ou de caractere. Esses caracteres são frequentemente usados como separadores de arquivos e, se forem codificados de forma rígida (hardcoded), a portabilidade pode ser afetada.

Esta inspeção não relata barras invertidas dentro de sequências de escape, nem barras imediatamente após o caractere '<' ou imediatamente antes do caractere '>', pois estas frequentemente indicam tags XML ou HTML em vez de nomes de arquivo. Strings que representam IDs de java.util.TimeZone, strings que sejam expressões regulares válidas ou strings equivalentes a tipos de mídia MIME registrados na IANA também não são relatadas.

Exemplo:


  new File("C:\\Users\\Name");

Use a opção para incluir example/* no conjunto de tipos de mídia reconhecidos. Geralmente, é um erro usar o tipo de mídia MIME example/* fora de exemplos (por exemplo, em um cabeçalho Content-Type).
