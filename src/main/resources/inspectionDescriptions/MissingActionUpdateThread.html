Relata ações, grupos de ações e outras classes ActionUpdateThreadAware que não especificam explicitamente o seu modo.

Quando uma ação ou um grupo de ações define seu próprio método update(), a plataforma IntelliJ tenta emular a antiga forma síncrona de chamar os métodos update() e getChildren() na thread de UI e fornece todos os dados do AnActionEvent.dataContext(). Para fazer isso, ela pré-armazena em cache todos os dados possíveis em uma thread de segundo plano (mesmo que não sejam necessários).

Sobrescreva o método getActionUpdateThread() para fornecer um dos dois modos: ActionUpdateThread.EDT ou ActionUpdateThread.BGT.

Para mais detalhes, consulte a documentação de ActionUpdateThread.
