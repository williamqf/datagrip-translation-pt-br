Relata elementos incorretos em arquivos de Bean Validation <constraint-mappings>:

  • Referência a classes que não podem ser instanciadas
  • Referência a uma classe que não estende a classe necessária
  • Referência a uma classe com escopo inadequado
  • Tags e valores de atributos vazios
  • Valores de tags e atributos que não correspondem ao padrão necessário (por exemplo, identificadores Java)
  • Tags que não contêm tags filhas ou atributos necessários
  • Tags que definem objetos com nomes duplicados

Exemplo:


<constraint-mappings version="1.1">
    <bean class="my.UserClass" ignore-annotations="false">
        <field name="name">
            <constraint annotation="java.lang.String"/>  <!--'java.lang.String' não é uma anotação Java -->
        </field>

        <field name="name">
            <constraint annotation="javax.validation.constraints.Size">
                <element name="min">non-number</element>    <!-- deve ser um número entre '-2.147.483.648' e '2.147.483.647' -->
            </constraint>
        </field>

        <getter name="zzz"> <!-- Não é possível resolver o símbolo 'zzz' -->
            <valid/>
        </getter>

    </bean>
</constraint-mappings>
