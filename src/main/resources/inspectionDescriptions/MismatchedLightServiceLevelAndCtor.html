Relata incompatibilidades entre o nível de um serviço leve e seu construtor.

Os seguintes problemas serão relatados:

  • Uma classe de serviço leve que possui um construtor com dois parâmetros dos tipos com.intellij.openapi.project.Project e kotlinx.coroutines.CoroutineScope, ou um parâmetro do tipo Project, mas não está anotada como um serviço em nível de projeto.
  • Uma classe de serviço leve anotada como um serviço em nível de aplicativo que não possui um construtor sem argumentos nem um construtor com um único parâmetro do tipo CoroutineScope.

Exemplo (Kotlin):


@Service                              // Sugere especificar o parâmetro 'Service.Level.PROJECT' na anotação '@Service'
class MyService(val project: Project) // Sugere remover o parâmetro do construtor 'MyService'

Após aplicar a correção rápida para especificar o parâmetro 'Service.Level.PROJECT' na anotação '@Service':


@Service(Service.Level.PROJECT)
class MyService(val project: Project)

Após aplicar a correção rápida para remover o parâmetro do construtor 'MyService':


@Service
class MyService()
