Relata o uso de @JvmOverloads no construtor de uma classe de anotação, pois ele não possui efeito.

Não há presença do @JvmOverloads no bytecode gerado nem nos metadados do Kotlin; portanto, o @JvmOverloads não afeta o bytecode gerado nem o comportamento do código.

A partir do Kotlin 1.4, o uso de @JvmOverloads em construtores de classes de anotação resulta em um erro de compilação.

Exemplo:


  annotation class A @JvmOverloads constructor(val x: Int = 1)

Após a aplicação da correção rápida:


  annotation class A constructor(val x: Int = 1)
