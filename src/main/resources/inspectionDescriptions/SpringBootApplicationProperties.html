Relata chaves de configuração não resolvidas e obsoletas, bem como valores inválidos em arquivos de configuração .properties de aplicativos Spring Boot, os quais podem levar a erros de tempo de execução.

Exemplo:


server.port=invalid # relata 'Não é possível converter 'invalid' para java.lang.Integer'

Se uma chave de configuração obsoleta tiver uma chave de substituição, você poderá aplicar a correção rápida 'Usar chave de substituição'.

Exemplo:


logging.path=${path} # relata 'Propriedade de configuração obsoleta 'logging.path''

Após aplicar a correção rápida:


logging.file.path=${path}

Se uma chave de configuração não estiver definida no spring-configuration-metadata.json, você poderá aplicar a correção rápida 'Definir chave de configuração' para criar o arquivo META-INF/spring-additional-configuration-metadata.json e definir as chaves necessárias.

Exemplo:


new.key=value # relata 'Não é possível resolver a propriedade de configuração 'new.key''

Após aplicar a correção rápida, o seguinte conteúdo será adicionado ao META-INF/spring-additional-configuration-metadata.json:


{
  "properties": [
    {
      "name": "new.key",
      "type": "java.lang.String",
      "description": "Description for new.key."
  }
] }

A inspeção também realça erros de notação de índice em chaves de configuração de listas e mapas.

Exemplo:


spring.datasource.schema[]=${schema} # relata 'Valor do índice ausente'

Use a opção Tokens de substituição para definir tokens usados para placeholders de valores nos arquivos de configuração. Esses tokens são especificados na forma beginToken*endToken. Se não houver *, assume-se que os tokens de início e fim são iguais.

Por exemplo, o valor padrão para os tokens de início e fim é @, o que permite definir placeholders como some.property=@another.property@.

Valores dentro de tokens de substituição (@property.key@) não serão realçados.
