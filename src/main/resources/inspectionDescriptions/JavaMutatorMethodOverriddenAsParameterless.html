Relata métodos sem cláusulas de parâmetros que substituem métodos Java com nomes tipo modificadores.

Scala permite substituir métodos Java sem parâmetros sem cláusulas de parâmetros. Isso ocorre principalmente para permitir que o código Scala substitua métodos acessores Java da mesma forma que métodos acessores Scala. Ou seja, sem cláusula de parâmetro e, portanto, aderir ao princípio de acesso uniforme, que diz que o código do cliente não deve ser afetado pela decisão de implementar um atributo como um campo ou método.

No entanto, para métodos com efeitos colaterais, é convencional ter uma cláusula de parâmetro vazia. Isso estaria de acordo com o princípio da substituição de Liskov, segundo o qual, quando um método substituído tem efeitos colaterais, o método de substituição também deve ser declarado como um método com efeitos colaterais.

A correção rápida adiciona uma cláusula de parâmetro vazia.

Exemplo:


  import java.util

  class Test extends util.ArrayList[String] {
    override def clear: Unit = ???
  }

Após a correção rápida ser aplicada:


  import java.util

  class Test extends util.ArrayList[String] {
    override def clear(): Unit = ???
  }
