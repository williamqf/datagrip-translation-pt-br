Relata os seguintes problemas em arquivos de configuração XML:
  • Referências a classes não instanciáveis
  • Referências a classes que não estendem a classe exigida
  • Referências a classes com escopo inadequado
  • Valores de tags e atributos que não correspondem ao padrão exigido (por exemplo, identificadores Java)
  • Tags que não contêm tags filhas ou atributos obrigatórios
  • Tags que definem objetos com nomes duplicados

Exemplo:



<persistence>
    <persistence-unit name="example-persistence-unit">
        <provider>org.example.SomeRandomClass</provider> // Erro: a classe provider deve implementar a interface PersistenceProvider
    </persistence-unit>
</persistence>
