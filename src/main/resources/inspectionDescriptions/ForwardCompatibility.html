Relata construções de código Java que podem falhar na compilação em versões futuras do Java.

Os problemas relatados são os seguintes:

  • Uso de assert, enum ou _ como identificadores
  • Uso dos identificadores restritos var, yield ou record como nomes de tipos
  • Chamadas não qualificadas ao método yield
  • Modificadores na instrução requires java.base dentro de module-info.java
  • Pontos e vírgulas redundantes entre instruções import

Exemplo:


  // Esta classe, anteriormente válida, não compila no Java 14,
  // pois 'yield' tornou-se um identificador restrito.
  public class yield {}

Corrigir esses problemas oportunamente pode simplificar a migração para versões futuras do Java.
