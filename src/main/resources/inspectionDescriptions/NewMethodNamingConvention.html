Relata métodos com nomes muito curtos, muito longos ou que não seguem o padrão de expressão regular especificado.

Métodos de instância que sobrescrevem métodos de biblioteca e construtores são ignorados por esta inspeção.

Exemplo: se esta inspeção estiver habilitada para métodos static e o comprimento mínimo especificado para o nome do método for 4 (padrão), o seguinte método static produzirá um aviso porque o comprimento do seu nome é 3, que é menor que 4: public static int max(int a, int b).

A correção rápida para renomear tais métodos está disponível apenas no editor.

Configurar inspeção:

Use a lista na seção Opções para especificar quais métodos devem ser verificados. Desmarque a caixa de seleção para os tipos de métodos que deseja ignorar. Especifique 0 nos campos de comprimento para ignorar a verificação correspondente.

As expressões regulares devem ser especificadas no formato padrão java.util.regex.
