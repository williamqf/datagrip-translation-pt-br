Relata propriedades que não seguem as convenções de nomenclatura.

As seguintes propriedades são relatadas:

  • Propriedades de nível superior
  • Propriedades em objetos e companion objects

Você pode especificar o padrão desejado nas opções de inspeção.

Convenções de nomenclatura sugeridas: devem começar com uma letra maiúscula, usar CamelCase e não conter sublinhados.

Exemplo:


  // Propriedade de nível superior
  val USER_NAME_FIELD = "UserName"
  // Propriedade de nível superior que mantém uma referência a um objeto singleton
  val PersonComparator: Comparator<Person> = /*...*/

  class Person {
    companion object {
      // Propriedade em um companion object
      val NO_NAME = Person()
    }
  }
