Relata instâncias em que palavras-chave reservadas do SQL são usadas como nomes de identificadores em consultas.

Exemplo (Microsoft SQL Server):

CREATE TABLE select (identity INT IDENTITY NOT NULL, order INT NOT NULL);

Usamos select, identity e order como nomes de tabela e coluna. No entanto, eles também são palavras-chave reservadas no Microsoft SQL Server. Portanto, para usá-los como nomes de objetos em consultas, você deve colocar esses identificadores entre aspas. Para colocá-los entre aspas, você pode usar a correção rápida Colocar identificador entre aspas.

Após aplicar a correção rápida:

CREATE TABLE [select] ([identity] INT IDENTITY NOT NULL, [order] INT NOT NULL);
