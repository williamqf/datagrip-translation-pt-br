Reporta chamadas que usam plus/minus na forma de operador (a + b, a - b) ou na forma de chamada regular (a.plus(b), a.minus(b)) para adicionar ou remover um java.nio.file.Path de coleções ou sequências Kotlin.

Como o java.nio.file.Path implementa Iterable<Path>, essas chamadas são resolvidas como uma sobrecarga inesperada das funções plus ou minus que aceitam uma coleção de elementos (neste caso, os elementos individuais do Path). No entanto, a intenção do código provavelmente é adicionar ou remover o próprio Path, e não os elementos individuais.

Exemplos:


// Forma de operador
val paths = listOf(path) + somePath
val paths2 = setOf(path) - somePath

// Forma de chamada regular
val paths = listOf(path).plus(somePath)
val paths2 = setOf(path).minus(somePath)

Correções rápidas:

  • Converter para plusElement/minusElement (muda a semântica para a intenção original): 
    
val paths = listOf(path).plusElement(somePath)
val paths2 = setOf(path).minusElement(somePath)
  • Converter o argumento Path em uma coleção para esclarecer a intenção sem alterar a semântica:
    • Para plus: envolva o argumento com toList() para preservar a ordem.
    • Para minus: envolva o argumento com toSet() para remoção eficiente.
    
val paths = listOf(path).plus(somePath.toList())
val paths2 = setOf(path).minus(somePath.toSet())
