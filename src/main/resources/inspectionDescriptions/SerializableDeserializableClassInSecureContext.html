Relata classes que podem ser serializadas ou desserializadas.

Se uma classe suportar a interface Serializable e seus métodos readObject() e writeObject() não forem definidos para sempre lançar uma exceção, ela poderá ser serializada. Classes serializáveis podem ser perigosas em códigos destinados ao uso seguro.

Exemplo:


  class DeserializableClass implements Serializable { // A classe não contém um método 'writeObject()' que lança uma exceção
    private int sensitive = 736326;

    private void readObject(ObjectInputStream in) {
        throw new Error();
    }
}

Após a aplicação do ajuste rápido:


  class DeserializableClass implements Serializable {
    private int sensitive = 736326;

    private void readObject(ObjectInputStream in) {
        throw new Error();
    }

    private void writeObject(java.io.ObjectOutputStream out) throws java.io.IOException {
        throw new java.io.NotSerializableException("DeserializableClass");
    }
  }

Configure a inspeção usando as seguintes opções:

  • Liste as classes cujos sucessores não devem ser relatados por esta inspeção. Isso se aplica a classes que herdam Serializable de uma superclasse, mas não se destinam à serialização. Observe que, em vez de ignorar essas classes, pode ser mais seguro adicionar os métodos readObject() e writeObject() que sempre lançam uma exceção.
  • Se as classes anônimas serializáveis devem ser ignoradas.
