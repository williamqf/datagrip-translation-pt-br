Relata chamadas a java.lang.Thread.sleep() feitas dentro de um loop.

Tais chamadas indicam uma "espera ocupada" (busy waiting). A espera ocupada é geralmente ineficiente e pode causar deadlocks inesperados, pois a thread em espera ocupada não libera os recursos bloqueados.

Exemplo:


  class X {
     volatile int x;
     public void waitX() throws Exception {
      while (x > 0) {
        Thread.sleep(10);// Aviso: Chamada a 'Thread.sleep()' dentro de um loop, possivelmente uma espera ocupada
      }
    }
  }
