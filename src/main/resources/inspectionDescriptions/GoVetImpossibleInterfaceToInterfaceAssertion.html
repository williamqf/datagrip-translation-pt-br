Relata asserções de tipo impossíveis de interface para interface.

Verifica asserções de tipo v.(T) e os casos de switch de tipo correspondentes em que o tipo estático V de v é uma interface que não pode implementar a interface de destino T. Isso ocorre quando V e T contêm métodos com o mesmo nome, mas assinaturas diferentes.

Exemplo:

var v interface {
  Read()
}
_ = v.(io.Reader)

O método Read em v possui uma assinatura diferente do método Read em io.Reader, portanto, essa asserção falhará.

Esta inspeção é relatada apenas se a versão da linguagem for 1.15 ou superior.
