Relata códigos que usam == ou != em vez de equals() para testar a igualdade de Number.

Ao usar autoboxing, é fácil cometer o erro de comparar duas instâncias de tipos wrapper em vez de dois tipos primitivos, por exemplo, comparando Integer em vez de int.

Exemplo:

  void foo(Integer a, Integer b) {
    final boolean bool = a == b;
  }

Se for conhecido que a não é nulo, a correção rápida "insegura" pode ser aplicada com segurança para obter um resultado semelhante ao seguinte:

  void foo(Integer a, Integer b) {
    final boolean bool = a.equals(b);
  }
