Relata classes que implementam java.lang.Comparable, mas não sobrescrevem equals().

Se equals() não for sobrescrito, a implementação de equals() será inconsistente com a implementação de compareTo(). Se objetos de tal classe forem adicionados a uma coleção como java.util.SortedSet, essa coleção violará o contrato de java.util.Set definido em termos de equals().

Exemplo:


  class Length implements Comparable<Length> {
    private int cm = 0;

    @Override
    public int compareTo(@NotNull Length o) {
      if (cm == o.cm) return 0;
      return cm < o.cm ? -1 : 1;
    }
  }

Após a aplicação da correção rápida:


  class Length implements Comparable<Length> {
    private int cm = 0;

    @Override
    public int compareTo(@NotNull Length o) {
      if (cm == o.cm) return 0;
      return cm < o.cm ? -1 : 1;
    }

    @Override
    public boolean equals(Object o) {
      return o instanceof Length && compareTo((Length) o) == 0;
    }
  }
