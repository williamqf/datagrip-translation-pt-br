Relata operações de atribuição que podem ser substituídas por atribuições compostas.

O código que utiliza operadores de atribuição é mais curto e, possivelmente, mais claro.

Exemplo:

  x = x + 3;
  x = x / 3;

Após a aplicação da correção rápida:

  x += 3;
  x /= 3;

Utilize a opção Ignorar operadores condicionais para ignorar && e ||. A substituição de operadores condicionais por atribuições compostas transformará a avaliação preguiçosa (lazy evaluation) em avaliação imediata (eager evaluation), o que pode alterar a semântica da expressão.

Utilize a opção Ignorar operadores ambíguos para ignorar os operadores menos conhecidos ^ e %.
