Relata as chamadas para métodos com nomes de semelhante a mutadores sem cláusulas de argumento.

Scala permite chamar métodos Java sem parâmetros sem cláusulas de argumento. Isto é principalmente para permitir o código Scala utilizar métodos de acesso Java sem parâmetros da mesma forma dos métodos de acesso Scala. Nomeadamente, sem qualquer cláusula de argumento e aderindo, por conseguinte, aoprincípio de acesso uniforme, que diz que o código cliente não deve ser afetado pela decisão de implementar um atributo como campo ou método.

Porém, para métodos com efeitos colaterais, por convenção é utilizado uma cláusula de argumento vazia.

A correção rápida adiciona uma cláusula de argumento vazia.

Exemplo:


  val stringBuilder = new java.util.ArrayList[String]
  stringBuilder.clear

Após a correção rápida ter sido aplicada:


  val stringBuilder = new java.util.ArrayList[String]
  stringBuilder.clear()
