Relata recursos de socket que não foram fechados com segurança. Os recursos de socket relatados por esta inspeção incluem java.net.Socket, java.net.DatagramSocket e java.net.ServerSocket.

Por padrão, esta inspeção assume que os recursos podem ser fechados por qualquer método que contenha 'close' ou 'cleanup' em seu nome.

Exemplo:


  byte[] getMessage(ServerSocket socket) throws IOException {
    Socket client = socket.accept(); // O socket não foi fechado
    return client.getInputStream().readAllBytes();
  }

Configure a inspeção usando as seguintes opções:

  • Se deve permitir a abertura de sockets dentro de blocos try. Este estilo é menos desejável por ser mais verboso do que abrir o recurso antes do bloco try.
  • Se o recurso pode ser fechado por qualquer chamada de método na qual o recurso seja passado como um argumento.
