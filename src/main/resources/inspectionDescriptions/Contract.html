Relata problemas em anotações @Contract de métodos. Os tipos de problemas que podem ser relatados são:
  • Erros de sintaxe do contrato
  • Contratos que não estão em conformidade com a assinatura do método (contagem de parâmetros incorreta)
  • Implementação do método inconsistente com o contrato (por exemplo, retornando true mesmo quando o contrato declara false)

Exemplo:


  // o método não possui parâmetros, mas o contrato espera 1
  @Contract("_ -> fail")
  void x() {
    throw new AssertionError();
  }
