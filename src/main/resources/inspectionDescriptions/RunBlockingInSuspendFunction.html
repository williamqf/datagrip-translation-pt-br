Relata chamadas runBlocking dentro de funções suspend.

O uso de runBlocking dentro de uma função suspend bloqueia a thread de chamada, contrariando o propósito da programação assíncrona.

A correção rápida pode substituir a chamada runBlocking por uma das seguintes alternativas, dependendo do contexto:

  • Uma chamada run.
  • Uma chamada withContext ao usar um CoroutineContext específico.
  • Código direto em linha sem o wrapper runBlocking.

Exemplo:


suspend fun something() {
  runBlocking {
    code() // A thread é bloqueada aqui
  }
}

Após aplicar a correção rápida:


suspend fun something() {
  code() // Executa de forma assíncrona
}
