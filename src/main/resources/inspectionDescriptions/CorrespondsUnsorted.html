Relata os usos de sameElements e corresponds com coleções não ordenadas.

Os métodos sameElements e corresponds usam iteradores das coleções fornecidas para comparar elementos dependendo da ordem de suas posições na coleção. O resultado é, portanto, imprevisível se a ordem dos elementos não estiver bem definida, como é o caso para conjuntos ou mapas não ordenados.

Para corrigir isso, considere usar == , subsetOf ou colocar os elementos em alguma ordem.

Exemplo:


  Set(3, 2, 1) sameElements Set(1, 2, 3) // é falso

  // mas

  Set(1, 2, 3, 4, 5) sameElements Set(5, 4, 3, 2, 1) // é verdadeiro
