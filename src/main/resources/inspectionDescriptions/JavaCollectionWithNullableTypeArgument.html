Relata casos em que argumentos de tipo anulável são fornecidos a coleções Java que não aceitam null.

Algumas coleções Java fornecidas na biblioteca padrão do Java não suportam elementos, chaves ou valores null. Os argumentos de tipo para tais coleções não devem ser instanciados com tipos anuláveis.

Exemplo:


val map = ConcurrentHashMap<String, String?>()

Após a aplicação do ajuste rápido:


val map = ConcurrentHashMap<String, String>()
