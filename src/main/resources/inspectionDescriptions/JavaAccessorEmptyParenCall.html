Informa as chamadas para métodos de acesso Java com cláusulas de argumento vazias.

Espera-se que os métodos que seguem o contrato JavaBean de nomenclatura para métodos de acesso não tenham efeitos colaterais. A convenção recomendada é usar um método sem parâmetros sempre que não houver parâmetros e o método não tiver efeito colateral. Esta convenção promove o princípio de acesso uniforme, que diz que o código do cliente não deve ser afetado pela decisão de implementar um atributo como um campo ou método.

O problema é que Java não implementa o princípio de acesso uniforme. Para preencher essa lacuna, o Scala permite que você deixe de fora os parênteses vazios em uma invocação de um método Java que não recebe argumentos.

A correção rápida remove a cláusula de argumento vazia.

Exemplo:


  "test".getClass()

Depois que a correção rápida é aplicada:


  "test".getClass
