Relata expressões aritméticas sem sentido. Tais expressões incluem adição ou subtração de 0, multiplicação por 0 ou 1 e divisão por 1.

Tais expressões podem ser resultado de uma refatoração automática e é improvável que sejam a intenção original do desenvolvedor.

A correção rápida simplifica essas expressões.

Exemplo: 


  void f(int a) {
    int x = a - a;
    int y = a + 0;
    int res = x / x;
  }

Após a aplicação da correção rápida:


  void f(int a) {
    int x = 0;
    int y = a;
    int res = 1;
  }

Observe que, em casos raros, a substituição sugerida pode não ser totalmente equivalente ao código original para todas as entradas possíveis. Por exemplo, esta inspeção sugere substituir x / x por 1. No entanto, se x for zero, o código original lançará uma ArithmeticException ou resultará em NaN. Além disso, se x for NaN, o resultado também será NaN. É improvável que tal comportamento seja intencional.
