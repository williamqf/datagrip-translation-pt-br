Relata o uso incorreto de anotações de caminho.

Esta inspeção detecta casos em que uma string com uma anotação de caminho é usada em um contexto que exige uma anotação de caminho diferente. Isso ajuda a garantir o uso correto das anotações @MultiRoutingFileSystemPath e @NativePath.

A inspeção destaca os seguintes problemas:

  • Quando uma string anotada com @NativePath é usada em um construtor de Path ou método factory
  • Quando uma string anotada com @NativePath é passada para um parâmetro de método anotado com @MultiRoutingFileSystemPath
  • Quando uma string anotada com @MultiRoutingFileSystemPath é passada para um parâmetro de método anotado com @NativePath
  • Quando um literal de string é usado em um contexto que exige qualquer uma das anotações

Correções rápidas (quick fixes) são fornecidas para adicionar a anotação apropriada onde necessário.

Esta inspeção ajuda a evitar erros de tempo de execução que podem ocorrer quando caminhos de formatos diferentes são usados incorretamente. A anotação @MultiRoutingFileSystemPath é usada para caminhos que devem funcionar em diferentes sistemas de arquivos, enquanto a anotação @NativePath é usada para caminhos específicos do sistema de arquivos nativo.
