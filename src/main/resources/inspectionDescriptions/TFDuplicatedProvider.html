Relata provedores duplicados.

Exemplo de problema:


provider "kubernetes" {
  config_path = "~/.kube/config"
}

provider "kubernetes" {
  config_path = "~/kube/kubeconfig-0728633f-2c4a-4d2b-9eb7-bc629d5e7f37"
}

Uma das maneiras de resolver esse problema é adicionar um alias e especificá-lo explicitamente ao criar recursos no provedor desejado.

Exemplo de correção:


  provider "kubernetes" {
  config_path = "~/.kube/config"
}

provider "kubernetes" {
  config_path = "~/kube/kubeconfig-0728633f-2c4a-4d2b-9eb7-bc629d5e7f37"
  alias = "cloud"
}
