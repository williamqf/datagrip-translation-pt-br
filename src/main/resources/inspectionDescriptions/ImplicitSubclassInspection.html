Relata situações em que o código impede que um framework (como Spring ou Hibernate) crie subclasses de uma classe em tempo de execução.

Exemplos típicos onde a subclassificação é necessária, mas impossível:

  • Classes final marcadas com anotações específicas de framework (por exemplo, @Configuration do Spring)
  • Métodos final, static ou private marcados com anotações específicas de framework (por exemplo, @Transactional do Spring)
  • Métodos marcados com anotações específicas de framework dentro de uma classe final

A lista de situações relatadas depende do framework utilizado.
