Relata instruções throw cujas exceções são sempre capturadas pela instrução try envolvente.

O uso de instruções throw como um "goto" para alterar o fluxo de controle local é confuso e resulta em desempenho insatisfatório.

Exemplo:


  try {
    if (!Files.isDirectory(PROJECTS)) {
        throw new IllegalStateException("Directory not found."); // Aviso: 'throw' capturado pela instrução 'try' envolvente
    }
    ...
  } catch (Exception e) {
      LOG.error("run failed");
  }

Use a opção Ignorar exceções relançadas para ignorar exceções que são lançadas novamente.
