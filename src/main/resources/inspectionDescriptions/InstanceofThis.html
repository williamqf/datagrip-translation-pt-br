Relata usos de instanceof ou getClass() == SomeClass.class nos quais a expressão this é verificada.

Tais expressões indicam uma falha no design orientado a objetos e devem ser substituídas por construções polimórficas.

Exemplo:


class Super {
  void process() {
    if (this instanceof Sub) { // aviso
      doSomething();
    } else {
      doSomethingElse();
    }
  }
}
  
class Sub extends Super {}

Para corrigir o problema, você pode utilizar a sobrescrita de métodos:


class Super {
  void process() {
    doSomethingElse();
  }
}
  
class Sub extends Super {
  @Override
  void process() {
    doSomething();
  }
}
