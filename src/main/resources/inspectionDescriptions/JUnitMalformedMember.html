Inspeciona a declaração de membros JUnit com uma formatação incorreta que provavelmente não serão reconhecidos pelo framework JUnit. Esta inspeção relata problemas como:
  • Campos anotados com @RegisterExtension que possuem o tipo errado ou que não foram declarados como static quando necessário.
  • Classes internas estáticas anotadas com @Nested.
  • Testes parametrizados definidos sem uma fonte.
  • Testes parametrizados com MethodSource contendo alvos desconhecidos, não estáticos ou sem argumentos.
  • Testes parametrizados com ValueSource ou EnumSource que não podem ser convertidos para parâmetros de método.
  • Testes anotados com @Test, @ParameterizedTest e @RepeatedTest simultaneamente.
  • Métodos setup() ou tearDown()
    • não públicos, com tipo de retorno diferente de void ou que aceitam argumentos.
  • Métodos não públicos ou não estáticos anotados com @BeforeClass, @AfterClass, @BeforeAll ou @AfterAll, com tipo de retorno diferente de void ou sem uma lista válida de parâmetros.
  • Métodos não públicos anotados com @Before, @After, @BeforeEach ou @AfterEach, com tipo de retorno diferente de void ou que aceitam argumentos.
  • RepetitionInfo injetado no método @BeforeAll ou @AfterAll.
  • RepetitionInfo injetado nos métodos @BeforeEach ou @AfterEach usados por @Test anotados em teste.
  • Campos e métodos anotados com @DataPoint ou @DataPoints que não são públicos ou estáticos.
  • Campos e métodos anotados com @Rule que não são públicos ou não são subtipos de TestRule ou MethodRule.
  • Campos e métodos anotados com @ClassRule que não são públicos, estáticos ou subtipos de TestRule.
  • Métodos não públicos no TestCase que possuem o prefixo test, cujo tipo de retorno não é void, aceitam argumentos ou são estáticos.
  • Métodos não públicos anotados com @Test, cujo tipo de retorno não é void, aceitam argumentos ou são estáticos.

Exemplo de método @Before com formatação incorreta (Java):

@Before private int foo(int arg) { ... }

Após a aplicação da correção rápida:

@Before public void foo() { ... }

Exemplo de método de origem ausente (Kotlin):


  class Example {
    @MethodSource("parameters")
    @ParameterizedTest
    fun foo(param: String) { ... }
  }

Após a aplicação da correção rápida:


  class Example {
    @MethodSource("parameters")
    @ParameterizedTest
    fun foo(param: String) { ... }

    companion object {
      @JvmStatic
      fun parameters(): Stream<Arguments> {
        TODO("Not yet implemented")
      }
    }
  }

As anotações podem ser especificadas usando as opções de inspeção. Os parâmetros anotados com uma dessas anotações não serão relatados nos métodos de teste.
