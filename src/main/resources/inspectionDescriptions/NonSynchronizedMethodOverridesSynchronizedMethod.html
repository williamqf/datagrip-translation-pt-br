Relata métodos não synchronized que sobrescrevem métodos synchronized.

Se um método da superclasse for declarado como synchronized, o método sobrescrito não será sincronizado automaticamente. Isso pode resultar em condições de corrida inesperadas ao usar a subclasse.

Exemplo:


  class Super {
    synchronized void process() {}
  }
  class Sub extends Super {
    // O método não sincronizado 'process()' sobrescreve um método sincronizado
    void process() {}
  }
