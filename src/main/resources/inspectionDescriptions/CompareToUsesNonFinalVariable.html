Relata o acesso a campos não final em implementações de compareTo().

Esse acesso pode fazer com que compareTo() retorne resultados diferentes em diferentes momentos do ciclo de vida do objeto, o que pode causar problemas ao usar classes de coleção padrão, como java.util.TreeSet.

Uma correção rápida para tornar o campo final só estará disponível se não houver acesso de escrita ao campo; caso contrário, nenhuma correção será sugerida.

Exemplo:


  class Foo implements Comparable<Foo>{
    private int index;
    Foo(int idx) {
      index = idx;
    }
    @Override
    public int compareTo(Foo foo) {
     return Integer.compare(this.index, foo.index);
    }
  }

Após a aplicação da correção rápida:


  class Foo implements Comparable<Foo>{
    private final int index;
    Foo(int idx) {
      index = idx;
    }
    @Override
    public int compareTo(Foo foo) {
      return Integer.compare(this.index, foo.index);
    }
  }
