Relata métodos que possuem a mesma assinatura de um método inacessível de uma superclasse, por exemplo, um método private ou um método package-private de uma superclasse em outro pacote.

Tais nomes de métodos podem ser confusos, pois o método na subclasse pode parecer uma sobrescrita, quando na verdade está apenas ocultando um método inacessível da superclasse. Além disso, se a visibilidade deste método na superclasse for alterada futuramente, isso poderá alterar silenciosamente a semântica da subclasse ou resultar em um erro de compilação.

A correção rápida sugere renomear o método.

Exemplo:


  public class Super {
    private void test() {
    }
  }

  public class Sub extends Super {
    void test() { // Tornar 'Super.test()' public resultará em um erro de compilação
                  // Tornar 'Super.test()' package-private fará com que 'Sub.test()' se torne uma sobrescrita
    }
  }
