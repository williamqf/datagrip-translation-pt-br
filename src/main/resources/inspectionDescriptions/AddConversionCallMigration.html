Relata expressões que resultarão no tipo Int, o que causará erros de compilação no Kotlin 1.9 e versões superiores.

Exemplo:


  fun takeByte(x: Byte) {}

  fun foo() {
      takeByte(1 + 1) // será resolvido como Int na versão 1.9
  }

Após a aplicação da correção rápida:


  fun takeByte(x: Byte) {}

  fun foo() {
      takeByte((1 + 1).toByte()) // será resolvido como Int na versão 1.9
  }

A inspeção está disponível a partir do nível da linguagem Kotlin 1.7.
