Relata classes de teste parametrizado do JUnit 4 que usam a anotação @RunWith(Parameterized.class), mas não incluem um método provedor de dados com a anotação @Parameterized.Parameters ou a assinatura desse método está incorreta. Tais classes de teste não podem ser executadas. Os métodos provedores de dados devem ser public e static e ter um tipo de retorno Iterable ou Object[].

Sugere a criação de um método provedor de parâmetros vazio ou a alteração da assinatura do método provedor de dados incorreto.

Exemplo:


  @RunWith(Parameterized.class)
  public class ImportantTest {
    private int input;
    private int expected;

    ImportantTest(int input, int expected) {
      this.input = input;
      this.expected = expected;
    }

    // ... casos de teste
  }

Após a aplicação da correção rápida:


  @RunWith(Parameterized.class)
  public class ImportantTest {
    private int input;
    private int expected;

    ImportantTest(int input, int expected) {
      this.input = input;
      this.expected = expected;
    }

    @Parameters
    public static Iterable<Object[]> parameters() {
      return null;
    }

    // ... test cases
  }
