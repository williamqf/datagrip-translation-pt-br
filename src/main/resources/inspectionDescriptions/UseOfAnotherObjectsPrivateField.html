Relata o acesso a campos private ou protected de outro objeto. O Java permite o acesso a esses campos de objetos que pertencem à mesma classe que o objeto atual, mas alguns estilos de codificação desencorajam esse uso. Além disso, esse acesso direto a campos private pode falhar em arquiteturas orientadas a componentes (como Spring ou Hibernate), que esperam que todo o acesso a outros objetos ocorra por meio de chamadas de método para que o framework possa mediar o acesso usando proxies.

Exemplo:


  public class Base {
    protected int bar;

    void increment(Base base) {
      bar++;
      base.bar++; // aviso: acesso direto a um campo não público de outro objeto
    }
  }
O campo pode ser encapsulado por meio de uma correção rápida.

Configuração da inspeção:

  • Use a opção Ignorar acesso da mesma classe para ignorar acessos originados na mesma classe e relatar apenas acessos de classes internas ou externas.

    Para ignorar o acesso de classes internas, use a opção aninhada Ignorar acesso de classes internas.

  • Use Ignorar acesso do método 'equals()' para ignorar o acesso vindo do método equals().
