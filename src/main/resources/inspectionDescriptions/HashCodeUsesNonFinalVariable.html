Relata implementações de hashCode() que acessam variáveis não final.

Esse acesso pode fazer com que o hashCode() retorne valores diferentes em diferentes momentos do ciclo de vida do objeto, o que, por sua vez, pode causar problemas ao usar classes de coleção padrão.

Exemplo:


  class Drink {
    String name;
    Drink(String name) { this.name = name; }
    @Override public int hashCode() {
      return Objects.hash(name); //aviso
    }
  }
  ...
    Drink coffee = new Drink("Coffee");
    priceMap.put(coffee, 10.0);
    coffee.name = "Tea";
    double coffeePrice = priceMap.get(coffee); //não encontrado

Uma correção rápida é sugerida para tornar o campo final:


  class Drink {
    final String name;
    ...
  }
