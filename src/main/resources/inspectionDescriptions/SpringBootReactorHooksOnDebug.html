Relata o uso de Hooks.onOperatorDebug() quando ReactorDebugAgent.init() é chamado.

Se o ReactorDebugAgent estiver no classpath e spring.reactor.debug-agent.enabled for true (valor padrão), o ReactorDebugAgent.init() será chamado automaticamente na inicialização. O uso simultâneo de Hooks.onOperatorDebug() e ReactorDebugAgent.init() faz com que o Reactor gere stack frames de depuração duplicados. Além disso, o Hooks.onOperatorDebug() causa sobrecarga de desempenho.

A correção rápida remove a chamada para Hooks.onOperatorDebug().

Exemplo:


  public void hook() {
    Hooks.onOperatorDebug(); // Relata 'Hooks.onOperatorDebug() chamado durante a inicialização do ReactorDebugAgent'
  }

Após aplicar a correção rápida:


  public void hook() {
  }

Esta inspeção é acionada apenas quando a versão do Spring Boot do projeto ou módulo é 2.2.0 ou superior.
