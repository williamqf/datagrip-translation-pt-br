Relata métodos com cláusula de parâmetros vazia que sobrescrevem métodos de acesso Java.

Métodos que seguem o contrato de nomenclatura JavaBean para acessadores devem não ter efeitos colaterais. A convenção recomendada é usar um método sem parâmetros sempre que não houver parâmetros e o método não tiver efeitos colaterais. Esta convenção promove o princípio de acesso uniforme, que afirma que o código do cliente não deve ser afetado pela decisão de implementar um atributo como um campo ou método.

O problema é que o Java não implementa o princípio de acesso uniforme e não permite métodos sem cláusulas de parâmetros. Para preencher essa lacuna, o Scala permite que você sobrescreva métodos com cláusulas de parâmetros vazias com métodos sem cláusulas de parâmetros.

A correção rápida remove a cláusula de parâmetro vazia.

Exemplo:


  class TestException extends Exception {
    override def getMessage(): String = ???
  }

Após a correção rápida ser aplicada:


  class TestException extends Exception {
    override def getMessage: String = ???
  }
