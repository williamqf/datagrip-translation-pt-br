Relata classes anônimas que podem ser substituídas por referências de método.

Observe que, se uma classe anônima for convertida em uma referência de método não vinculada, o Java runtime poderá reutilizar o mesmo objeto de referência de método em chamadas subsequentes. Por outro lado, ao usar uma classe anônima, objetos distintos são criados a cada vez. Portanto, em casos raros, a aplicação da correção rápida pode resultar em uma mudança de semântica, por exemplo, quando uma instância de classe anônima é usada como uma chave de HashMap.

Exemplo:


  Runnable r = new Runnable() {
    @Override
    public void run() {
      System.out.println();
    }
  };

Após a aplicação da correção rápida:


  Runnable r = System.out::println;

Use a opção Relatar quando a interface não estiver anotada com @FunctionalInterface para habilitar esta inspeção para interfaces que não usam a anotação @FunctionalInterface.
