Relata variáveis atribuídas como null fora da declaração.

O principal objetivo do null no Java é representar variáveis de referência não inicializadas. Em casos raros, atribuir explicitamente uma variável como null ajuda na coleta de lixo (garbage collection). No entanto, o uso de null para representar valores ausentes, não especificados, inválidos ou elementos não encontrados é considerado uma má prática e pode tornar seu código mais propenso a NullPointerExceptions. Em vez disso, considere definir objetos sentinela com a semântica pretendida ou usar tipos de biblioteca como Optional para representar a ausência de valor.

Exemplo:


  Integer convert(String s) {
    Integer value;
    try {
      value = Integer.parseInt(s);
    } catch (NumberFormatException e) {
      // Aviso: null é usado para representar um 'valor inválido'
      value = null;
    }
    return value;
  }

Use a opção Ignorar atribuições a campos para ignorar atribuições a campos.
