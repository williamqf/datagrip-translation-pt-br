Relata a inicialização com chaves duplas.

A inicialização com chaves duplas tem um desempenho inferior em comparação com a inicialização convencional, pois requer o carregamento de uma classe adicional.

Ela também pode causar falhas na comparação equals() se o método equals() não aceitar subclasses como parâmetros.

O uso da inicialização com chaves duplas em um contexto não estático pode causar vazamentos de memória. Isso ocorre porque ela define uma classe anônima que, quando compilada com versões do javac anteriores ao Java 18, fará referência ao objeto circundante.

Além disso, em versões anteriores ao Java 9, a inicialização com chaves duplas não podia ser combinada com o operador diamond, pois era incompatível com classes anônimas.

Exemplo:


  List<Integer> list = new ArrayList<>() {{
    add(1);
    add(2);
  }};

Após a aplicação da correção rápida:


  List<Integer> list = new ArrayList<>();
  list.add(1);
  list.add(2);
