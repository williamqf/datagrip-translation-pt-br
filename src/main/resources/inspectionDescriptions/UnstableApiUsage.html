Reporta o uso de APIs marcadas como instáveis por uma das anotações. Tais APIs podem ser alteradas ou removidas em versões futuras, o que pode quebrar o código que as utiliza.

As anotações usadas para marcar APIs instáveis são exibidas na lista abaixo.

Por padrão, esta inspeção ignora o uso de APIs instáveis se a declaração estiver no código-fonte do mesmo projeto. Nesses casos, os usos podem ser atualizados ao alterar a API. No entanto, isso pode ser inconveniente em projetos grandes; portanto, você pode desativar a opção Ignorar APIs declaradas neste projeto para reportar APIs instáveis declaradas tanto no código-fonte do projeto quanto em bibliotecas.
