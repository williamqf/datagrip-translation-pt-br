Relata métodos com nomes semelhantes a acessores que possuem uma cláusula de parâmetro vazia.

Métodos que seguem o contrato de nomenclatura JavaBean para acessores devem não ter efeitos colaterais. A convenção recomendada para esses métodos é usar um método sem parâmetros sempre que não houver parâmetros.

Esta convenção promove o princípio de acesso uniforme, que diz que o código do cliente não deve ser afetado por uma decisão de implementar um atributo como um campo ou método.

A correção rápida remove a cláusula de parâmetro vazia.

Exemplo:


  trait Teste {
    def hasProperty(): Boolean
  }

Após a correção rápida ser aplicada:


  trait Teste {
    def hasProperty: Boolean
  }

Esta inspeção não relatará métodos que sobrescrevem outros membros.
