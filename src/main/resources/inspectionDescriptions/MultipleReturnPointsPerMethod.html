Relata métodos nos quais o número de pontos de return excede o valor máximo especificado. Métodos com muitos pontos de return podem ser confusos e difíceis de refatorar.

Um ponto de return pode ser uma instrução return ou o encerramento implícito ao final de um método void ou construtor.

Exemplo:

Se apenas duas instruções return forem permitidas, o seguinte método será relatado:


  void doSmth(User[] users) {
    for (User user : users) {
      if (cond1(user)) {
        user.setId(getId());
        return;
      } else if (cond2(user)) {
        if (cond3(user)) {
          user.setId(getId());
          return;
        }
      }
    }
  }

Considere reescrever o método para torná-lo mais fácil de entender:


  void doSmth(User[] users) {
    for (User user : users) {
      if (cond1(user) || cond2(user) && cond3(user)) {
        user.setId(getId());
        return;
      }
    }
  }

Configuração da inspeção:

  • Use o campo Limite de pontos de return para especificar o número máximo permitido de pontos de return para um método.
  • Use a opção Ignorar guard clauses para ignorar cláusulas de guarda. Uma guard clause é uma instrução if que contém apenas uma instrução return.
  • Use a opção Ignorar para métodos 'equals()' para ignorar pontos de return dentro de métodos equals().
