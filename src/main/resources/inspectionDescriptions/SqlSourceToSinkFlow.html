Relata casos em que strings não seguras são passadas para métodos como consultas SQL nas linguagens Java e Kotlin. Isso pode ser uma causa de injeção de SQL. A lista de métodos é obtida em Configurações - Injeção de linguagem para SQL, JPA QL, Hibernate QL e PostgreSQL.

Objetos seguros são:

  • Literais de string, instâncias de interface ou objetos enum, int e seus wrappers, boolean e seus wrappers, objetos de classe
  • Resultados de chamadas de método onde o receptor e os argumentos são seguros
  • Campos private no mesmo arquivo atribuídos apenas com literais de string e que possuem inicializadores seguros
  • Campos final no mesmo arquivo que possuem inicializadores seguros
  • Variáveis locais atribuídas a partir de objetos seguros
Este campo, variável local ou parâmetro não deve ser passado como argumento para um método ou usado como um qualificador, ou deve ser um primitivo, seu wrapper ou objeto imutável. Campos static final são considerados seguros.

A análise é realizada apenas dentro de um único arquivo. Exemplo: 


    public void save(String sql) {
        JdbcTemplate jdbcTemplate = new JdbcTemplate();
        jdbcTemplate.queryForList(sql);
    }
