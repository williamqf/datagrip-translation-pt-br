Relata chamadas de métodos bloqueantes de thread em fragmentos de código onde as threads não devem ser bloqueadas.

Exemplo (Project Reactor):


Flux.just("1").flatMap(f -> {
    Flux<String> just = loadUsersFromDatabase();
    just.toIterable(); // Erro: chamada de operador bloqueante em um escopo não bloqueante
    return just;
  }
);

Considere executar o código bloqueante com um scheduler apropriado (por exemplo, Schedulers.boundedElastic()) ou tente encontrar uma API não bloqueante alternativa.


Exemplo (Kotlin Coroutines):


suspend fun exampleFun() {
    Thread.sleep(100); // Erro: chamada de método bloqueante em uma função suspend
}

Considere executar o código bloqueante com um dispatcher especial (por exemplo, Dispatchers.IO) ou tente encontrar uma API não bloqueante alternativa.

Configuração da inspeção:

  • Na lista Anotações bloqueantes, especifique as anotações que marcam métodos bloqueantes de thread.
  • Na lista Anotações não bloqueantes, especifique as anotações que marcam métodos não bloqueantes.

As anotações especificadas podem ser usadas como anotações externas.
