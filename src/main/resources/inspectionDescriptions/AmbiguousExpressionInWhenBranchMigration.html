Relata expressões lógicas ambíguas em ramificações when que causarão erros de compilação no Kotlin 1.8 e superior.


  fun Int.matches(strict: Boolean): Boolean = when (strict) {
      true -> this == 6
      this in (4..7) -> true // ambígua
      else -> false
  }

Após aplicar o ajuste rápido:


  fun Int.matches(strict: Boolean): Boolean = when (strict) {
      true -> this == 6
      (this in (4..7)) -> true // envolvida em parênteses
      else -> false
  }

A inspeção está disponível a partir do nível de linguagem Kotlin 1.7.
