Relata a subtração no método compareTo() e em métodos que implementam java.util.Comparator.compare().

Embora o uso do resultado da subtração de inteiros como o resultado de um método compareTo() seja uma prática comum, essa construção pode causar bugs sutis em caso de estouro (overflow) de inteiros. Na maioria dos casos, é preferível comparar os valores inteiros diretamente e retornar -1, 0 ou 1.

Subtrações de valores de ponto flutuante convertidos diretamente para o tipo inteiro também são relatadas, pois o arredondamento pode levar à perda de precisão.

A inspeção não relata quando é determinado estaticamente que o intervalo de valores é limitado e que o estouro nunca ocorrerá. Além disso, a subtração de números int maiores ou iguais a 0 nunca causará estouro. Portanto, esta inspeção não emite avisos nesses casos.

Métodos que sempre retornam zero ou um valor maior podem ser marcados com a anotação javax.annotation.Nonnegative ou especificados nas opções desta inspeção.

Exemplo:


  class DoubleHolder implements Comparable<DoubleHolder> {
    double d;
    public int compareTo(DoubleHolder that) {
        return (int)(this.d - that.d);
    }
 }

Este é um exemplo sem aviso, pois sabe-se que String.length() não é negativo:


  class A implements Comparable<A> {
    final String s = "";
    public int compareTo(A a) {
        return s.length() - a.s.length();
    }
 }

Use estas opções para listar os métodos cujo uso em subtrações é seguro. Um método é seguro se sempre retornar um valor int maior ou igual a 0.
