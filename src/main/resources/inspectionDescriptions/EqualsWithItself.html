Relata chamadas a equals(), compareTo() ou métodos semelhantes que comparam um objeto com ele mesmo. Os contratos desses métodos especificam que tais chamadas sempre retornarão true para equals() ou 0 para compareTo(). Esta inspeção também verifica chamadas a Objects.equals(), Objects.deepEquals(), Arrays.equals(), Comparator.compare(), métodos assertEquals() de frameworks de teste (JUnit, TestNG, AssertJ), Integer.compare(), Integer.compareUnsigned() e métodos semelhantes.

Observe que, em casos raros, esta inspeção pode relatar uma chamada equals() que retorna false porque, embora as expressões em ambos os lados sejam as mesmas, elas produzem objetos distintos e a comparação é executada por referência, em vez de por conteúdo. O exemplo mais simples é new Object().equals(new Object()). Em qualquer caso, tais chamadas são suspeitas e podem ter tido outras intenções.

Exemplo:


  class Foo {
    boolean foo(Object o) {
        return o.equals(o); // aviso
    }

    boolean bar(String[] ss) {
        return Arrays.equals(ss, ss); // aviso
    }
}
