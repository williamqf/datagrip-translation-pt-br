Relata chamadas redundantes de asSequence() que nunca produzirão um impacto positivo no desempenho.

O asSequence() acelera o processamento de coleções que envolvem múltiplas operações, pois executa as operações de forma adiada (lazy) e não cria coleções intermediárias.

No entanto, se uma operação terminal (por exemplo, toList()) for usada imediatamente após asSequence(), nenhum efeito positivo de desempenho será obtido.

Exemplo:


  fun test(list: List<String>) {
      list.asSequence().last()
  }

Após a aplicação da correção rápida:


  fun test(list: List<String>) {
      list.last()
  }
