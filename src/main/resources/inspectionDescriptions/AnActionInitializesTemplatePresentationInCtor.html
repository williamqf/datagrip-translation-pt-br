Inspeciona quaisquer ações registradas no arquivo plugin.xml e inicializa suas representações (texto, descrição e/ou ícone) em seus construtores.

Esta prática é desencorajada pois pode levar ao carregamento de texto, descrição e/ou ícone e força a criação de representações de ação que podem não ser sempre necessárias. É mais eficiente utilizar construtores sem argumento de AnAction e outras classes base, e seguir a convenção para definir texto, descrição e ícone. A plataforma IntelliJ irá carregar texto, descrição e ícone apenas quando a ação for realmente exibida em algum lugar.

A convenção para definir texto, descrição e ícone é:

  • Defina o atributo id para a ação no arquivo plugin.xml.
  • Se necessário, o atributo icon também pode ser definido.
  • Ao definir as chaves para texto e descrição nos arquivos de propriedades, siga a seguinte estrutura:
    • action.<action-id>.text=Texto traduzido da ação
    • action.<action-id>.description=Descrição traduzida da ação

Exemplo ruim:


  // NewKotlinFileAction.kt
  internal class NewKotlinFileAction : AnAction(
    KotlinBundle.message("action.new.file.text"),
    KotlinBundle.message("action.new.file.description"),
    KotlinIcons.FILE
  )


  // plugin.xml
  <action
    class="org.jetbrains.kotlin.idea.actions.NewKotlinFileAction"
  </action>

Exemplo correto:


  // NewKotlinFileAction.kt
  internal class NewKotlinFileAction : AnAction()


  // plugin.xml
  <action
    id="Kotlin.NewFile"
    class="org.jetbrains.kotlin.idea.actions.NewKotlinFileAction"
    icon="org.jetbrains.kotlin.idea.KotlinIcons.FILE">
  </action>


  // KotlinBundle.properties
  action.Kotlin.NewFile.text=Classe/arquivo Kotlin
  action.Kotlin.NewFile.description=Cria uma nova classe ou arquivo Kotlin
