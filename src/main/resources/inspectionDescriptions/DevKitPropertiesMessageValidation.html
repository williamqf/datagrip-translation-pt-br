Relata os seguintes problemas de MessageFormat em valores de propriedade:

Tipo de formato desconhecido

O MessageFormat suporta apenas os seguintes tipos de formato:

  • number
  • date
  • time
  • choice
Outros tipos de formato serão relatados.

Aspas não correspondentes 


  property.key=Should't happen: {0}

Uma aspa simples será interpretada como um escape até o final da propriedade e não aparecerá na string de resultado. Na maioria dos casos, isso não é intencional. Se uma aspa simples deve aparecer na string de resultado, aspas simples duplicadas devem ser usadas na propriedade.

Chaves não correspondentes

Cada marcador (placeholder) deve ter uma chave de fechamento.

Excesso de aspas

Em alguns casos (por exemplo, it's), espera-se apenas uma aspa na string de resultado. Relata casos em que duas ou mais aspas são colocadas juntas na string de resultado.

Limites inferiores incorretos para ChoiceFormat aninhado

Os limites inferiores devem ser números e devem estar em ordem crescente.

Número incorreto de aspas ao redor dos parâmetros

Em padrões java.text.MessageFormat, aspas simples são usadas para escape. Para que uma aspa seja visível, aspas duplicadas devem ser usadas. Por exemplo, ao passar 1:

  • '{0}' → {0}
  • ''{0}'' → '1'

Ao usar o formato choice, o formato aninhado é avaliado como a própria string de formato e as aspas devem ser duplicadas duas vezes. Por exemplo, ao passar 1:

  • {0, choice, 0#no|1#''{0}'' files} → {0} files
  • {0, choice, 0#no|1#''''{0}'''' files} → '1' files

Notas

Os valores das propriedades são validados apenas se contiverem o texto literal {0}, {0,, {1} ou {1,. Isso serve para garantir que esses valores de propriedade sejam realmente usados como padrões de MessageFormat.
