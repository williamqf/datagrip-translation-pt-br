Relata recursos JDBC que não foram fechados com segurança. Os recursos JDBC relatados por esta inspeção incluem java.sql.Connection, java.sql.Statement, java.sql.PreparedStatement, java.sql.CallableStatement e java.sql.ResultSet.

Por padrão, esta inspeção assume que o recurso pode ser fechado por qualquer método que contenha 'close' ou 'cleanup' em seu nome.

Exemplo:


  ResultSet findAllElements(Connection connection) throws SQLException {
    PreparedStatement statement = connection.prepareStatement("SELECT * FROM TABLE");// instrução não fechada
    statement.execute();
    return statement.getResultSet();
  }

Configure a inspeção usando as seguintes opções:

  • Se deve ser permitido abrir recursos JDBC dentro de blocos try. Este estilo é menos desejável por ser mais verboso do que abrir o recurso antes do bloco try.
  • Se o recurso pode ser fechado por qualquer chamada de método que passe o recurso como um argumento.
