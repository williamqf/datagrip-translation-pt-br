Reporta loops while que realizam spinning sobre o valor de um campo não volatile, aguardando que ele seja alterado por outra thread.

Além de possivelmente consumirem muita CPU quando pouco trabalho é realizado dentro do loop, tais loops provavelmente têm semânticas diferentes das pretendidas. O Modelo de Memória do Java permite que o loop nunca termine, mesmo que outra thread altere o valor do campo.

Exemplo:


  class SpinsOnField {
      boolean ready = false;

      void run() {
          // Mesmo após markAsReady ser chamado de outra thread,
          // este loop pode nunca terminar
          while (!ready) {
          }
          // executa algum trabalho
      }

      void markAsReady() {
          ready = true;
      }
  }

Além disso, a partir do Java 9, recomenda-se chamar Thread.onSpinWait() dentro de loops de spinning em campos volatile, o que pode melhorar significativamente o desempenho em determinados hardwares.
