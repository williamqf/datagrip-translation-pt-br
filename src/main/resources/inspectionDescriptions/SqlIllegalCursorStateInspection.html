Relata estados de cursor inválidos em rotinas SQL.
  • A rotina possui uma instrução CLOSE ou FETCH, mas o cursor pode estar fechado.
  • A rotina possui uma instrução OPEN, mas o cursor pode estar aberto.

Exemplo (Microsoft SQL Server):

CREATE TABLE t(col INT);

CREATE PROCEDURE foo() AS
BEGIN
DECLARE my_cursor CURSOR FOR SELECT * FROM t;
DECLARE a INT;
FETCH my_cursor INTO a;
CLOSE my_cursor;
END;

De acordo com o CLOSE (Transact-SQL) no docs.microsoft.com, o CLOSE deve ser executado para um cursor aberto e não é permitido em cursores que foram apenas declarados ou que já estão fechados. Portanto, precisamos abrir o cursor para corrigir o aviso.

CREATE PROCEDURE foo() AS
BEGIN
DECLARE my_cursor CURSOR FOR SELECT * FROM t;
DECLARE a INT;
OPEN my_cursor;
FETCH my_cursor INTO a;
CLOSE my_cursor;
END;
