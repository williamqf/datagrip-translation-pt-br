Relata chamadas let redundantes baseadas em argumentos.

Quando um parâmetro lambda é usado apenas como um qualificador em uma expressão de chamada, o let é redundante.

Se for necessário nomear a expressão de qualificação, declare uma variável local.

Exemplo:


  fun splitNumbers() {
      "1,2,3".let { it.split(',') }
  }

A correção rápida remove a chamada let() extra:


  fun example() {
      "1,2,3".split(',')
  }

Ou:


  fun splitNumbers() {
      val numbers = "1,2,3"
      numbers.split(',')
  }
