Relata anotações de opt-in desnecessárias que podem ser removidas com segurança.

A anotação @OptIn é necessária para códigos que utilizam APIs experimentais que podem mudar a qualquer momento no futuro. Se tais APIs não forem utilizadas (por exemplo, quando uma API experimental se torna estável e não exige mais o opt-in), esta anotação torna-se irrelevante e pode ser enganosa.

A correção rápida Remover anotação pode ser usada para remover anotações @OptIn desnecessárias.

Exemplo:


  @OptIn(ExperimentalApi::class)
  fun foo(x: Bar) {
      x.baz()
  }

Após aplicar a correção rápida:


  fun foo(x: Bar) {
      x.baz()
  }
