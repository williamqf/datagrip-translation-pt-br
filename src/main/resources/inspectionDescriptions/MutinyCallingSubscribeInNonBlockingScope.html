Relata chamadas subscribe() em métodos "reativos".

Métodos que retornam um Publisher (incluindo Uni e Multi) não devem chamar o método subscribe() diretamente, pois isso pode romper a cadeia de chamadas reativas.

Em vez de usar subscribe(), considere utilizar operadores de composição, como flatMap(), merge(), etc.

Exemplo:


  Multi<String> stringMulti(){
    Multi<String> multi = Multi.createFrom().item("abc");
    multi.subscribe(); // <- chamada 'subscribe' bloqueante em um contexto não bloqueante
    return multi;
  }
