Relata o uso de serviços de aplicativo alocados para campos/propriedades estáticos finais.

Essas alocações levam ao estado global, o que impede remover um aplicativo e configurar outro em testes. Assim, testes repetidos no mesmo processo podem falhar. A única exceção é para chamadas de construtor explícitas usadas para armazenar instâncias virtuais/padrão.

A forma recomendada de evitar armazenamento de serviços envolvem buscar o serviço localmente ou envolvê-lo em um java.util.function.Supplier.

Exemplo:


  // Jeito incorreto
  private static final ManagingFS ourInstance = ApplicationManager.getApplication().getService(ManagingFS.class);


  // Jeito correto
  private static final Supplier<ManagingFS> ourInstance = CachedSingletonsRegistry.lazy(() -> {
    return ApplicationManager.getApplication().getService(ManagingFS.class);
  });


  // Exceção
  private static final UniqueVFilePathBuilder DUMMY_BUILDER = new UniqueVFilePathBuilder()
