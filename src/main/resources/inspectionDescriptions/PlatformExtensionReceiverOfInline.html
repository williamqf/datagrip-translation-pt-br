Relata chamadas potencialmente inseguras para funções inline com receptores de extensão flexíveis anuláveis (tipos de plataforma com nulidade desconhecida).

Antes do Kotlin 1.2, chamadas para inline fun com receptores de extensão flexíveis anuláveis (tipos de plataforma com nulidade desconhecida) não incluíam verificações de nulidade no bytecode. Desde o Kotlin 1.2, as verificações de nulidade são incluídas no bytecode (consulte KT-12899).

Recomenda-se adicionar um !! explícito para lançar uma exceção ou considerar a alteração do tipo do receptor da função para anulável (se ela puder ser executada sem exceções).

Exemplo:


  inline fun String.removePrefix(prefix: String): String {
      return this.substring(prefix.length)
  }

  fun main() {
      // `System.getProperty` retorna o tipo não representável `String!`
      val property = System.getProperty("user.dir")
      println(property.removePrefix("/home"))
  }

Após a aplicação da correção rápida:


  inline fun String.removePrefix(prefix: String): String {
      return this.substring(prefix.length)
  }

  fun main() {
      // `System.getProperty` retorna o tipo não representável `String!`
      val property = System.getProperty("user.dir")
      println(property!!.removePrefix("/home"))
  }

Esta inspeção relata apenas se o nível da linguagem Kotlin do projeto ou módulo for inferior a 1.2.
