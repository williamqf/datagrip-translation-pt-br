Reporta argumentos de chamada do tipo Boolean que não possuem nomes de parâmetros explícitos especificados.

Ao passar vários literais booleanos em sequência, é fácil esquecer a ordem dos parâmetros, o que pode levar a erros. Nomes de parâmetros explícitos tornam o código mais fácil de ler e entender.

Exemplo:


  fun check(checkName: Boolean, checkAddress: Boolean, checkPhone: Boolean) {}

  fun usage() {
      check(true, false, true) // o que isso significa?
  }

A correção rápida adiciona os nomes de parâmetros ausentes:


  fun check(checkName: Boolean, checkAddress: Boolean, checkPhone: Boolean) {}

  fun usage() {
      check(checkName = true, checkAddress = false, checkPhone = true)
  }
