Relata expressões if em cascata em que o último ramo else faz parte de uma qualificação por ponto ou de uma expressão binária.

Isso pode levar a um comportamento inesperado, pois apenas a expressão if-else mais interna torna-se o receptor ou o operando à esquerda da expressão externa, o que pode não coincidir com a lógica pretendida.

Exemplo:


fun printNumberSign(num: Int) {
    if (num > 0) {
        "positive"
    } else if (num < 0) {
        "negative"
    } else {
        "zero"
    }.let { print(it) }
}

fun main() {
    printNumberSign(1) // Nada é impresso, pois o receptor do 'let' é apenas a expressão 'if-else' interna
}

Após aplicar a primeira correção rápida (converter para when):


fun printNumberSign(num: Int) {
    when {
        num > 0 -> {
            "positive"
        }

        num < 0 -> {
            "negative"
        }

        else -> {
            "zero"
        }
    }.let { print(it) }
}

fun main() {
    printNumberSign(1) // Imprime "positive"
}

Alternativamente, a segunda correção rápida adiciona chaves de esclarecimento para manter a semântica original enquanto torna o ramo else aninhado explícito:


fun printNumberSign(num: Int) {
    if (num > 0) {
        "positive"
    } else {
        if (num < 0) {
            "negative"
        } else {
            "zero"
        }.let { print(it) }
    }
}

fun main() {
    printNumberSign(1) // Nada é impresso, mas o código está mais claro
}
