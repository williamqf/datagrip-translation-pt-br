Relata cadeias de chamadas da Stream API que terminam com uma operação count() e podem ser otimizadas.

As seguintes cadeias de chamadas podem ser substituídas por esta inspeção:

  • Collection.stream().count() → Collection.size(). No Java 8, Collection.stream().count() na verdade itera sobre os elementos da coleção para realizar a contagem, enquanto Collection.size() é muito mais rápido para a maioria das coleções.
  • Stream.flatMap(Collection::stream).count() → Stream.mapToLong(Collection::size).sum(). Da mesma forma, não há necessidade de iterar sobre todas as coleções aninhadas. Em vez disso, seus tamanhos podem ser somados.
  • Stream.filter(o -> ...).count() > 0 → Stream.anyMatch(o -> ...). Diferente da chamada original, o anyMatch() pode interromper o processamento assim que um elemento correspondente for encontrado.
  • Stream.filter(o -> ...).count() == 0 → Stream.noneMatch(o -> ...). Semelhante ao caso acima.

Observe que, se a substituição envolver uma operação de curto-circuito como anyMatch(), poderá haver uma mudança visível no comportamento caso as operações intermediárias da stream produzam efeitos colaterais. Geralmente, efeitos colaterais devem ser evitados em chamadas da Stream API.
