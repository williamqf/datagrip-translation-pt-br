Reporta identificadores considerados muito longos. Os limites de tamanho variam de acordo com o banco de dados.

Exemplo (MySQL):

CREATE TABLE ooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooo
(
   ooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooo int
);

No MySQL, identificadores não devem ultrapassar 64 caracteres.

Considere as seguintes limitações de acordo com o banco de dados:

  • Sybase ASE:

    • Nomes entre aspas: 253 caracteres
    • Nomes sem aspas: 255 caracteres

  • Db2 LUW:

    • BUFFER POOL, INDEX EXTENSION, TABLESPACE, TRANSFORM, TYPE MAPPING: 18 caracteres
    • Outros objetos: 128 caracteres
  • Db2 iSeries: 128 caracteres

  • Db2 z/OS:

    • DATABASE e TABLESPACE: 8 caracteres
    • Colunas e cursores com WITH RETURN: 30 caracteres
    • Outros objetos: 128 caracteres
  • Derby: 128 caracteres
  • Exasol: 128 caracteres
  • HSQLDB: 128 caracteres

  • MySQL:

    • Apelidos em consultas: 256 caracteres
    • USER e ROLE: 32 caracteres
    • Outros objetos: 64 caracteres

  • MariaDB:

    • USER: 80 caracteres
    • ROLE: 128 caracteres
    • Outros objetos: igual ao MySQL
  • Microsoft SQL Server: 128 caracteres
  • Vertica: 128 caracteres
  • SQL 92: 128 caracteres
  • Snowflake: 255 caracteres
  • Redshift: 127 caracteres
  • PostgreSQL/Greenplum/Cockroach: 63 caracteres

  • Oracle:

    • Nomes longos estão habilitados (database.oracle.names.long está especificado, e a versão do Oracle é anterior à 12.2 ou desconhecida):

      • DISKGROUP, ROLLBACK SEGMENT, TABLESPACE, PLUGGABLE DATABASE: 30 caracteres
      • DATABASE: 8 caracteres
      • Outros objetos: 128 caracteres
    • Nomes longos estão desabilitados:
      • DATABASE: 8 caracteres
      • DATABASE LINK: 128 caracteres
      • Outros objetos: 30 caracteres
