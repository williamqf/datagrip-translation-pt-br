Detecta arquivos YAML que podem ser interpretados como parte de uma especificação OpenAPI/Swagger.

Tais arquivos não contêm atributos de especificação explícitos, mas são referenciados a partir de arquivos de especificação primários próximos.

O exemplo a seguir contém dois arquivos localizados no mesmo diretório. O primeiro é um arquivo de especificação primário simples. O segundo é referenciado pelo primeiro e, portanto, sugere-se que também seja considerado uma especificação.

Arquivo de especificação primário openapi.yaml:


openapi: 3.1.0
components:
  schemas:
    CustomSchema:
      description: Custom schema object
      properties:
        foo:
          $ref: 'common.components.yaml#/components/schemas/CommonSchema'

Candidato a arquivo de especificação common.components.yaml:


components: # Realce 'Mark file as OpenAPI specification'
  schemas:
    CommonSchema:
      description: Common schema object reused in several specifications
