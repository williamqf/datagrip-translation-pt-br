Inspeciona chamadas de log SLF4J ou Log4j 2 como logger.info("{}: {}", key), onde o número de marcadores de posição {} na mensagem do logger não corresponde ao número de outros argumentos na chamada de log.

Utilize a opção de inspeção para especificar qual implementação o SLF4J deve usar. Se a opção Verificar automaticamente for selecionada, o org.apache.logging.slf4j.Log4jLogger será procurado no caminho da classe. Se esse arquivo for encontrado ou se Sim for selecionado, os casos em que o último argumento com um tipo de exceção tiver um marcador de posição não serão reportados para a API SLFJ4. Exemplo: 


// Não será reportado
log.error("For id {}: {}", id, new RuntimeException());
Essa opção só se aplica ao SLF4J.
