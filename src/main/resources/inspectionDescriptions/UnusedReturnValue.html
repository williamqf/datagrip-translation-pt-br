Relata métodos cujos valores de retorno nunca são utilizados quando chamados. O tipo de retorno de tais métodos pode ser definido como void.

Métodos anotados com a anotação @CanIgnoreReturnValue do Error Prone ou AssertJ não serão relatados. O ajuste rápido atualiza a assinatura do método e remove as instruções return do método.

Exemplo:


  // Relatado quando a visibilidade está definida como Protected ou Public
  protected String myToUpperCase(String s) {
    return s.toUpperCase();
  }

  // Setter simples, relatado dependendo das configurações
  public String setStr(String str) {
    myStr = str;
    return myStr;
  }

  void test() {
    setStr("value");          // valor de retorno não utilizado
    myToUpperCase("result");  // valor de retorno não utilizado
  }

Após aplicar o ajuste rápido a ambos os métodos:


    protected void myToUpperCase(String s) {
      // 'return' removido completamente
      // pois 's.toUpperCase()' não tem efeitos colaterais
    }

    public void setStr(String str) {
      myStr = str;
      // 'return' removido
    }
  ...

Nota: Por motivos de desempenho, alguns métodos podem não ser relatados durante o realce no editor. Para ver todos os resultados, execute a inspeção por meio de Código | Inspecionar código ou Código | Analisar código | Executar inspeção por nome.

Use a opção Ignorar métodos encadeáveis para ignorar valores de retorno não utilizados de chamadas encadeáveis.

Use a opção Visibilidade máxima do método relatado para controlar a visibilidade máxima dos métodos a serem relatados.
