Relata chamadas de log do SLF4J, Log4j2 e akka.event.LoggingAdapter, como logger.info("{}: {}", key), nas quais a quantidade de espaços reservados {} na mensagem do logger não corresponde à quantidade de outros argumentos da chamada de log.

Use a opção de inspeção para especificar qual implementação o SLF4J utiliza. Se Verificar automaticamente estiver selecionado, o org.apache.logging.slf4j.Log4jLogger será pesquisado no classpath. Se este arquivo for encontrado ou se Sim for selecionado, situações em que o último parâmetro com um tipo de exceção possui um espaço reservado não serão relatadas para a API SLF4J.
Exemplo: 


// Esse caso não será relatado se a opção "Sim" for selecionada
log.error("For id {}: {}", "1", new RuntimeException());
Nesse caso, 'new RuntimeException()' será impresso usando 'toString()' (seu stack trace não será impresso): 

For id 1: java.lang.RuntimeException
Caso contrário, ele será destacado porque o último espaço reservado não é utilizado: 

For id 1: {}
java.lang.RuntimeException: null
A opção Não pode ser usada para sempre destacar esses casos quando um espaço reservado é usado para uma exceção, mesmo que o org.apache.logging.slf4j.Log4jLogger seja usado como backend.
Esta opção aplica-se apenas ao SLF4J.
