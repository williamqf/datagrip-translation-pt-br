Reporta o uso desnecessário dos esquemas grpc ou http[s] padrão em requisições gRPC.

Exemplos de requisição: 


# O esquema `grpc` pode ser omitido, pois uma conexão insegura é assumida por padrão
GRPC grpc://localhost/TestService/testRpc


# O esquema `http` pode ser omitido: prefira um esquema vazio para conexões inseguras e o esquema dedicado `grpcs` para conexões seguras
GRPC http://localhost/TestService/testRpc


# O esquema `https` deve ser substituído por `grpcs`: o esquema dedicado `grpcs` é preferível para indicar que o Transport Layer Security deve estar habilitado para executar a requisição
GRPC https://localhost/TestService/testRpc

Para evitar confusão, recomenda-se o uso do esquema dedicado grpcs em requisições gRPC quando a requisição deve usar um canal seguro subjacente. Caso contrário, o esquema pode ser omitido completamente.
