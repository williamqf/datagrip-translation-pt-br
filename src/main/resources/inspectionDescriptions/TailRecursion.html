Relata a recursão de cauda, ou seja, quando um método chama a si mesmo como sua última operação antes de retornar.

A recursão de cauda sempre pode ser substituída por um loop, que é significativamente mais rápido. Algumas JVMs realizam a otimização de chamada de cauda (tail-call optimization), enquanto outras não. Portanto, as soluções de recursão de cauda podem ter características de desempenho consideravelmente diferentes em diferentes máquinas virtuais.

Exemplo:


  int factorial(int val, int runningVal) {
    if (val == 1) {
      return runningVal;
    } else {
      return factorial(val - 1, runningVal * val);
    }
  }

Após a aplicação do ajuste rápido:


  int factorial(int val, int runningVal) {
    while (true) {
      if (val == 1) {
        return runningVal;
      } else {
        runningVal = runningVal * val;
        val = val - 1;
      }
    }
  }
