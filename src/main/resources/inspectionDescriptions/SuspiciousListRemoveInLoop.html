Reporta chamadas list.remove(index) em loops de contagem crescente.

Isso é suspeito porque, após a remoção, a lista encurta e o próximo elemento é ignorado. Uma correção simples é decrementar a variável de índice após a remoção, mas uma abordagem mais robusta seria remover por meio de um iterador ou usar o método removeIf() (Java 8 e superior). Se você não espera que o remove() seja chamado mais de uma vez no loop, considere adicionar um break após a chamada.

Exemplo:

  public static void main(String[] args) {
    process(new ArrayList<>(
      Arrays.asList("1", "2", "|", "3", "4")));
  }

  static void process(List<String> list) {
    for (int i = 0; i < list.size(); i++) {
      if (list.get(i).equals("|")) {
        list.remove(i);
        continue;
      }
      System.out.println(list.get(i));
    }
  }

O código parece que imprimirá 1 2 3 4, mas, na verdade, o 3 será ignorado na saída.
