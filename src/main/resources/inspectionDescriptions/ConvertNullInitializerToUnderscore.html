Relatórios de definições de variáveis que são explicitamente inicializadas com null.

Há 3 casos diferentes de var v: T = null:

  1. v é um valor opcional
    → usa o tipo Option[T], que é idiomático e seguro para tipos
  2. v é um valor não inicializado (um valor deve ser atribuído)
    → use _ (Scala 2) ou scala.compiletime.uninitialized (Scala 3) como inicializador, que é claro, conciso e independente de plataforma
  3. É uma otimização de desempenho (você realmente usa o valor null)
    → reprimiu explicitamente a inspeção para o código não seguro

Duas correções rápidas serão oferecidas:

  • Usar inicializador _ substituirá null com _
  • Usar tipo Option substituirá o tipo da variável com Option e irá inicializá-la com None

Exemplo:


  class Test {
    var optional: String = null
    var uninit: String = null

    uninit = "initialized later"
  }

Após as correções rápidas serem aplicadas:


  class Test {
    var optional: Option[String] = None
    var uninit: String = _

    uninit = "initialized later"
  }
