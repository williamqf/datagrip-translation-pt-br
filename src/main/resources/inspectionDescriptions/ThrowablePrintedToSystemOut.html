Relata chamadas System.out.println() que utilizam uma exceção como argumento.

O uso de instruções de impressão para registrar exceções oculta o stack trace, o que pode complicar a investigação de problemas. Recomenda-se o uso de um logger em vez disso.

Chamadas a System.out.print(), System.err.println() e System.err.print() com uma exceção como argumento também são relatadas. É aconselhável usar um logger para registrar exceções.

A correção rápida fornecida suporta SLF4J e Log4j 2. Ela substitui a chamada System.out.println() por uma chamada de log.

Exemplo:


  try {
      foo();
  } catch (Exception e) {
      System.out.println(e);
  }

Após aplicar a correção rápida:


  try {
      foo();
  } catch (Exception e) {
    log.error("e: ", e);
  }

Use a opção Método de log para correção para especificar o método utilizado para registrar as mensagens.
