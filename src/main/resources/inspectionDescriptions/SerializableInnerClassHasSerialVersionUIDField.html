Relata classes internas não estáticas que implementam java.io.Serializable, mas não definem um campo serialVersionUID.

Sem um campo serialVersionUID, qualquer alteração feita na classe fará com que as versões serializadas anteriormente tornem-se ilegíveis. É fortemente recomendado que classes internas Serializable não estáticas contenham um campo serialVersionUID; caso contrário, o algoritmo de serialização padrão pode resultar em incompatibilidades de versão de serialização entre compiladores devido a diferenças nos métodos acessores sintéticos.

Uma correção rápida é sugerida para adicionar o campo serialVersionUID ausente.

Exemplo:


  class Outer {
      class Inner implements Serializable {}
  }

Após aplicar a correção rápida:


  class Outer {
      class Inner implements Serializable {
          private static final long serialVersionUID = -7004458730436243902L;
      }
  }

Configure a inspeção usando as seguintes opções:

  • Listar classes cujos sucessores não devem ser relatados por esta inspeção. Isso se aplica a classes que herdam Serializable de uma superclasse, mas não são destinadas à serialização.
  • Se as classes anônimas Serializable devem ser ignoradas.
