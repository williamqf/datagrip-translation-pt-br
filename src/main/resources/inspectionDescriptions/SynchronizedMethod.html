Relata o modificador synchronized em métodos.

Existem várias razões pelas quais usar o modificador synchronized em um método pode não ser uma boa ideia:

  1. O mínimo de trabalho possível deve ser realizado sob um bloqueio. Portanto, geralmente é melhor usar um bloco synchronized, mantendo apenas o código que manipula o estado compartilhado.
  2. A sincronização torna-se parte da interface do método. Isso dificulta a transição para mecanismos de bloqueio diferentes.
  3. Torna-se mais difícil rastrear o que está bloqueando um determinado objeto.
  4. Ataques de DoS (Negação de Serviço), intencionais ou acidentais, tornam-se viáveis ao herdar da classe que contém o método.

Como alternativa, considere a sincronização em um objeto de bloqueio private final, que permite o controle total sobre o acesso a esse objeto.

Uma correção rápida está disponível para envolver o corpo do método com synchronized(this).

Exemplo:


  class Main {
    public synchronized void fooBar() {
    }
  }

Após a aplicação da correção rápida:


  class Main {
    public void fooBar() {
      synchronized (this) {
      }
    }
  }

Você pode configurar as seguintes opções para esta inspeção:

  1. Incluir métodos nativos - Inclui métodos nativos no escopo da inspeção.
  2. Ignorar métodos que sobrescrevem métodos sincronizados - Não relata métodos que sobrescrevem um método synchronized.
