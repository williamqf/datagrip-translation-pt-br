Relata instâncias de AutoCloseable que não são usadas em uma instrução try-with-resources, também conhecida como Gerenciamento Automático de Recursos.

Isso significa que o estilo "abrir recurso antes/dentro de try, fechar em finally", usado antes do try-with-resources estar disponível, também é relatado. Esta inspeção visa substituir todas as inspeções de aberto, mas não fechado com segurança ao desenvolver no Java 7 e versões superiores.

Exemplo:


  private static void foo() throws IOException {
    InputStream profile = Thread.currentThread().getContextClassLoader().getResourceAsStream("/someFile");
    System.out.println(profile.read());
  }

Configure a inspeção usando as seguintes opções:

  • Liste as subclasses de AutoCloseable que não precisam ser fechadas e devem ser ignoradas por esta inspeção.
    Observação: mesmo que java.util.stream.Stream seja listado para ser ignorado, a inspeção ainda relatará streams retornados dos métodos de java.nio.file.Files: lines(), walk(), list() e find(). Esses streams contêm recursos de I/O associados que precisam ser fechados.
  • Liste os métodos que retornam AutoCloseable que devem ser ignorados quando chamados.
  • Se deve ignorar o AutoCloseable quando ele for o resultado de uma chamada de método. Quando ativado, os resultados de métodos factory também serão ignorados.
  • Se a inspeção deve relatar quando uma instância de AutoCloseable é passada como argumento de chamada de método. Se ativado, a inspeção assumirá que o recurso é fechado no método chamado. Chamadas de método dentro de um bloco finally e argumentos AutoCloseable com 'close' no nome não serão ignorados.
  • Se deve ignorar referências de método para construtores de classe de recurso.
  • Se deve ignorar métodos que retornam um recurso e cujo nome começa com 'get'. Isso pode reduzir falsos positivos, pois a maioria dos getters não transfere a propriedade do recurso e seus locais de chamada não são responsáveis por fechá-lo.
