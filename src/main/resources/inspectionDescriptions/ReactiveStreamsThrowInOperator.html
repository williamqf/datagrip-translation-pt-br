Relata expressões throw em códigos de operadores Reactor/RxJava.

Lançar exceções a partir de um operador Reactor/RxJava indica um possível problema, pois você pode retornar um erro "reativo": Mono.error() ou Flowable.error() de um flatMap(), ou chamar sink.error() a partir de um operador handle() do Reactor.

Além disso, os métodos de fábrica do Reactor permitem retornar exceções verificadas (checked exceptions) sem erros, enquanto lançar tais exceções sem a classe Exceptions resultará em erros de compilação.

Exemplo:


Flux.just(1, 2, 3).flatMap(i -> {
            throw new RuntimeException();
        })

Após a aplicação da correção rápida:


Flux.just(1, 2, 3).flatMap(i -> {
            return Flux.error(new RuntimeException());
        })
