Relata o uso irregular de iota dentro de declarações de constantes.

O identificador iota é reiniciado para cada declaração de constante e incrementado automaticamente para cada especificação de constante. Dentro de uma única especificação de constante, o identificador mantém seu valor. Referenciar explicitamente iota não reinicia o contador.

Para mais detalhes, consulte Iota na especificação do Go.

Esta inspeção é acionada se duas especificações de constante tiverem listas de expressões textualmente idênticas contendo ao menos uma referência a iota, e houver apenas especificações de constante sem listas de expressões entre elas.

Considere omitir a lista de expressões redundante ou escrever a lista de expressões todas as vezes.

Exemplo:

const (
  a = iota  // 0
  b         // 1
  c = iota  // 2
)

A inspeção é acionada porque o iota na definição de c é redundante.

Exemplo:

const (
  a, aa = iota, iota       // 0, 0
  b, bb                    // 1, 1
  c, cc = iota + 40, iota  // 42, 2
)

A inspeção não é acionada porque nenhuma lista de expressões é redundante.
