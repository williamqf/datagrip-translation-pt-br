Relata implementações de Iterator.hasNext() ou ListIterator.hasPrevious() que chamam Iterator.next() ou ListIterator.previous() em uma instância de iterador. Tais chamadas são quase certamente um erro, pois métodos como hasNext() não devem modificar o estado do iterador, enquanto next() deve fazê-lo.

Exemplo:


  class MyIterator implements Iterator<Integer> {
    public boolean hasNext() {
      return next() != null;
    }
  }
