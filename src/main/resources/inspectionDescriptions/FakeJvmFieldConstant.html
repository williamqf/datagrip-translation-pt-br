Relata propriedades Kotlin que não são const e são usadas como argumentos de anotação Java.

Por exemplo, uma propriedade com a anotação @JvmField que possui um inicializador que pode ser avaliado em tempo de compilação e tem um tipo primitivo ou tipo String.

Tais propriedades possuem o atributo ConstantValue no bytecode no Kotlin 1.1-1.2. Este atributo permite que o javac faça o fold dos usos do campo correspondente e use esse campo em anotações. Isso pode levar a um comportamento incorreto em casos de compilação separada ou incremental em código misto Java/Kotlin. Este comportamento está sujeito a mudanças no Kotlin 1.3 (o atributo ConstantValue não é mais utilizado).

Exemplo:

Código Kotlin no arquivo foo.kt:


  annotation class Ann(val s: String)
  @JvmField val importantString = "important"

Código Java:


  public class JavaUser {
      // Isso é perigoso
      @Ann(s = FooKt.importantString)
      public void foo() {}
  }

Para corrigir este problema, substitua a anotação @JvmField pelo modificador const na propriedade Kotlin relevante ou faça o inline dela.
