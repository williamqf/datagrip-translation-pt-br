Relata os seguintes erros em consultas Persistence QL:
  • Tipos de expressão incompatíveis
  • Parâmetros incorretos
  • Condições vazias ou constantes
  • Símbolos não resolvidos

Exemplos de consulta:


SELECT OBJECT(e) FROM JavaEntity e WHERE TRUE // Aviso: Condição constante

SELECT OBJECT(e) FROM JavaEntity e WHERE e.id BETWEEN 0 AND '2' // Erro: esperado valor numérico, mas encontrada string

SELECT e from JavaEntity e where e.id=?1 and e.name=?3 // Erro: existem apenas 2 parâmetros de consulta reais
