Relata instruções synchronized aninhadas. Recomenda-se evitar o aninhamento de sincronização sempre que possível, pois em certos casos, a sincronização aninhada pode levar a um deadlock.

Exemplo:


  synchronized (lockA){
    // thread 1 aguardando lockB
    synchronized (lockB){ // aviso
    }
  }
  ...
  synchronized (lockB) {
    // thread 2 aguardando lockA
    synchronized (lockA) { // aviso
    }
  }
