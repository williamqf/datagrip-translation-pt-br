Relata declarações propagadas pela anotação @Deprecated que causarão erros de compilação a partir da versão 1.9.

Tipos de motivação:

  • Mudanças na implementação são necessárias por motivos de design/arquitetura de implementação
  • Inconsistência no design (tratado de forma diferente em diferentes contextos)

Mais detalhes: KT-47902: Não propagar a depreciação de método por meio de sobrescritas (overrides)

A correção rápida copia a anotação @Deprecated da declaração pai.

Exemplo:


  open class Base {
      @Deprecated("Don't use")
      open fun foo() {}
  }

  class Derived : Base() {
      override fun foo() {}
  }

Após aplicar a correção rápida:


  open class Base {
      @Deprecated("Don't use")
      open fun foo() {}
  }

  class Derived : Base() {
      @Deprecated("Don't use")
      override fun foo() {}
  }

Esta inspeção relata apenas se o nível de linguagem Kotlin do projeto ou módulo for 1.6 ou superior.
