Relata a delegação quando o objeto delegado é uma instância de uma classe que sobrescreve (no caso de classes final e sealed) ou pode sobrescrever (no caso de classes open e abstract) métodos default do Java.

Por design, a delegação do Kotlin não sobrescreve os métodos default do Java, o que pode levar a bugs difíceis de encontrar, especialmente em hierarquias de herança profundas.

Exemplo:


// Base.java:
public interface Base {
  default void print() {
    System.out.println("Base");
  }
}


// BaseImpl.kt:
class BaseImpl : Base {
  override fun print() {
    println("BaseImpl")
  }
}


// Main.kt:
fun main() {
  val inherited = BaseImpl()
  val delegated = object : Base by inherited {}
  inherited.print() // Saída: BaseImpl
  delegated.print() // Saída: Base
}

Os desenvolvedores geralmente esperam que todos os métodos (incluindo métodos default sobrescritos no delegado) sejam encaminhados. Em vez disso, o método default do Java é chamado, o que pode levar a resultados inesperados.

Um quick-fix é sobrescrever esses métodos delegando a implementação ao objeto delegado: 


// Main.kt:
fun main() {
  val inherited = BaseImpl()
  val delegated = object : Base by inherited {
    override fun print() {
      inherited.print()
    }
  }
  inherited.print() // Saída: BaseImpl
  delegated.print() // Saída: BaseImpl
}

Outro quick-fix é sobrescrever esses métodos delegando explicitamente a implementação para a superclasse: 


// Main.kt:
fun main() {
  val inherited = BaseImpl()
  val delegated = object : Base by inherited {
    override fun print() {
      super.print()
    }
  }
  inherited.print() // Saída: BaseImpl
  delegated.print() // Saída: Base
}
