Relata tags de compilação malformadas e tags de compilação colocadas em locais incorretos. A ferramenta go exige que as tags de compilação sejam colocadas em locais específicos e sigam uma sintaxe especial. Se esses requisitos não forem atendidos, as tags de compilação podem ser ignoradas ou os arquivos podem ser excluídos incorretamente da compilação.

Consulte Build Constraints no go.dev.

Exemplo:

package main

// +build ignore

func main() {}

A parte // +build ignore deve estar localizada antes da declaração do pacote. Para corrigir isso, você pode aplicar a correção rápida Colocar tag de compilação antes do pacote. Após aplicar a correção rápida:

// +build ignore

package main

import "fmt"
