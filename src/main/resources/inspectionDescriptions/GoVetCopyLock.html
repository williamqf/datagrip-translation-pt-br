Relata locks passados incorretamente por valor.

Copiar acidentalmente um valor que contém um lock pode fazer com que ambas as cópias não funcionem corretamente. Normalmente, esses valores devem ser referenciados por meio de um ponteiro. Aqui, um lock representa um tipo que implementa sync.Locker, como sync.Mutex ou sync.WaitGroup.

Exemplo:

type SafeInt struct {
  m sync.Mutex
  i int
}

func (s SafeInt) Inc() { // mutex é copiado
  s.m.Lock()
  s.i++
  s.m.Unlock()
}

Após aplicar o ajuste rápido Adicionar ponteiro:

type SafeInt struct {
  m sync.Mutex
  i int
}

func (s *SafeInt) Inc() {
  s.m.Lock()
  s.i++
  s.m.Unlock()
}
