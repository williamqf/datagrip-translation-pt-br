Relata os seguintes problemas no arquivo de configuração application.xml:
  • Referências a classes não instanciáveis
  • Referências a classes que não estendem as classes necessárias
  • Referências a classes com escopo inadequado
  • Valores de tags e atributos que não correspondem aos padrões exigidos (por exemplo, identificadores Java)
  • Tags que não contêm as sub-tags ou atributos necessários
  • Tags que definem objetos com nomes duplicados

Exemplo:



<application>
    <module id="ExampleWebModule">
      <web> // Erro: a sub-tag 'web-uri' deve ser definida
        <context-root>ExampleWebApp</context-root>
      </web>
    </module>
</application>
