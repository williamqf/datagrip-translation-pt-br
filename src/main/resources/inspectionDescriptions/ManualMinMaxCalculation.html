Relata casos em que chamadas de Math.max() ou Math.min() podem ser usadas em vez de calcular manualmente o valor mínimo ou máximo de dois números.

Exemplo:


  public int min(int a, int b) {
    return b < a ? b : a;
  }

Após a aplicação da correção rápida:


  public int min(int a, int b) {
    return Math.min(a, b);
  }

Use a opção Desativar para float e double para desativar esta inspeção para os tipos double e float. Isso é útil porque a correção rápida pode alterar ligeiramente a semântica dos tipos float/double ao lidar com NaN. No entanto, na maioria dos casos, esta correção rápida na verdade corrige bugs sutis que ocorrem quando o NaN não é considerado.
