Relata chamadas a métodos substituíveis da classe atual durante a construção do objeto.

Um método é chamado durante a construção do objeto se estiver dentro de:

  • Construtores
  • Inicializadores de instância não static
  • Inicializadores de campo não static
  • Método clone()
  • Método readObject()
  • Método readObjectNoData()

Um método é substituível se não for declarado como final, static ou private. Métodos com visibilidade de pacote (package-local) são considerados seguros, mesmo se forem substituíveis. Tais chamadas podem resultar em bugs difíceis de encontrar, pois a inicialização da subclasse pode ainda não ter ocorrido no momento da chamada do método.

Exemplo:


  class Parent {
    void someMethod() { }
  }

  class Child extends Parent {
    Child() {
      someMethod();
    }
  }

Esta inspeção compartilha funcionalidades com as seguintes inspeções:

  • Método abstrato chamado durante a construção do objeto
  • Método substituído chamado durante a construção do objeto

Apenas uma inspeção deve ser ativada por vez para evitar avisos duplicados.
