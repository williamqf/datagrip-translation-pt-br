Relata chamadas a java.sql.Statement.execute() ou qualquer uma de suas variantes que recebam strings construídas dinamicamente como a consulta a ser executada.

Instruções SQL construídas são uma fonte comum de vulnerabilidades de segurança. Por padrão, esta inspeção ignora constantes em tempo de compilação.

Exemplo:


  ResultSet execute(Statement statement, String name) throws SQLException {
    return statement.executeQuery("select * from " + name); // relata um aviso
  }

Use as opções de inspeção para tratar qualquer campo static final como uma constante. Tome cuidado, pois quando essa opção está ativada, strings como a seguinte serão ignoradas: 


  private static final String SQL = "SELECT * FROM user WHERE name='" + getUserInput() + "'";
