Relata chamadas JUnit assertEquals() que podem ser substituídas por chamadas assertSame() equivalentes. Isso pode ocorrer quando os argumentos são instâncias de classes final que não sobrescrevem o método Object.equals(), o que torna explícito que a identidade do objeto está sendo comparada.

Sugere substituir assertEquals() por assertSame().

Exemplo:


  @Test
  public void testObjectType() {
    Object o = getObject();
    Assert.assertEquals(String.class, o.getClass());
  }

Após a aplicação da correção rápida:


  @Test
  public void testSort() {
    Object o = getObject();
    Assert.assertSame(String.class, o.getClass());
  }
