Relata o uso de operadores de igualdade que podem causar coerção de tipo inesperada. Sugere a substituição dos operadores de igualdade == ou != pelos operadores de tipo seguro === ou !==.

Dependendo da opção selecionada, um dos seguintes casos será relatado:

  • Todos os usos dos operadores == e !=.
  • Todos os usos, exceto comparações com null. Alguns estilos de código permitem o uso de x == null em vez de x === null || x === undefined.
  • Apenas expressões suspeitas, como comparações de == ou != com 0, '', null, true, false ou undefined.
