Relata o acesso a campos static não constantes que estão bloqueados em this ou em um campo de instância de this.

Bloquear um campo static em dados de instância não impede que outras instâncias modifiquem esse campo, o que pode levar a condições de corrida inesperadas.

Exemplo:


  static String test;
  public void foo() {
      synchronized (this) {
          System.out.println(test); // aviso
      }
  }

Existe uma correção rápida que permite ignorar campos static de tipos específicos. Os tipos ignorados podem ser gerenciados nas opções de inspeção.

Use as opções de inspeção para especificar quais classes usadas para campos static devem ser ignoradas.
