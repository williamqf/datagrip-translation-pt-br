Relata chamadas da função errors.As quando o segundo argumento não é um ponteiro para uma interface ou para um tipo que implementa um erro.

Essas chamadas causarão pânico no tempo de execução.

Para obter mais informações sobre a função As, consulte func As at go.dev.

Exemplo:

_, err := os.Open("não existente")
var pathError *fs.PathError
se errors.As(err, pathError) { // é necessário um ponteiro para *fs.PathError
}

Após a aplicação da correção rápida Adicionar '&':

_, err := os.Open("não existente")
var pathError *fs.PathError
se errors.As(err, &pathError) {
}

Esta inspeção somente será relatada se a versão da linguagem for 1.13 ou superior.
