Relata instruções if, while, do ou for sem chaves. Alguns estilos de código, como o Google Java Style Guide, exigem chaves para todas as instruções de controle.

Ao adicionar mais instruções a uma instrução de controle sem chaves, é importante não esquecer de adicionar as chaves. Ao comentar uma linha de código, também é necessário ter mais cuidado ao não usar chaves, para evitar que a instrução seguinte se torne parte da instrução de fluxo de controle inadvertidamente. Sempre usar chaves torna mais seguro inserir ou comentar uma linha de código.

Se um estilo de código que sempre utiliza chaves fosse seguido, é muito provável que a vulnerabilidade goto fail não teria ocorrido. Instruções de controle com chaves tornam o fluxo de controle mais fácil de visualizar, sem depender de uma indentação que pode estar incorreta.

Exemplo:


  class Strange {
    void x(boolean one, boolean two) {
      if(one)
        if(two)
          foo();
      else
        bar();
    }

    void foo() {}
    void bar() {}
  }

A correção rápida envolve o corpo da instrução com chaves:


  class Strange {
    void x(boolean one, boolean two) {
      if(one) {
        if(two) {
          foo();
        } else {
          bar();
        }
      }
    }

    void foo() {}
    void bar() {}
  }
