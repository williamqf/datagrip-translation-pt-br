Relatórios quando uma palavra-chave return é usada dentro de uma função anônima para saltar para fora dela mais cedo. Retornos não locais são implementados lançando e capturando scala.runtime.NonLocalReturnException-s. Isto raramente é o que é pretendido pelo programador. Pode ser problemático devido ao custo de desempenho oculto de lançar e capturar exceções.

Além disso, é uma implementação com vazamento: um manipulador de exceção de captura geral pode interceptar um NonLocalReturnException.


def foo(numbers: Seq[Int]): Int =
  numbers.map { n =>
    if (n > 5)
      return 0 // highlighted
    n * 2
  }.sum
