Relata problemas causados pelo uso incorreto de valores nil.

O IDE analisará o fluxo de dados para determinar se uma variável pode ter um valor nil ou não-nil. O IDE relatará problemas potenciais no código com base nesse resultado. Considere as seguintes situações que podem levar a consequências inesperadas:

  • Chamadas de método com um receptor nil podem levar a uma 'desreferenciação de ponteiro nil'.
  • A indexação de um slice nil pode causar um panic.
  • Comparações como v == nil podem não ter sentido se já se sabe que v é sempre nil ou não-nil.
  • Variáveis que possuem um erro correspondente não são verificadas para garantir que não são nil. Um erro corresponde a uma variável quando a variável e o erro são definidos ou atribuídos juntos em uma instrução como v, err := foo().
