Relata declarações de membros de teste JUnit que estão com formato incorreto e que provavelmente não serão reconhecidas pelo framework de teste JUnit. Esta inspeção relata os seguintes problemas:
  • Classes de teste que não podem ser instanciadas
  • Campos anotados com @RegisterExtension com tipo incorreto ou que não foram declarados como static quando necessário
  • Classes internas static ou private anotadas com @Nested
  • Testes parametrizados definidos sem uma fonte
  • Testes parametrizados contendo um @MethodSource com um alvo desconhecido, não static ou sem argumentos
  • Incompatibilidade de tipo entre os parâmetros do método de teste parametrizado e os valores especificados em @ValueSource ou @EnumSource
  • Testes anotados com mais de uma das anotações: @Test, @ParameterizedTest ou @RepeatedTest
  • Métodos setup() ou tearDown() não public, cujo tipo de retorno não seja void ou que aceitem argumentos
  • Método suite() que seja private, aceite argumentos ou não seja static
  • Métodos anotados com @BeforeClass, @AfterClass, @BeforeAll ou @AfterAll que não sejam public, não sejam static, cujo tipo de retorno não seja void ou que não possuam uma lista de parâmetros válida
  • Métodos anotados com @Before, @After, @BeforeEach ou @AfterEach que não sejam public, cujo tipo de retorno não seja void ou que aceitem argumentos
  • RepetitionInfo injetado em métodos @BeforeAll ou @AfterAll
  • RepetitionInfo injetado em métodos @BeforeEach ou @AfterEach usados por testes anotados com @Test
  • Campos e métodos não public ou não static anotados com @DataPoint ou @DataPoints
  • Campos e métodos anotados com @Rule que não sejam public ou não sejam subtipos de TestRule ou MethodRule
  • Campos e métodos anotados com @ClassRule que não sejam public, não sejam static ou não sejam subtipos de TestRule
  • Métodos dentro de uma subclasse de TestCase com o prefixo test que não sejam public, cujo tipo de retorno não seja void, que aceitem argumentos ou sejam static
  • Métodos anotados com @Test que não sejam public, cujo tipo de retorno não seja void, aceitem argumentos ou sejam static
Observe que, em Kotlin, as funções de suspensão possuem argumentos e tipos de retorno não void. Portanto, elas também não serão executadas pelo executor de testes JUnit. Esta inspeção também relatará este problema.

Exemplo de método @Before com formato incorreto:


  @Before private int foo(int arg) { }

Após a aplicação da correção rápida:


  @Before public void foo() { }

Exemplo de fonte de método ausente (Kotlin):


  class Example {
    @MethodSource("parameters")
    @ParameterizedTest
    fun foo(param: String) { ... }
  }

Após a aplicação da correção rápida:


  class Example {
    @MethodSource("parameters")
    @ParameterizedTest
    fun foo(param: String) { ... }

    companion object {
      @JvmStatic
      fun parameters(): Stream<Arguments> {
        TODO("Not yet implemented")
      }
    }
  }

Utilize as opções da inspeção para especificar as anotações. Parâmetros anotados com qualquer uma das anotações especificadas não serão relatados.
