Relata referências a variáveis de loop dentro de literais func em instruções defer e go. Tais variáveis podem ter valores inesperados porque não são copiadas para o literal func, e os literais func em defer e go não são executados imediatamente.

Para mais detalhes sobre closures e goroutines, consulte What happens with closures running as goroutines? em go.dev.

Exemplo:

for _, v := range []string{"a", "b", "c"} {
  go func() {
    fmt.Println(v) // a saída provavelmente será `c c c`, em vez de `a b c`.
  }()
}

Após a aplicação da correção rápida:

for _, v := range []string{"a", "b", "c"} {
  v := v // `v` agora é copiado
  go func() {
    fmt.Println(v)
  }()
}

Observe que o analisador verifica as instruções defer e go apenas se forem as últimas instruções no corpo do loop. Caso contrário, a análise pode produzir detecções incorretas.
