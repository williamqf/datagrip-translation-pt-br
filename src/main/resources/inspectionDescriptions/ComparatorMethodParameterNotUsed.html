Relata problemas nas implementações de Comparator.compare() e Comparable.compareTo().

Os seguintes casos serão relatados:

  • Parâmetros não utilizados. É provável que isto seja um erro de digitação, onde um parâmetro é comparado consigo mesmo, ou o método não foi implementado corretamente.
  • O método obviamente não retorna 0 para elementos idênticos. Tal método de comparação viola o contrato e pode produzir resultados imprevisíveis ao encontrar elementos iguais. Em particular, a ordenação pode falhar com uma exceção para determinados dados.
  • O método de comparação nunca retorna valores positivos ou negativos. Para cumprir o contrato, se um método de comparação retorna um valor positivo, ele também deve retornar um valor negativo se os argumentos forem fornecidos na ordem inversa.
  • O método de comparação retorna Integer.MIN_VALUE. Embora permitido pelo contrato, isso pode ser propenso a erros, pois alguns locais de chamada podem tentar inverter incorretamente o valor de retorno do método de comparação usando um operador de menos unário. O valor negado de Integer.MIN_VALUE é Integer.MIN_VALUE.

Exemplo:


  Comparator<String> lambda =
    (a, b) -> a.length() > b.length()
              ? 0
              : Math.random() > 0.5 ? -1 : 1;
