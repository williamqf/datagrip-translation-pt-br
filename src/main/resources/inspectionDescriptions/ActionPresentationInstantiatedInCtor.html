Relata qualquer ação registrada no arquivo plugin.xml que instancie um objeto com.intellij.openapi.actionSystem.Presentation em seu construtor.

Qualquer construtor de AnAction com parâmetros instancia um objeto Presentation. No entanto, instanciar um objeto Presentation no construtor resulta em uma alocação de recursos que pode ser desnecessária. Em vez de criar uma instância de Presentation para armazenar texto, descrição ou ícones, é mais eficiente utilizar construtores sem argumentos de AnAction e outras classes base, seguindo a convenção de configurar texto, descrição e ícones no plugin.xml. O IDE carregará o texto, a descrição e os ícones apenas quando a ação for efetivamente exibida na UI.

A convenção para configurar texto, descrição e ícones é a seguinte:

  • Defina o atributo id para a ação no arquivo plugin.xml.
  • Defina o atributo icon se um ícone for necessário.
  • Defina o texto e a descrição no pacote de mensagens associado (pode ser sobrescrito em <actions>):
    • action.<action-id>.text=Texto da ação traduzido
    • action.<action-id>.description=Descrição da ação traduzida

Exemplo ruim:


  // NewKotlinFileAction.kt
  internal class NewKotlinFileAction : AnAction(
    KotlinBundle.message("action.new.file.text"),
    KotlinBundle.message("action.new.file.description"),
    KotlinIcons.FILE
  )


  
  <action
    class="org.jetbrains.kotlin.idea.actions.NewKotlinFileAction"
  </action>

Exemplo bom:


  // NewKotlinFileAction.kt
  internal class NewKotlinFileAction : AnAction()


  
  <action
    id="Kotlin.NewFile"
    class="org.jetbrains.kotlin.idea.actions.NewKotlinFileAction"
    icon="org.jetbrains.kotlin.idea.KotlinIcons.FILE">
  </action>


  # KotlinBundle.properties
  action.Kotlin.NewFile.text=Classe/arquivo Kotlin
  action.Kotlin.NewFile.description=Cria uma nova classe ou arquivo Kotlin
