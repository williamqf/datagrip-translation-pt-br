Relata métodos sobrecarregados que utilizam interfaces funcionais com assinaturas de métodos abstratos conflitantes.

Tal sobrecarga introduz ambiguidade e exige que o chamador realize o cast do lambda para um tipo específico ou especifique explicitamente os tipos dos parâmetros do lambda. É recomendável atribuir nomes diferentes aos métodos sobrecarregados para eliminar a ambiguidade.

Exemplo:


  interface MyExecutor {
    void execute(Supplier<?> supplier);
    void execute(Callable<?> callable);
  }

Aqui, Supplier e Callable são interfaces funcionais com um único método abstrato que não aceita parâmetros e retorna um valor não void. Como resultado, o tipo do lambda não pode ser inferido no local da chamada, a menos que um cast explícito seja utilizado.
