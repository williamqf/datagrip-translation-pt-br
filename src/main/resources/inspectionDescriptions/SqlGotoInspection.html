Relata o uso de instruções GOTO para trás e instruções GOTO usadas para sair de loops.

O uso extensivo de instruções GOTO geralmente não é recomendado. Para mais detalhes, consulte GOTO statement in SQL procedures no ibm.com.

Considere o uso de loops em vez de usar GOTO para retornar a uma instrução anterior.

Considere o uso de outra instrução de controle de fluxo, como RETURN ou BREAK, em vez de usar GOTO para sair de um loop WHILE.

Exemplo (Oracle):

CREATE PROCEDURE test(n INT) AS
DECLARE
    x INT;
BEGIN
    x := 0;
    GOTO a;
    <<a>> x := 1;
    IF (n = 0) THEN
        GOTO a;
    END IF;
    WHILE TRUE
        LOOP
            GOTO b;
        END LOOP;
    <<b>> x := 3;
END;
