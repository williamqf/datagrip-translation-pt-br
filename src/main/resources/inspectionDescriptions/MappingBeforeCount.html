Relata chamadas redundantes da API Stream, como map() ou boxed() antes de uma chamada count().

Tais chamadas não alteram a contagem final e, portanto, podem ser removidas. O código pode depender de efeitos colaterais de lambdas nessas chamadas de mapeamento. No entanto, depender de efeitos colaterais dentro de uma cadeia de Stream é uma prática extremamente ruim. Não há garantia de que tal chamada não será otimizada em versões futuras do Java.

Exemplo:


  // a chamada map() é redundante
  long count = list.stream().filter(s -> !s.isEmpty()).map(s -> s.trim()).count();

Novidade na versão 2024.1
