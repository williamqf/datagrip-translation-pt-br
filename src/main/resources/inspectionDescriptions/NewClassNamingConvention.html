Reporta classes com nomes muito curtos, muito longos ou que não seguem o padrão de expressão regular especificado.

Exemplo: se a inspeção estiver ativada para testes e o comprimento mínimo do nome da classe for especificado como 8 (padrão), a seguinte classe de teste gerará um aviso, pois o comprimento do nome é 6, que é menor que 8: public class MyTest{}.

A correção rápida para renomear tal classe está disponível apenas no editor.

Configuração da inspeção:

Use a lista na seção Opções para especificar quais classes devem ser verificadas. Desmarque a caixa de seleção das classes para as quais deseja ignorar a verificação.

Para cada tipo de classe, use os campos de entrada fornecidos para especificar o comprimento mínimo, o comprimento máximo e a expressão regular para o nome da classe. Especifique 0 nos campos de comprimento para ignorar a verificação correspondente.

As expressões regulares devem ser especificadas no formato padrão java.util.regex.
