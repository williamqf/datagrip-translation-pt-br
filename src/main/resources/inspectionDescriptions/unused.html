Relata classes, métodos ou campos que não são utilizados ou não podem ser acessados a partir de pontos de entrada.

Um ponto de entrada pode ser um método main, testes, classes de fora do escopo especificado, classes acessíveis a partir do module-info.java, etc. Pontos de entrada personalizados podem ser configurados usando padrões de nome ou anotações.

Exemplo:


  public class Department {
    private Organization myOrganization;
  }

Neste exemplo, Department referencia explicitamente Organization, mas se a classe Department em si não for utilizada, a inspeção relatará ambas as classes.

A inspeção também relata parâmetros formais não utilizados por seus métodos e por todas as implementações e sobrescritas de métodos, bem como variáveis locais declaradas, mas não utilizadas.

Observação: Alguns membros não utilizados podem não ser relatados durante o realce de código no editor. Por razões de desempenho, membros não privados são verificados apenas se o nome deles aparecer raramente no projeto. Para ver todos os resultados, execute a inspeção selecionando Código | Inspecionar código ou Código | Analisar código | Executar inspeção por nome no menu principal.

Use as configurações de visibilidade abaixo para configurar quais membros relatar. Por exemplo, configurar para relatar apenas métodos private significa que métodos public de classes internas private serão relatados, enquanto métodos protected de classes de nível superior serão ignorados.

Use a guia Pontos de entrada para configurar os pontos de entrada a serem considerados durante a execução da inspeção.

Pontos de entrada podem ser adicionados manualmente assim que os resultados da inspeção estiverem prontos.

Se o seu código usa um framework não suportado, existem várias opções:

  • Se o framework depender de anotações, use o botão Anotações... para configurar as anotações do framework.
  • Se o framework não depender de anotações, tente configurar padrões de nome de classe que o framework espera.

Dessa forma, o código anotado acessível por componentes internos do framework será considerado como utilizado.
