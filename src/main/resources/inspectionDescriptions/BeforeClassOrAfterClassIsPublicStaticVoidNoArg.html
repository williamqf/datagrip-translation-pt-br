A inspeção informa métodos JUnit 4.0 @BeforeClass ou @AfterClass que não são declarados como public e static, não retornam void ou recebem argumentos. Tais métodos podem ser criados sem querer, mas o executor de teste JUnit não irá executá-los.

Exemplo:


  public class MainTest {
    @BeforeClass
    String beforeClass(int i) {
    }
  }

Após a aplicação da correção rápida, o método passa a ser:


  class MainTest {
    @BeforeClass
    public static void beforeClass() {
    }
  }
