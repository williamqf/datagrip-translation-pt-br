Relata anotações @GuardedBy nas quais o campo de guarda não é final.

A proteção em um campo não final pode levar a condições de corrida inesperadas, pois o valor do campo seria bloqueado (podendo mudar), em vez do próprio campo.

Exemplo:


    private ReadWriteLock lock = new ReentrantReadWriteLock(); // campo de guarda não final
    private Object state;

    @GuardedBy("lock")
    public void bar() {
        state = new Object();
    }

As anotações @GuardedBy suportadas são:

  • net.jcip.annotations.GuardedBy
  • javax.annotation.concurrent.GuardedBy
  • org.apache.http.annotation.GuardedBy
  • com.android.annotations.concurrency.GuardedBy
  • androidx.annotation.GuardedBy
  • com.google.errorprone.annotations.concurrent.GuardedBy
