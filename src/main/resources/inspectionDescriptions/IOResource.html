Relata recursos de E/S que não foram fechados com segurança. Os recursos de E/S verificados por esta inspeção incluem java.io.InputStream, java.io.OutputStream, java.io.Reader, java.io.Writer, java.util.zip.ZipFile, java.io.Closeable e java.io.RandomAccessFile.

Recursos de E/S encapsulados por outros recursos de E/S não são relatados, pois o recurso que encapsula fechará o recurso encapsulado.

Por padrão, esta inspeção pressupõe que o recurso pode ser fechado por qualquer método que contenha 'close' ou 'cleanup' no nome.

Exemplo:


  void save() throws IOException {
    FileWriter writer = new FileWriter("filename.txt"); //aviso
    writer.write("sample");
  }

Configure esta inspeção usando as seguintes opções:

  • Listar classes de recursos de E/S que não precisam ser fechadas e que devem ser ignoradas por esta inspeção.
  • Se deve ser permitido abrir recursos de E/S dentro de um bloco try. Este estilo é menos desejável porque é mais verboso do que abrir o recurso antes do bloco try.
  • Se o recurso pode ser fechado por qualquer chamada de método que passe o recurso como um argumento.
