Relata métodos que sobrescrevem métodos de uma superclasse, mas não possuem a anotação @java.lang.Override.

O uso da anotação @java.lang.Override melhora a legibilidade do código, pois indica explicitamente a intenção de sobrescrever um método. Além disso, o compilador emitirá um erro se a assinatura do método sobrescrito não corresponder à do método da superclasse.

Exemplo:


    class X {
      public String toString() {
        return "hello world";
      }
    }

Após a aplicação da correção rápida:


    class X {
      @Override
      public String toString() {
        return "hello world";
      }
    }

Configuração da inspeção:

  • Use a opção Ignorar métodos 'equals()', 'hashCode()' e 'toString()' para ignorar os seguintes métodos de java.lang.Object: equals(), hashCode() e toString(). O risco desses métodos desaparecerem e o código deixar de compilar devido à anotação @Override é relativamente pequeno.
  • Use a opção Ignorar métodos em classes anônimas para ignorar métodos em classes anônimas.
  • Desative a opção Destacar o método quando nem todos os seus métodos sobrescreventes possuírem a anotação '@Override' para alertar apenas sobre o método com a anotação @Override ausente, em vez de alertar sobre um método sobrescrito onde um ou mais descendentes não possuam a anotação @Override.
