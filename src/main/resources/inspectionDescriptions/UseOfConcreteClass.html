Relata o uso de classes concretas em vez de interfaces. Tais declarações podem representar uma falha de abstração e podem tornar os testes mais difíceis.

Esta inspeção não relata declarações cujas classes provenham do sistema ou de bibliotecas de terceiros. Casts, instanceof e variáveis locais não são relatados em implementações do método equals(). Além disso, casts não são relatados em implementações do método clone().

Exemplo:


  interface Entity {}
  class EntityImpl implements Entity {}

  void processObject(Object obj) {
    // Aviso: instanceof de classe concreta
    if (obj instanceof EntityImpl) {
      // Aviso: cast para classe concreta,
      // em vez de interface
      processEntity((EntityImpl)obj);
    }
  }
  // Aviso: parâmetro de classe concreta
  void processEntity(EntityImpl obj) {
  }

Use a opção Ignorar tipos de classe abstrata para ignorar casts para classes abstratas.

Use as opções subsequentes para controlar o contexto no qual os problemas são relatados.
