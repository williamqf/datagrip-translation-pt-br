Relata instruções when não exaustivas que causarão erros de compilação a partir da versão 1.7.

Tipos de motivação:

  • Padrões de uso problemáticos/sem sentido que precisam ser desencorajados/bloqueados (por exemplo, comportamento contraintuitivo)
    • O código é propenso a erros
  • Inconsistência no design (tratamento diferente em diferentes contextos)

Tipos de impacto:

  • Compilação. Alguns códigos que costumavam ser compilados não serão mais compilados
    • Em alguns casos, tais códigos funcionavam sem exceções
      • Alguns desses códigos podiam ser compilados sem qualquer aviso

Mais detalhes: KT-47709: Tornar as instruções when com sujeitos enum, sealed e boolean exaustivas por padrão

A correção rápida adiciona o ramo else -> {} ausente.

Exemplo:


  sealed class Base {
      class A : Base()
      class B : Base()
  }

  fun test(base: Base) {
      when (base) {
          is Base.A -> ""
      }
  }

Após aplicar a correção rápida:


  sealed class Base {
      class A : Base()
      class B : Base()
  }

  fun test(base: Base) {
      when (base) {
          is Base.A -> ""
          else -> {}
      }
  }

Esta inspeção só é relatada se o nível de linguagem Kotlin do projeto ou módulo for 1.6 ou superior.
