Relata condições duplicadas em expressões && e ||, bem como em ramos de instruções if. Essas condições duplicadas às vezes são intencionais, mas geralmente são resultado de um descuido.

Exemplo:


  boolean result = digit1 != digit2 || digit1 != digit2;

Para ignorar condições que possam produzir efeitos colaterais, use a opção Ignorar condições com efeitos colaterais. Desativar esta opção pode levar a falsos positivos, por exemplo, quando o mesmo método retorna valores diferentes em chamadas subsequentes.

Exemplo:


  native boolean unknownMethod();
  
  ...
  
  if (unknownMethod() || unknownMethod()) {
    System.out.println("Got it");
  }

Devido aos possíveis efeitos colaterais do unknownMethod() (no exemplo), o alerta só será disparado se a opção Ignorar condições com efeitos colaterais estiver desativada.
