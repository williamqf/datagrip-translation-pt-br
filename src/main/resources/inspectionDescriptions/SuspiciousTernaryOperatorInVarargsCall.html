Relata chamadas de método vararg que usam um operador ternário com ramos de array e não-array mistos.

No tempo de compilação, ambos os ramos são envolvidos em arrays. Como resultado, o ramo de array torna-se um array bidimensional, o que pode indicar um problema.

A correção rápida envolverá o ramo que não é array em um array para evitar que o compilador realize a conversão.

Exemplo:


    static void bar(boolean flag) {
        Object[] a = {1, 2};
        Object b = "hello";
        foo(flag ? a : b);
    }
    static void foo(Object... obj) {
    }

Após a aplicação da correção rápida: 


    static void bar(boolean flag) {
        Object[] a = {1, 2};
        Object b = "hello";
        foo(flag ? a : new Object[]{b});
    }
    static void foo(Object... obj) {
    }
