Relata condições que são redundantes por serem completamente cobertas por condições subsequentes.

Por exemplo, na condição value != -1 && value > 0, a primeira parte é redundante: se ela for false, a segunda parte também será false. Ou em condições como obj != null && obj instanceof String, a verificação de null é redundante porque o operador instanceof implica não ser null.

Novidade na versão 2018.3
