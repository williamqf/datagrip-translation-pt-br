Relata instruções defer dentro de loops.

O uso de defer dentro de um loop pode causar vazamentos de recursos ou uma ordem de execução imprevisível das instruções.

Exemplo:

func main() {
 for {
  field, err := db.Query("SELECT 1")
  if err != nil {
   // ...
  }
  defer field.Close()
  // ...
 }
}

A chamada defer row.Close() dentro do loop não será executada até que a função termine sua execução, e não ao final de cada iteração do loop for. Essa implementação pode levar ao estouro da pilha (stack overflow) da função e outros problemas.
