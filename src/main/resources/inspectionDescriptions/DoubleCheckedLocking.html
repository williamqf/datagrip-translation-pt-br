Relata o bloqueio de verificação dupla (double-checked locking).

O bloqueio de verificação dupla tenta inicializar campos sob demanda de maneira thread-safe, evitando ao mesmo tempo o custo de sincronização. Infelizmente, ele não é thread-safe quando usado em campos que não foram declarados como volatile. Ao usar o Java 1.4 ou versões anteriores, o bloqueio de verificação dupla não funciona mesmo para campos volatile. Leia o artigo no link acima para uma explicação detalhada sobre o problema.

Exemplo de bloqueio de verificação dupla incorreto:


  class Foo {
    private Helper helper = null;
    public Helper getHelper() {
      if (helper == null)
        synchronized(this) {
          if (helper == null) helper = new Helper();
        }
        return helper;
      }
    }
    // Outras funções e membros...
  }
