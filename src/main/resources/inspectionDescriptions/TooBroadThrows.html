Relata cláusulas throws que são mais genéricas do que as exceções realmente lançadas pelo método.

Exemplo:


  public void createFile() throws Exception { // Aviso: 'throws Exception' é muito amplo, ocultando a exceção 'IOException'
    File file = new File("pathToFile");
    file.createNewFile();
  }

Após aplicar a correção rápida:


  public void createFile() throws IOException {
    File file = new File("pathToFile");
    file.createNewFile();
  }

Configuração da inspeção:

  • Use o campo Número máximo de exceções ocultas para alertar para ignorar exceções que ocultam mais do que o número especificado de outras exceções.
  • Use a opção Avisar apenas para RuntimeException, Exception, Error ou Throwable para que esta inspeção alerte apenas sobre as exceções mais comuns.
  • Use a opção Ignorar exceções declaradas em métodos que sobrescrevem métodos de biblioteca para ignorar cláusulas throws excessivamente amplas em métodos que sobrescrevem métodos de biblioteca.
  • Use a opção Ignorar exceções que ocultam outras exceções, mas são elas mesmas lançadas para ignorar qualquer exceção que oculte outras exceções, mas que ainda possa ser lançada pelo corpo do método e, portanto, tecnicamente não seja excessivamente ampla.
