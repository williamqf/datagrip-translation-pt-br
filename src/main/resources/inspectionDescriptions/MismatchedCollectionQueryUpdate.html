Relata coleções cujo conteúdo é consultado, mas não atualizado, ou atualizado, mas não consultado.

Tais consultas e atualizações inconsistentes não fazem sentido e podem indicar código morto ou erros de digitação.

Use as configurações de inspeção para especificar padrões de nome correspondentes aos métodos de atualização/consulta. Métodos de consulta que retornam elementos são detectados automaticamente; você só precisa especificar métodos de consulta que gravam dados em parâmetros de saída (por exemplo, um OutputStream).

Exemplo:

Suponha que você tenha uma classe FixedStack personalizada que contém o método store(): 


  public class FixedStack<T> extends Collection<T> {
    public T store(T t) {
      // implementação
    }
  }

Você pode adicionar store à tabela de métodos de atualização para relatar consultas sem correspondência, por exemplo: 


  void test(int i) {
    FixedStack<Integer> stack = new FixedStack<>();
    stack.store(i);
  }
