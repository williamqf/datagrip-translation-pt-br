Relatórios sobre métodos com nomes do tipo acessador que tem tipo de resultado Unit.

Métodos que seguem o contrato de nomenclatura JavaBean para acessadores são esperados por não apresentarem efeitos colaterais. Entretanto, métodos com um tipo de resultado Unit são executados somente para seus efeitos colaterais.

Isso indica que o método deveria resultar em um tipo diferente de Unit ou ser renomeado para refletir melhor o efeito colateral.

Exemplo:


  trait Test {
    def getTarget(): Unit
  }

Esta inspeção não relatará métodos que sobrepõem outros membros.
