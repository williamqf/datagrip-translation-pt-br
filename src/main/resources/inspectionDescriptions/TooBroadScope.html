Relata quaisquer declarações de variáveis que podem ser movidas para um escopo menor.

Esta inspeção é especialmente útil para declarações de estilo Pascal no início de métodos. Além disso, variáveis com um escopo muito amplo são frequentemente deixadas para trás após a refatoração.

Exemplo:


  StringBuilder sb = new StringBuilder();
  System.out.println();
  sb.append(1);

Após a aplicação da correção rápida:


  System.out.println();
  StringBuilder sb = new StringBuilder();
  sb.append(1);

Configuração da inspeção:

  • Use a opção Relatar apenas variáveis que podem ser movidas para dentro de blocos internos para relatar apenas as variáveis que podem ser movidas para blocos de código mais profundos. Por exemplo, quando esta opção estiver ativada, a movimentação da variável sb acima não será sugerida. No entanto, ela será sugerida para o seguinte código: 
    
  StringBuilder sb = new StringBuilder(a);
  if (flag) {
    sb.append(1);
  }
  • Use a opção Relatar variáveis com novas expressões como inicializadores (potencialmente inseguro) para relatar variáveis inicializadas com novas expressões. Isso torna a inspeção potencialmente insegura quando o construtor possui efeitos colaterais não locais. Por exemplo, quando esta opção estiver ativada, a movimentação da variável foo será sugerida: 
    
  class Foo {
    static List<Foo> fooList = new ArrayList<>();
    String bar;

    Foo(String bar) {
        this.bar = bar;
        fooList.add(this);
    }

    public static void main(String[] args) {
      // Embora inseguro, também pode ser movido
      Foo foo = new Foo("bar");
      System.out.println(fooList.size());
      System.out.println(foo.bar);
    }
  }
