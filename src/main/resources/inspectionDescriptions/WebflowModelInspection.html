Relata erros de configuração em definições <flow> baseadas em XML do Spring Webflow.

  • Referências de Bean não resolvidas
  • Tags ou atributos obrigatórios ausentes
  • Tipos de propriedade incorretos
  • Propriedades enum inconsistentes
  • Tipo incorreto do Bean referenciado

Exemplo:


  <flow
    start-state="unknown-state" <!-- Não é possível resolver o símbolo 'unknown-state' ->
    parent="unknownParentFlow,another-flow" <!-- Não foi possível encontrar o fluxo 'unknownParentFlow' -->>

    <input name="" // O valor não deve estar vazio
         type="foo.com.UnknownType" <!-- Não é possível resolver a classe 'UnknownType' -->
         value="42"/>

    <exception-handler bean="dummyBean"; <!-- O Bean deve ser do tipo 'org.springframework.webflow.engine.FlowExecutionExceptionHandler' -->

    <view-state id="validFlowButInvalidState_forParent"
           parent="another-flow#unknownParentState"/> <!-- Não é possível resolver o estado pai 'unknownParentState' -->

    <bean-import resource="foo.xml" /> <!-- Não é possível resolver o arquivo 'foo.xml' -->
    <bean-import resource="highlighting-20-wrong-flow.xml"/> <!-- Não é um arquivo spring.xml válido -->
  </flow>
