Relata parênteses desnecessários em expressões, tipos e padrões.

Parênteses são considerados desnecessários se a semântica do programa permanecer inalterada quando os parênteses forem removidos.

Use as caixas de seleção abaixo para personalizar o comportamento da inspeção.

Parênteses são considerados esclarecedores se eles separam uma expressão infixa, elemento de tipo ou padrão como parte de um elemento infixo maior.

Exemplo:


  val x = (3)

Após a correção rápida ser aplicada:


  val x = 3
