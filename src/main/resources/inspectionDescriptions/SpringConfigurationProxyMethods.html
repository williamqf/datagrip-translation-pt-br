Relata avisos sobre métodos proxy usados incorretamente. O Spring Framework 5.2 introduziu uma otimização para o processamento de classes @Configuration, que pode ser habilitada por meio do atributo @Configuration(proxyBeanMethods = false).

Se o proxyBeanMethods for desativado, as instâncias proxy não serão mais criadas e a chamada ao método poderá ser executada novamente (retornando uma nova instância a cada vez). Como resultado, você não tem garantia de que está realmente injetando o Bean correspondente no contexto.

Exemplo de chamada de método Bean incorreta:


@Configuration(proxyBeanMethods = false)
class TestConfiguration {
  @Bean
  public FirstBean firstBean() {
    return new FirstBean();
  }

  @Bean
  public SecondBean secondBean() {
    return new SecondBean(firstBean()); // -> Chamada incorreta
  }
}

Você pode definir proxyBeanMethods como true ou reescrever o código conforme mostrado abaixo:


@Configuration(proxyBeanMethods = false)
class TestConfiguration {
  @Bean
  public FirstBean firstBean() {
    return new FirstBean();
  }

  @Bean
  public SecondBean secondBean(FirstBean someBean) { // -> Instância injetada corretamente
    return new SecondBean(someBean);
  }
}

Além disso, esta inspeção verifica chamadas de métodos @Bean em classes sem o estereótipo @Configuration (no "modo Bean lite"):


@Component
class TestComponent {
  @Bean
  public FirstBean firstBean() {
    return new FirstBean();
  }

  @Bean
  public SecondBean secondBean() {
    return new SecondBean(firstBean()); // -> Chamada incorreta
  }
}
