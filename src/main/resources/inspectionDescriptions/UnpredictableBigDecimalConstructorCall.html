Reporta chamadas aos construtores de BigDecimal que aceitam um valor double. Esses construtores produzem um BigDecimal que é exatamente igual ao valor double fornecido. No entanto, como o double é codificado no formato de ponto flutuante binário de precisão dupla de 64 bits IEEE 754, o valor exato pode ser inesperado.

Por exemplo, new BigDecimal(0.1) produz um objeto BigDecimal com o valor 0.1000000000000000055511151231257827021181583404541015625, que é o número mais próximo de 0.1 representável como um double. Para obter um BigDecimal contendo o valor esperado 0.1, use new BigDecimal("0.1") ou BigDecimal.valueOf(0.1).

Exemplo:


  class Constructor {
    void foo() {
      new BigDecimal(0.1);
    }
  }

Após a aplicação da correção rápida:


  class Constructor {
    void foo() {
      new BigDecimal("0.1");
    }
  }
