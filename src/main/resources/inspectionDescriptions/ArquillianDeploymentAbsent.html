Relata classes de caso de teste Arquillian sem implantação.

Segundo a documentação do Arquillian, as classes de teste Arquillian devem definir o arquivo de implantação utilizando um método public static com uma assinatura específica e anotado com @Deployment.

A ação rápida adicionará o método necessário de acordo com o modelo de código editável pelo usuário Método de implantação Arquillian.

Exemplo:


  // Este caso de teste não pode ser inicializado pelo Arquillian
  @RunWith(Arquillian.class)
  public class SomeTest {
    @Test
    public void testSomething() {
      Assert.fail("To be implemented");
    }
  }

Após a aplicação da correção rápida:


  @RunWith(Arquillian.class)
  public class SomeTest {
    // Adicionado pela correção rápida
    @Deployment
    public static JavaArchive ${DEPLOYMENT_NAME}() {
      return ShrinkWrap.create(JavaArchive.class)
                       .addAsManifestResource(EmptyAsset.INSTANCE, "beans.xml");
    }

    @Test
    public void testSomething() {
      Assert.fail("To be implemented");
    }
  }
