Reporta instruções return com valor null. Embora ocasionalmente útil, essa construção pode fazer com que o código falhe devido a uma maior propensão a NullPointerException.

Se um método for projetado para retornar null, recomenda-se marcá-lo com a anotação @Nullable - tais métodos serão ignorados por esta inspeção.

Exemplo:


  class Person {
    public String getName () {
      return null;
    }
  }

Após a aplicação do ajuste rápido:


  class Person {
    public @Nullable String getName () {
      return null;
    }
  }

Se o tipo de retorno for java.util.Optional, um ajuste rápido adicional é sugerido para converter null em Optional.empty().

Configure a inspeção utilizando as seguintes opções:

  • Se deve ignorar métodos private. Isso também ignorará retornos null de classes anônimas e lambdas.
  • Se devem ser relatados valores null retornados para arrays, objetos de coleção, objetos simples ou uma combinação dos três. Sempre relata null retornados em métodos cujo tipo de retorno seja java.util.Optional.
  • Clique em Configurar anotações para especificar quais anotações devem ser consideradas como 'nullable'.
