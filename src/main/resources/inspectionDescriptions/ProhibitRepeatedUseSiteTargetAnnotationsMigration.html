Relata o uso repetido de anotações não @Repeatable em acessadores de propriedade.

Devido ao uso múltiplo de uma anotação não @Repeatable, ambos os usos da anotação aparecerão no bytecode, resultando em ambiguidade em chamadas de reflexão.

A partir do Kotlin 1.4, as anotações devem ser marcadas como @Repeatable ou não devem ser repetidas; caso contrário, ocorrerá um erro de compilação.

Exemplo:


  annotation class Foo(val x: Int)

  @get:Foo(10)
  val a: String
      @Foo(20) get() = "foo" // Anotação repetida duas vezes, mas não marcada como @Repeatable

Esta inspeção relata apenas se o nível de linguagem Kotlin do projeto ou módulo for 1.4 ou superior.
