Relata métodos públicos que aceitam parâmetros boolean.

Se um método público (parte de uma API) não for um setter, adicionar um parâmetro boolean a esse método é quase sempre uma má prática. Ao ler código que utiliza tais métodos, pode ser difícil entender o significado do boolean sem consultar o código-fonte ou a documentação.

Este problema também é conhecido como a armadilha do booleano (boolean trap). Parâmetros boolean geralmente podem ser substituídos por um enum.

Exemplo:


  // Aviso: ao olhar para chamadas a este método,
  // é difícil entender o significado dos
  // parâmetros booleanos
  public boolean setPermission(File f,
                               int access,
                               boolean enable,
                               boolean ownerOnly) {
    // ...
  }

Use a opção Relatar apenas métodos com vários parâmetros booleanos para emitir um aviso apenas quando um método contiver vários parâmetros booleanos.
