Relata o uso incorreto das anotações @Rule ou @ClassRule. Não é possível executar testes com regras mal formadas.
  • O membro anotado com @Rule deve ser public e um subtipo de org.junit.rules.TestRule ou org.junit.rules.MethodRule
  • O membro anotado com @ClassRule deve ser public e static e um subtipo de org.junit.rules.TestRule

Exemplo:


  public class MomentousTest {
    @Rule
    private final TemporaryFolder tempFolder =
      new TemporaryFolder();

    // ... o teste vai até aqui
  }

Correção rápida disponível para corrigir o modificador:


  public class MomentousTest {
    @Rule
    public final TemporaryFolder tempFolder =
      new TemporaryFolder();

    // ... o teste vai até aqui
  }
