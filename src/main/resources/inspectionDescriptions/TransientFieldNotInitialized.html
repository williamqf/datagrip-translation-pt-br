Relata campos transient que são inicializados durante a construção normal do objeto, mas cuja classe não possui o método readObject.

Os campos transient não são serializados e, portanto, precisam ser inicializados separadamente no método readObject() durante a desserialização.

Qualquer campo transient que não seja inicializado durante a construção normal do objeto é considerado como utilizando a inicialização padrão e não será relatado por esta inspeção.

Exemplo:


  class Person implements Serializable {
    transient String name = "Default"; // Aviso: na verdade pode ser null após a desserialização
    transient String surname; // null é considerado como o valor padrão e não é relatado
  }
