Relata métodos que podem ser definidos como static com segurança. Tornar métodos static sempre que possível reduz o consumo de memória e melhora a qualidade do código.

Um método pode ser static se as seguintes condições forem atendidas:

  • Não é synchronized, native ou abstract,
  • Não faz referência a nenhum método não static ou campo não static na classe que o contém,
  • Não é uma sobrescrita e não é sobrescrito em subclasses.

Configure a inspeção usando as seguintes opções:

  • Se deve relatar apenas métodos private e final para melhorar o desempenho desta inspeção.
  • Se deve ignorar métodos vazios.
  • Se deve ignorar métodos default em interfaces ao usar o Java 8 ou superior.
  • Se a correção rápida deve substituir o qualificador de instância por uma referência de classe ao chamar um método que foi definido como static; por exemplo, a chamada myClass.m() será substituída por MyClass.m().
