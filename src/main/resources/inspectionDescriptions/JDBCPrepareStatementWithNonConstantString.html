Relata chamadas a java.sql.Connection.prepareStatement(), java.sql.Connection.prepareCall() ou qualquer uma de suas variantes que aceitem uma string construída dinamicamente como a instrução a ser preparada.

Instruções SQL construídas são uma fonte comum de vulnerabilidades de segurança. Por padrão, esta inspeção ignora constantes de tempo de compilação.

Exemplo:


  String bar() { return "bar"; }

  Connection connection = DriverManager.getConnection("", "", "");
  connection.("SELECT * FROM user WHERE name='" + bar() + "'");
Use as configurações de inspeção para tratar qualquer campo static final como uma constante. Tenha cuidado, pois quando esta opção estiver ativada, strings como a seguinte serão ignoradas: 

  static final String SQL = "SELECT * FROM user WHERE name='" + getUserInput() + "'";
