Relatórios de métodos com nomes de tipo mutators que não possuem cláusulas de parâmetros.

Para um método com efeitos colaterais, é uma convenção ter uma cláusula de parâmetros vazia para indicar que não é apenas um método de acesso.

O quick-fix adiciona uma cláusula de parâmetros vazia.

Exemplo:


  trait Test {
    def updateAll: Boolean
  }

Após o quick-fix ser aplicado:


  trait Test {
    def updateAll(): Boolean
  }

レポートには、ミューテーターのような名前を持ち、パラメーターの節を持たないメソッドが表示されます。

副作用を持つメソッドの場合、単なるアクセサーメソッドではないことを示すために空のパラメーターの節を持つことが一般的な慣習となっています。

クイックフィックスを使用すると、空のパラメーターの節が追加されます。

例:


  trait Test {
    def updateAll: Boolean
  }

クイックフィックス適用後:


  trait Test {
    def updateAll(): Boolean
  }

변경자와 같은 이름이 있지만 매개변수 절이 없는 메서드를 보고합니다.

부작용 메서드의 경우, 단순한 접근자 메서드가 아님을 나타내기 위해 빈 매개변수 절이 있는 것이 규칙입니다.

빠른 수정을 통해 빈 매개변수 절이 추가됩니다.

예:


  trait Test {
    def updateAll: Boolean
  }

빠른 수정을 적용한 후:


  trait Test {
    def updateAll(): Boolean
  }

报告具有类似转换器名称且没有参数子句的方法。

对于具有副作用的方法，通常约定是一个空参数子句来指明它不仅仅是一个访问器方法。

快速修复添加一个空参数子句。

示例：


  trait Test {
    def updateAll: Boolean
  }

在应用快速修复后：


  trait Test {
    def updateAll(): Boolean
  }
