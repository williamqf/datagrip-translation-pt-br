Detecta possíveis tokens (senha ou informação de segurança), inseridos em YAML, usando análise de entropia e expressões regulares.

Esta inspeção usa análise de entropia e expressões regulares para escanear o código base em busca de strings como senhas e tokens de segurança. Ela destaca os resultados encontrados para que desenvolvedores possam identificar vulnerabilidades em potencial e proteger o código. A eficácia desta inspeção depende dos padrões definidos em sua configuração, permitindo que ela se adapte a diferentes ambientes e requisitos de codificação. 


// Exemplo de string que pode ser detectada como possível token
possibleToken: pa#sw@rd;

// Exemplo de padrão de expressão regular usada para detecção
regexPattern = '/[0-9]+:AA[0-9A-Za-z\-_]{33}/'

O texto que segue esta observação será exibido apenas nas configurações de inspeção.
