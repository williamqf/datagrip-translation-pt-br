Relata loops while que realizam espera ativa (spin) sobre o valor de um campo não volatile, aguardando que ele seja alterado por outra thread.

Além de serem potencialmente muito intensivos em termos de CPU quando pouco trabalho é feito dentro do loop, tais loops provavelmente possuem uma semântica diferente da pretendida. O Modelo de Memória do Java permite que esses loops nunca terminem, mesmo que o valor do campo seja alterado por outra thread.

Além disso, a partir do Java 9, recomenda-se chamar Thread.onSpinWait() em loops de spin em campos volatile, o que pode melhorar significativamente o desempenho em alguns hardwares.

Exemplo:


  class SpinsOnField {
    boolean ready = false;

    void run() {
      while (!ready) {
      }
      // realiza algum trabalho
    }

    void markAsReady() {
      ready = true;
    }
  }

Após a aplicação da correção rápida:


  class SpinsOnField {
    volatile boolean ready = false;

    void run() {
      while (!ready) {
        Thread.onSpinWait();
      }
      // realiza algum trabalho
    }

    void markAsReady() {
      ready = true;
    }
  }

Use as opções de inspeção para relatar apenas loops while vazios.
