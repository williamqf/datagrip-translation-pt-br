Relata a concatenação de strings não constantes usadas como argumentos para métodos de log do SLF4J e Log4j 2. Concatenações não constantes são avaliadas em tempo de execução, mesmo que a mensagem de log não seja registrada; isso tem um impacto negativo no desempenho. Recomenda-se o uso de mensagens de log parametrizadas, que não são avaliadas quando o log está desativado.

Exemplo:


  public class Vital {
    private static final Logger LOG = LoggerFactory.getLogger(Vital.class);

    public void saveTheWorld(int i, String s, boolean b) {
      LOG.info("saveTheWorld(" + i + ", " + s + ", "  + b + ")");
      // todo
    }
  }

Após a aplicação do Quick Fix:


  public class Vital {
    private static final Logger LOG = LoggerFactory.getLogger(Vital.class);

    public void saveTheWorld(int i, String s, boolean b) {
      LOG.info("saveTheWorld({}, {}, {})", i, s, b);
      // todo
    }
  }

Configuração da inspeção:

  • Use a lista Avisar sobre para ignorar certos níveis de log mais altos. Níveis de log mais altos podem ser ativados mesmo em produção e os argumentos são sempre avaliados.
