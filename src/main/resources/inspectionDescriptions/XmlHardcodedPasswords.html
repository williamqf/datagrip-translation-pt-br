Detecta possíveis tokens de segurança ou senhas contidas em xml usando análise de entropia e expressões regulares.

Esta inspeção faz a varredura do código base usando análise de entropia e expressões regulares para buscar strings semelhantes a tokens de segurança ou senhas. Ela destaca os resultados encontrados para que os desenvolvedores possam identificar e proteger vulnerabilidades em potencial. A eficácia desta inspeção depende dos padrões definidos em sua configuração, o que a torna adaptável a diversos ambientes e requisitos de codificação. 


// Exemplo de string detectada como um token em potencial:
  <possibleToken>pa#sw@rd;</possibleToken>

// Exemplo de um padrão de expressão regular utilizado para detecção:
regexPattern = '/[0-9]+:AA[0-9A-Za-z\-_]{33}/'

Os textos que seguem este comentário serão exibidos somente nas configurações de inspeção.
