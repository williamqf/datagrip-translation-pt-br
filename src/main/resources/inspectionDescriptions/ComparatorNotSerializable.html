Relata classes que implementam java.lang.Comparator, mas não implementam java.io.Serializable.

Se um comparador não serializável for usado para construir coleções ordenadas como java.util.TreeMap ou java.util.TreeSet, a coleção também não será serializável. Isso pode resultar em erros inesperados e difíceis de diagnosticar.

As subclasses de java.lang.Comparator geralmente não possuem estado, portanto, esses problemas podem ser evitados simplesmente marcando-as como serializáveis.

Exemplo:


  class Foo implements Comparator { // Aviso
      @Override
      public int compare(Object o1, Object o2) {
          /* ... */
      }
  }

Após a aplicação da correção rápida:


  class Foo implements Comparator, Serializable { // Nenhum aviso aqui
      @Override
      public int compare(Object o1, Object o2) {
          /* ... */
      }
  }
