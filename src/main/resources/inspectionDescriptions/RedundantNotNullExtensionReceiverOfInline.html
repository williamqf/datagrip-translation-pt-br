Relata funções inline com receptores de extensão não anuláveis que não utilizam o fato de o receptor de extensão ser não anulável.

Antes do Kotlin 1.2, chamadas para inline fun usando receptores de extensão anuláveis flexíveis (tipos de plataforma com nulidade desconhecida) não incluíam verificações de nulidade no bytecode. A partir do Kotlin 1.2, as verificações de nulidade são incluídas no bytecode (consulte KT-12899).

Portanto, em versões anteriores ao Kotlin 1.2, funções que não utilizam o fato de o receptor de extensão ser não anulável são perigosas, e recomenda-se que tais funções tenham um receptor anulável.

Exemplo:


  inline fun String.greet() {
      println("Hello, $this!")
  }

  fun main() {
      // `System.getProperty` retorna o tipo `String!` não representável
      val user = System.getProperty("user.name")
      user.greet()
  }

Após a aplicação da correção rápida:


  inline fun String.greet() {
      println("Hello, $this!")
  }

  fun main() {
      // `System.getProperty` retorna o tipo `String!` não representável
      val user = System.getProperty("user.name")
      user.greet()
  }

Esta inspeção relata apenas se o nível de linguagem Kotlin do projeto ou módulo for inferior a 1.2.
