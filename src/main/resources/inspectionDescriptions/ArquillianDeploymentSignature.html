Informa sobre uma assinatura incorreta de métodos anotados com @Deployment

De acordo com a documentação do Arquillian, uma classe de teste do Arquillian deve definir um arquivo de implantação usando um método public static com uma assinatura específica, anotado com @Deployment.

Exemplo:


  // Este caso de teste não será iniciado pelo Arquillian, pois o método Deployment não é estático
  @RunWith(Arquillian.class)
  public class SomeTest {
    @Deployment
    public JavaArchive createDeployment() {
      throw new UnsupportedOperationException("Implement me");
    }

    @Test
    public void testSomething() {
      Assert.fail("To be implemented");
    }
  }

Após a correção rápida:


  @RunWith(Arquillian.class)
  public class SomeTest {
    @Deployment
    public static JavaArchive createDeployment() {
      throw new UnsupportedOperationException("Implement me");
    }

    @Test
    public void testSomething() {
      Assert.fail("To be implemented");
    }
  }
