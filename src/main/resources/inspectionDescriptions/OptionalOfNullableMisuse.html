Relata o uso de Optional.ofNullable() onde o argumento passado é sempre nulo ou sempre não nulo. Nesse caso, o uso de Optional.ofNullable() não faz sentido: Optional.empty() ou Optional.of() devem ser usados para declarar explicitamente a intenção de criar um Optional que é sempre vazio ou sempre não vazio, respectivamente. Também é possível que haja um erro no argumento de Optional.ofNullable(), portanto, ele deve ser verificado.

Exemplo: 


Optional<String> empty = Optional.ofNullable(null); // deve ser Optional.empty();
Optional<String> present = Optional.ofNullable("value"); // deve ser Optional.of("value");
