Relata classes vazias e arquivos Java vazios.

Uma classe é considerada vazia se não contiver campos, métodos, construtores ou inicializadores. Classes vazias às vezes permanecem após grandes mudanças ou refatorações.

Exemplo:


  class Example {
    List<String> getList() {
       return new ArrayList<>() {

       };
    }
  }

Após aplicar a correção rápida:


  class Example {
    List<String> getList() {
       return new ArrayList<>();
    }
  }

Configuração da inspeção:

  • Use a opção Ignorar se anotado por para especificar anotações especiais. A inspeção ignorará as classes marcadas com essas anotações.

  • Use a opção Ignorar se a classe for uma parametrização de um supertipo para ignorar classes que parametrizam uma superclasse. Exemplo: 

    class MyList extends ArrayList<String> {}
  • Use a opção Ignorar subclasses de java.lang.Throwable para ignorar classes que estendem java.lang.Throwable.
  • Use a opção Comentários contam como conteúdo para ignorar classes que contêm comentários.
