Relata rótulos redundantes que causam erros de compilação a partir do Kotlin 1.4.

Desde o Kotlin 1.0, qualquer instrução podia ser marcada com um rótulo:


  fun foo() {
    L1@ val x = L2@bar()
  }
No entanto, esses rótulos só podem ser referenciados de algumas formas limitadas:
  • break/continue de loops
  • Retornos não locais de um lambda inline ou função anônima inline

Tais rótulos são proibidos a partir do Kotlin 1.4.

Esta inspeção relata apenas se o nível da linguagem Kotlin do projeto ou módulo for 1.4 ou superior.
