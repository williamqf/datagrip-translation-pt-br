Relata nomes de <bean> que não seguem as convenções de nomenclatura de Bean.

A convenção ao nomear Beans é usar as convenções padrão do Java para nomes de campos de instância. Ou seja, o nome do Bean começa com uma letra minúscula e utiliza camelCase a partir daí. Exemplos de tais nomes incluem accountManager, accountService, userDao, loginController, etc. Nomear Beans de forma consistente torna sua configuração mais fácil de ler e entender. Além disso, se você estiver usando Spring AOP, isso é muito útil ao aplicar advice a um conjunto de Beans relacionados pelo nome.
