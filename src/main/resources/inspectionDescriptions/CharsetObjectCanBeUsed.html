Relata métodos e construtores onde literais String de conjuntos de caracteres constantes (por exemplo, "UTF-8") podem ser substituídos pelo código StandardCharsets predefinido.

A correção pode tornar o código mais rápido, pois a busca pelo conjunto de caracteres torna-se desnecessária. Além disso, você pode não precisar mais capturar UnsupportedEncodingException. Nesse caso, o bloco catch será removido automaticamente.

Exemplo:


  try {
    byte[] bytes = "str".getBytes("UTF-8");
  } catch (UnsupportedEncodingException e) {
  }

Após a aplicação do quick fix:


  byte[] bytes = "str".getBytes(StandardCharsets.UTF_8);

Esta inspeção está disponível no Java 7 e versões superiores.

Novidade na versão 2018.2
