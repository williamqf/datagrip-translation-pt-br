Relata structs inicializadas sem nomes de campos. Por padrão, esta inspeção está disponível apenas ao usar tipos definidos em pacotes diferentes.

Ao inicializar uma struct, recomenda-se indicar explicitamente os nomes dos campos para garantir que eles continuem a ser referenciados corretamente caso a ordem ou os nomes dos campos sejam alterados.

Exemplo:

_ = io.LimitedReader{nil, 10}

A inicialização de LimitedReader será destacada porque faltam os nomes explícitos dos campos da struct. Você pode aplicar o quick fix Adicionar chaves e remover valores zero a esta inicialização de struct. Após aplicar o quick fix, o código ficará assim:

_ = io.LimitedReader{N: 10}

Esta inspeção possui as seguintes opções:

  • Relatar tipos definidos no pacote atual: relata casos em que uma struct é declarada e inicializada no mesmo pacote. Considere o exemplo abaixo.

    d := struct {
  c string
 }{ "dsd "}

  • Relatar literais de struct sem tipo: relata campos sem nome quando o tipo (struct ou interface) é omitido. O exemplo a seguir mostra o caso em que o tipo é omitido.

    _ = []io.LimitedReader{
 {nil, 10},
 {nil, 20},
 }

    No exemplo abaixo, o tipo está presente, mas é redundante.

    _ = []io.LimitedReader{
 io.LimitedReader{nil, 10},
 io.LimitedReader{nil, 20},
}
