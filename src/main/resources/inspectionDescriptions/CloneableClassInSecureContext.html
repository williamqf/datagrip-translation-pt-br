Relata classes que podem ser clonadas.

Se uma classe for compatível com a interface Cloneable e seu método clone() não estiver definido para lançar um erro imediatamente, a classe poderá ser clonada. Classes clonáveis podem ser perigosas em códigos destinados ao uso seguro.

Exemplo:

class SecureBean implements Cloneable {}

Após a aplicação da correção rápida:

class SecureBean {}

Quando a classe estende uma classe clonável existente ou implementa uma interface clonável, após a aplicação da correção rápida, o código poderá ser o seguinte: 

class SecureBean extends ParentBean {
    @Override
    protected SecureBean clone() throws CloneNotSupportedException {
        throw new CloneNotSupportedException();
    }
}
