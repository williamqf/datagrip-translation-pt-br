Relata exceções lançadas de dentro de um bloco catch que não "envolvem" a exceção capturada.

Quando uma exceção é lançada em resposta a outra exceção, envolver a exceção inicial evita a perda de informações contextuais valiosas, como frames de pilha e números de linha.

Exemplo:


  ...
  catch (IOException e) {
    closeAllConnections();
    throw new ConnectException("Connection problem."); // Aviso: 'throw' dentro do bloco 'catch' ignora a exceção capturada 'e'
  }

Configuração da inspeção:

  • Use a opção Ignorar se o resultado de uma chamada de método de exceção for usado para indicar se a inspeção deve ignorar exceções onde o argumento é o resultado de uma chamada de método na exceção original, por exemplo, getMessage().
  • Use a opção Ignorar se a exceção lançada não puder envolver a exceção para ignorar instruções throw que lançam exceções sem um construtor que aceite uma causa Throwable.
