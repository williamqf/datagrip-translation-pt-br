Relata nomes de propriedades const que não seguem as convenções de nomenclatura recomendadas.

Uma nomenclatura consistente torna o código mais fácil de ler e entender. De acordo com o guia de estilo oficial do Kotlin, as propriedades const devem usar nomes em letras maiúsculas separados por sublinhados (underscores).

Exemplo:


  const val Planck: Double = 6.62607015E-34

A correção rápida renomeia a propriedade:


  const val PLANCK: Double = 6.62607015E-34
