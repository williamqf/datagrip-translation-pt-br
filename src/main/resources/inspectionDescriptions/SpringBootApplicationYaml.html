Relata chaves de configuração não resolvidas e depreciadas, bem como valores inválidos em arquivos de configuração .yaml de aplicativos Spring Boot, que podem causar erros em tempo de execução.

Exemplo:


server:
  port: invalid # relata 'não é possível converter 'invalid' para java.lang.Integer'

Se uma chave de configuração depreciada possuir uma chave de substituição, você poderá aplicar a correção rápida "Usar chave de substituição".

Exemplo:


logging:
  path: ${path} # relata 'propriedade de configuração depreciada 'logging.path''

Após aplicar a correção rápida:


logging:
  file:
    path: ${path}

Se uma chave de configuração não estiver definida em spring-configuration-metadata.json, você poderá aplicar a correção rápida "Definir chave de configuração" para criar o arquivo META-INF/spring-additional-configuration-metadata.json e definir as chaves necessárias.

Exemplo:


new:
  key: value # relata 'não foi possível resolver a propriedade de configuração 'new.key''

Após aplicar a correção rápida, o seguinte conteúdo será adicionado ao arquivo META-INF/spring-additional-configuration-metadata.json:


{
  "properties": [
    {
      "name": "new.key",
      "type": "java.lang.String",
      "description": "Description for new.key."
  }
] }

Use a opção Tokens de substituição para definir os tokens usados para placeholders de valores em arquivos de configuração. Esses tokens são especificados no formato beginToken*endToken. Se não houver um *, assume-se que os tokens inicial e final sejam idênticos.

Por exemplo, o valor padrão para os tokens inicial e final é @, o que permite definir placeholders como some.property=@another.property@.

Valores dentro de tokens de substituição (@property.key@) não serão destacados.
