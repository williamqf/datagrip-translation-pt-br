Relata expressões que podem ser substituídas por chamadas ao método Integer.compare() ou métodos semelhantes das classes Long, Short, Byte, Double ou Float (em vez de construções mais detalhadas ou menos eficientes).

Se x e y forem inteiros empacotados (boxed), x.compareTo(y) é sugerido; se forem primitivos, Integer.compare(x, y) é sugerido.

Exemplo:


  public int compare(int x, int y) {
    return x > y ? 1 : x < y ? -1 : 0;
  }

Após a aplicação da correção rápida:


  public int compare(int x, int y) {
    return Integer.compare(x, y);
  }

Observe que Double.compare e Float.compare alteram ligeiramente a semântica do código. Em particular, eles tornam -0.0 e 0.0 distinguíveis (Double.compare(-0.0, 0.0) produz -1). Além disso, eles tratam valores NaN de forma consistente. Na maioria dos casos, essa mudança semântica realmente melhora o código. Se a mudança semântica for inaceitável em seu caso, use a caixa de seleção para desativar esta inspeção para números de ponto flutuante.
