Reporta cláusulas redundantes que limitam linhas em uma consulta, como FETCH e LIMIT.

Exemplo (PostgreSQL):

CREATE TABLE foo(a INT);

SELECT * FROM foo WHERE EXISTS(SELECT * FROM foo LIMIT 2);
SELECT * FROM foo WHERE EXISTS(SELECT * FROM foo FETCH FIRST 2 ROWS ONLY);

Para corrigir o aviso, você pode adicionar um OFFSET à cláusula de limite. Se o OFFSET estiver ausente, o LIMIT é redundante, pois o uso de LIMIT não afeta o resultado da operação EXISTS. No caso de um OFFSET, as primeiras N linhas são ignoradas, o que afetará a saída.

SELECT * FROM foo WHERE EXISTS(SELECT * FROM foo OFFSET 1 ROW LIMIT 2);
SELECT * FROM foo WHERE EXISTS(SELECT * FROM foo OFFSET 1 ROW FETCH FIRST 2 ROWS ONLY);
