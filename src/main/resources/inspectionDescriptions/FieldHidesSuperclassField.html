Relata campos em uma classe derivada que possuem o mesmo nome de um campo em uma superclasse. Os campos Java não podem ser sobrescritos em classes derivadas, portanto, um campo na classe derivada ocultará o campo da superclasse.

Devido a essa nomenclatura, você pode acabar usando acidentalmente o campo da classe derivada quando a intenção era usar o campo de mesmo nome da classe base.

Sugere-se renomear o campo na classe derivada por meio de uma correção rápida.

Exemplo:

class Parent {
  Parent parent;
}
class Child extends Parent {
  Child parent;
}

Você pode configurar as seguintes opções para esta inspeção:

  1. Ignorar campos inacessíveis - Indica se esta inspeção deve relatar todos os conflitos de nome ou apenas conflitos com campos visíveis para a subclasse.
  2. Ignorar campos static que ocultam campos static - Ignora campos static que ocultam campos static na classe base.
