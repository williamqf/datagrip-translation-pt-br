Relata classes que implementam a interface Cloneable, mas não sobrescrevem o método clone().

Tais classes usam a implementação padrão de clone(), que não é public, mas sim protected, e não copia o estado mutável da classe.

Uma correção rápida pode ser usada para gerar um método clone() básico, que pode servir como base para um método clone() que funcione corretamente, conforme esperado para uma classe Cloneable.

Exemplo:


  public class Data implements Cloneable {
    private String[] names;
  }

Após aplicar a correção rápida:


  public class Data implements Cloneable {
    private String[] names;

    @Override
    public Data clone() {
      try {
        Data clone = (Data) super.clone();
        // TODO: copie o estado mutável aqui, para que o clone não possa alterar as partes internas do original 
         return clone;
      } catch (CloneNotSupportedException e) {
        throw new AssertionError();
      }
    }
  }

Use a opção Ignorar classes clonáveis devido à herança para ignorar classes que são Cloneable por herdarem de uma classe Cloneable.

Use a opção Ignorar casos em que Cloneable é necessário para chamar o método clone() da superclasse para ignorar classes que exigem a implementação de Cloneable por chamarem o método clone() da superclasse.
