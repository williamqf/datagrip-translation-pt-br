Relata recursos Java ME javax.microedition.rms.RecordStore que não são abertos antes de um bloco try e fechados em um bloco finally correspondente.

Tais recursos podem sofrer vazamento acidental se uma exceção for lançada antes de o recurso ser fechado.

Esta inspeção destina-se ao Java ME e a outros ambientes com restrição severa de recursos. Aplicar os resultados desta inspeção de forma indiscriminada pode afetar negativamente a clareza e o design do código.

Exemplo:


  void foo1() throws RecordStoreException {
      RecordStore rs = RecordStore.openRecordStore("bar", true); // aviso
  }
  void foo2() throws RecordStoreException {
      RecordStore rs = RecordStore.openRecordStore("bar", true); // sem aviso
      try {
          /* ... */
      } finally {
          rs.closeRecordStore();
      }
  }
