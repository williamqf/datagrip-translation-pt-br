Relata classes que implementam java.io.Serializable e nas quais o método readResolve() ou writeReplace() não foi declarado como protected.

Declarar os métodos readResolve() e writeReplace() como private pode forçar as subclasses a ignorá-los silenciosamente, enquanto declará-los como public permite que códigos não confiáveis os chamem.

Se a classe envolvente for declarada como final, esses métodos podem ser declarados como private.

Exemplo:


  class ClassWithSerialization implements Serializable {
    public Object writeReplace() { // Aviso: 'writeReplace()' não foi declarado como protected
        ...
    }
  }
