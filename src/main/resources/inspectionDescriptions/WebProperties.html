Relata os seguintes problemas nos descritores usados para implantar módulos Web em um servidor:
  • Referências a classes não instanciáveis
  • Referências a classes que não estendem a classe necessária
  • Referências a classes com escopo inadequado
  • Tags e valores de atributos vazios
  • Valores de tags e atributos que não correspondem ao padrão exigido (por exemplo, identificadores Java)
  • Tags que não contêm as sub-tags ou atributos obrigatórios
  • Tags que definem objetos com nomes duplicados

Exemplo:



<web-app>
  <filter-mapping>
    <filter-name>CustomFilter</filter-name> // Erro: a sub-tag servlet-name ou url-pattern deve ser definida
  </filter-mapping>
  <servlet-mapping>
    <servlet-name>CustomServletName</servlet-name>
    <url-pattern>/ baseUrl</url-pattern> // Erro: o padrão URI não deve conter espaços em branco
  </servlet-mapping>
</web-app>
