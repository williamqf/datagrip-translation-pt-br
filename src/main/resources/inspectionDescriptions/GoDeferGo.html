Reporta instruções defer e go que chamam diretamente panic() ou recover().

Tais instruções raramente são úteis e podem indicar um uso incorreto dos mecanismos panic() e recover(). Deve-se prestar atenção especial ao seguinte:

  • go panic(): uma goroutine recém-iniciada entrará em panic imediatamente.
  • defer panic(): uma função com esta instrução sempre entrará em panic ao ser encerrada.
  • go recover(): não faz nada, pois uma goroutine recém-iniciada não pode estar em estado de panic.
  • defer recover(): uma função com esta instrução interromperá um panic silenciosamente. Este uso pode ser válido, mas a maneira idiomática é verificar o valor retornado por recover(): 
    defer func() {
  if r := recover(); r != nil {
    fmt.Println("Recovered from: ", r)
  }
}()

Para mais detalhes sobre instruções Go e tratamento de panic, consulte Handling panics e Go statements na especificação da linguagem Go.
