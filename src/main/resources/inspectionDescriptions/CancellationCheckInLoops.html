Relata loops, métodos semelhantes a forEach e ContainerUtil.process() que não possuem verificações de cancelamento.

É executado apenas em métodos com a anotação com.intellij.util.concurrency.annotations.RequiresReadLock.

Exemplo:


@RequiresReadLock
fun doSomething() {
  ...
  for (item in items) {
        ProgressManager.checkCanceled() // deve estar na primeira linha
    ...
  }

  items.forEach {
    ProgressManager.checkCanceled() // deve estar na primeira linha
    ...
  }
  ...
}

No caso de loops aninhados sem nada entre eles:


@RequiresReadLock
fun doSomething() {
  ...
  for (item in items) {
        // nada entre eles
        for (inner in item.inners) {
            ProgressManager.checkCanceled() // deve estar apenas na primeira linha do loop interno
      ...
    }
  }
  ...
}

com.intellij.openapi.progress.ProgressManager.checkCanceled() deve ser usado em contextos de bloqueio, enquanto com.intellij.openapi.progress.CoroutinesKt.checkCancelled() deve ser usado em contextos de suspensão.

Para mais detalhes, consulte Cancelamento de processos em segundo plano na documentação do SDK do Plugin da Plataforma IntelliJ.
