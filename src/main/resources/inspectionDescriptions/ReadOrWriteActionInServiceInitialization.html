Inspeciona ações de leitura e gravação executadas no escopo de inicialização do serviço:
  • Construtores e blocos de inicialização do serviço (incluindo estáticos)
  • Blocos de inicialização do objeto companheiro do serviço (Kotlin)
  • Métodos de serviço usados em blocos de inicialização e inicialização de campos
  • Métodos de ciclo de vida de inicialização de PersistentStateComponent:
    • loadState
    • noStateLoaded
    • initializeComponent

Executar operações de leitura ou gravação durante a inicialização do serviço pode causar deadlocks.

Exemplo:

Kotlin:


@Service
internal class MyService {
  private val myData = initMyData();

  constructor() {
    val data = runReadAction { // Erro: execução de uma ação de leitura no construtor
      // Leitura de dados
    }
  }

  private fun initMyData(): Data {
    return runWriteAction { // Erro: invocado durante a inicialização da propriedade myData
      // Gravação de dados
    }
  }

  companion object {
    lateinit var companionData: String
    init {
      companionData = runReadAction { // Erro: execução de uma ação de leitura no bloco init
        // Leitura de dados
      }
    }
  }
}

Java:


@Service
class MyService {
  private static final Data ourData1 = ReadAction.compute(() -> {
    // Leitura de dados
  });
  private static final Data ourData2;

  static {
    ourData2 = ReadAction.compute(() -> { // Erro: presença de ação de leitura no bloco de inicialização estático
      // Leitura de dados
    });
  }

  private final Data myData2 = initMyData();

  MyService() {
    Data data = WriteAction.compute(() -> { // Erro: execução de uma ação de leitura no construtor
      // Gravação de dados
    });
  }

  private Data initMyData() {
    return ReadAction.compute(() -> { // Erro: invocado durante a inicialização da propriedade myData
      // Leitura de dados
    });
  }
}
