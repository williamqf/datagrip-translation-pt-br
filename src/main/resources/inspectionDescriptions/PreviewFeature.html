Relata o uso de APIs de recursos de visualização (preview), ou seja, módulos, pacotes, classes, interfaces, métodos, construtores, campos ou constantes de enumeração nos espaços de nomes java.* ou javax.* anotados com @PreviewFeature.

Um recurso de visualização (preview feature) é uma nova funcionalidade da linguagem Java, da Java Virtual Machine ou da API Java SE que está totalmente especificada e implementada, mas não é permanente. O conceito de recursos de visualização é definido na JEP 12.

Se um trecho de código depender de uma API de visualização, ele poderá deixar de ser compilado em versões futuras do JDK caso a funcionalidade seja alterada ou removida.

Esta inspeção relata apenas se o nível de linguagem do projeto ou módulo estiver definido como Preview.
