Relata chamadas a notify() ou notifyAll() em objetos java.util.concurrent.locks.Condition.

Isso provavelmente é um erro de programação, no qual alguma variante dos métodos signal() ou signalAll() deveria ter sido usada; caso contrário, uma IllegalMonitorStateException poderá ocorrer.

Exemplo:


  class C {
    final Lock l = new ReentrantLock();
    final Condition c = l.newCondition();

    void release() {
      l.lock();
      try {
        c.notifyAll(); // 'signalAll()' provavelmente era o pretendido aqui
      } finally {
        l.unlock();
      }
    }
  }
