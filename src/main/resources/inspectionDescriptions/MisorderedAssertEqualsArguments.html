Relata chamadas de assertEquals() com a ordem incorreta dos argumentos expected (esperado) e actual (atual).

Para o JUnit 3, 4 e 5, a ordem correta é (expected, actual). Para o TestNG, a ordem correta é (actual, expected).

Tais chamadas não apresentam problemas para asserções que passam, mas podem resultar em relatórios de erro confusos em caso de falha. Use a correção rápida para inverter a ordem dos argumentos.

Exemplo (JUnit):


  assertEquals(actual, expected)

Após a aplicação da correção rápida:


  assertEquals(expected, actual)
