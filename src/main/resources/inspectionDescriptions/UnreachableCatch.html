Reporta seções catch que nunca são executadas, apesar de serem permitidas pela Especificação da Linguagem Java.

Embora seções catch inalcançáveis geralmente não sejam permitidas pelo compilador Java e sejam reportadas como erros de compilação, em alguns casos, a análise exigida pela Especificação da Linguagem Java é incompleta. Esta inspeção fornece uma análise aprimorada e reporta seções catch inalcançáveis que o compilador não reporta. Essas seções são redundantes e podem ser removidas com segurança.

Exemplo:


  void method() {
    try { 
      throw new FileNotFoundException(); 
    }
    catch (FileNotFoundException e) { 
      
    }
    catch (IOException e) {
      // A especificação permite este catch
      // mas ele nunca é executado
    }
  }

Uma correção rápida é fornecida para remover seções catch redundantes:


  void method() {
    try { 
      throw new FileNotFoundException(); 
    }
    catch (FileNotFoundException e) { 
      
    }
  }
