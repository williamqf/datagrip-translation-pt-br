Relata métodos clone() que não declaram throws CloneNotSupportedException.

Se throws CloneNotSupportedException não for declarado, as subclasses do método não poderão proibir a clonagem da maneira padrão. Esta inspeção não relata métodos clone() declarados como final e métodos clone() dentro de classes final.

Configuração da inspeção:

Use a opção Avisar apenas em métodos clone 'protected' para indicar que esta inspeção deve gerar avisos apenas para métodos protected clone(). O livro Effective Java (2ª e 3ª edições) recomenda omitir a declaração CloneNotSupportedException em métodos public, pois métodos que não lançam exceções verificadas (checked exceptions) são mais fáceis de usar.

Exemplo:


  public class Example implements Cloneable {
    // O método não declara 'throws CloneNotSupportedException'
    protected Object clone() {
        try {
            return super.clone();
        } catch (CloneNotSupportedException e) {
            return null;
        }
    }
  }
