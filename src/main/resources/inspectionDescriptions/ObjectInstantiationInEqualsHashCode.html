Relata a construção de novos objetos (temporários) dentro dos métodos equals(), hashCode(), compareTo() e Comparator.compare().

Além de chamadas de construtor, novos objetos podem ser criados por autoboxing ou pela criação de iteradores dentro de uma instrução foreach. Isso pode levar a problemas de desempenho; por exemplo, ao adicionar objetos a um Set ou Map, esses métodos serão chamados com frequência.

Esta inspeção não relatará quando um objeto for criado em uma instrução throw ou assert.

Exemplo:


  class Person {
    private String name;
    private int age;

    public boolean equals(Object o) {
      return Arrays.equals(new Object[] {name, age}, new Object[] {((Foo)o).name, ((Foo)o).age});
    }

    public int hashCode() {
      return (name + age).hashCode();
    }
  }

Neste exemplo, dois arrays adicionais são criados dentro de equals(), o uso do campo age requer boxing e name + age cria implicitamente uma nova string.
