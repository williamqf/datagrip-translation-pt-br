Relata blocos catch que estão vazios ou podem ignorar exceções.

Embora ocasionalmente intencionais, blocos catch vazios podem complicar a depuração. Além disso, ignorar o parâmetro catch pode ser um erro.

Esta inspeção não relata nenhum parâmetro catch nomeado como ignore, ignored ou _.

Você pode usar uma correção rápida para alterar o nome da exceção para _.

Exemplo:


  try {
    throwingMethod()
  } catch (ex: IOException) {

  }

Após aplicar a correção rápida:


  try {
    throwingMethod()
  } catch (_: IOException) {

  }
Use a opção Não avisar se o bloco 'catch' contiver comentários para ignorar blocos catch que contenham comentários.
