Relata expressões de alias que podem se sobrepor aos nomes das colunas na tabela e podem ser redundantes.

Exemplo (PostgreSQL):

CREATE TABLE foo(a INT, b INT);

SELECT * FROM foo foo(a, b);
SELECT * FROM foo foo(a);
SELECT * FROM foo foo(x);
SELECT * FROM foo foo(x, y);

Os dois primeiros aliases usam os mesmos nomes das colunas na tabela foo. Estes são considerados redundantes porque as colunas possuem o mesmo nome.
