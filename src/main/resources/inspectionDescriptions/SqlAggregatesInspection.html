Relata o uso inválido de funções de agregação SQL.

As seguintes situações são consideradas:

  • Colunas usadas nas cláusulas HAVING e ORDER BY, mas ausentes na cláusula GROUP BY.

    CREATE TABLE foo(id INT PRIMARY KEY, a INT, b INT);
SELECT a, MAX(b) FROM foo GROUP BY a HAVING b > 0;
SELECT * FROM foo GROUP BY a ORDER BY b;

    Esta regra não se aplica ao agrupar pela chave primária.

    SELECT * FROM foo GROUP BY id ORDER BY b;

  • Funções de agregação em um contexto incorreto. Geralmente, você pode usar funções de agregação nos seguintes contextos: lista de expressões no SELECT; nas seções HAVING e ORDER BY; e outros casos específicos de dialetos. As consultas a seguir exibirão um erro.

    SELECT a FROM foo WHERE MAX(b) > 0;
SELECT a FROM foo GROUP BY MAX(a);

  • Chamadas aninhadas de funções de agregação.

    SELECT MAX(SUM(a)) FROM foo GROUP BY a;

    Esta regra não se aplica a funções analíticas. A consulta a seguir é válida e correta.

    SELECT MAX(SUM(a) OVER ()) FROM foo;

  • Uso de HAVING sem funções de agregação. Nesse caso, considere reescrever seu código usando uma seção WHERE.

    SELECT a, MAX(b) FROM foo GROUP BY a HAVING a > 0;
