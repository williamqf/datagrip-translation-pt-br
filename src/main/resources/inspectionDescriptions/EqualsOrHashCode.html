Relata classes que sobrescrevem equals(), mas não sobrescrevem hashCode() (ou vice-versa). Também relata declarações de objeto que sobrescrevem equals() ou hashCode().

Isso pode causar comportamentos indesejados ao adicionar a classe a uma Collection.

Exemplo:


  class C1 {
      override fun equals(other: Any?) = true
  }

  class C2 {
      override fun hashCode() = 0
  }

  object O1 {
      override fun equals(other: Any?) = true
  }

  object O2 {
      override fun hashCode() = 0
  }

A correção rápida sobrescreve os métodos equals() ou hashCode() para classes e remove esses métodos para objetos:


   class C1 {
       override fun equals(other: Any?) = true
       override fun hashCode(): Int {
           return javaClass.hashCode()
       }
   }

   class C2 {
       override fun hashCode() = 0
       override fun equals(other: Any?): Boolean {
           if (this === other) return true
           if (javaClass != other?.javaClass) return false
           return true
       }
   }

   object O1 {
   }

   object O2 {
   }
