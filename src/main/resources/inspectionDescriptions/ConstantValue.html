Relata expressões e condições que sempre produzem o mesmo resultado, como true, false, null ou 0. Tais expressões podem ser substituídas pelo valor constante correspondente ou podem indicar um bug no código.

Exemplo:

  // Sempre true
  // Motivo: || foi usado em vez de &&
  if (x > 0 || x < 10) {}

  System.out.println(str.trim());
  // Sempre false
  // Motivo: a variável foi desreferenciada antes da verificação de null
  if (str == null) {}

O comportamento da inspeção pode ser controlado por anotações, como anotações de nulidade, a anotação @Contract, a anotação @Range, etc.

Configuração da inspeção:

  • Use a opção Não relatar asserções com condições provadas estaticamente como sempre true para evitar relatar asserções que são provadas estaticamente como sempre verdadeiras. Isso inclui condições como if (alwaysFalseCondition) throw new IllegalArgumentException();.
  • Use a opção Ignorar instruções assert para controlar como a inspeção lida com instruções assert. Por padrão, presume-se que as asserções sejam executadas (modo -ea). Quando esta opção está ativada, as asserções são ignoradas (modo -da).
  • Use a opção Avisar quando uma constante for armazenada em uma variável para exibir avisos quando uma variável cujo valor é conhecido por ser constante for usada.

Antes da versão 2022.3 do IntelliJ IDEA, esta inspeção fazia parte da inspeção Condições constantes e exceções. Esta inspeção foi dividida em duas: Valores constantes e Nulidade e problemas de fluxo de dados.
