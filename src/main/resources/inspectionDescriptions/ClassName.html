Relata nomes de classes que não seguem as convenções de nomenclatura recomendadas.

Uma nomenclatura consistente torna o código mais fácil de ler e entender. De acordo com o guia de estilo oficial do Kotlin, os nomes de classes devem começar com uma letra maiúscula e usar CamelCase.

Outras regras de nomenclatura podem ser introduzidas alterando a expressão regular do "Padrão".

Exemplo:


  class user(val name: String)

A correção rápida renomeará a classe de acordo com as convenções de nomenclatura do Kotlin:


  class User(val name: String)
