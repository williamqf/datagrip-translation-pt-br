Relata campos que podem ser definidos como final com segurança. Todos os campos final devem possuir um valor que não muda, tornando o código mais fácil de raciocinar sobre.

Para evitar um custo de análise excessivo, esta inspeção relata apenas se o campo possui o modificador private ou se é definido em uma classe local ou anônima. Um campo pode ser final se:

  • O campo for static e for inicializado uma única vez em sua declaração ou em um inicializador static.
  • O campo não for static e for inicializado uma única vez em sua declaração, em um inicializador de instância ou em cada construtor
e o campo não for modificado em nenhum outro lugar.

Exemplo:


  public class Person {
    private String name; // pode ser final

    Person(String name) {
      this.name = name;
    }

    public String getName() {
      return name;
    }
  }

Após a aplicação da correção rápida:


  public class Person {
    private final String name;

    Person(String name) {
      this.name = name;
    }

    public String getName() {
      return name;
    }
  }

Use o botão "Anotações" para modificar a lista de anotações que assumem a escrita implícita de campos.
