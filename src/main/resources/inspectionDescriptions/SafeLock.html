Relata recursos java.util.concurrent.locks.Lock que não são adquiridos antes de um bloco try ou não são desbloqueados em um bloco finally correspondente. Se uma exceção for lançada antes que o recurso seja liberado, tais recursos podem sofrer vazamento acidentalmente.

Exemplo:


  lock.lock(); // Relatado devido à ausência do bloco 'finally'
  try {
    doSmthWithLock();
  } catch (IOException e) {
    throw new UncheckedIOException(e);
  }
  lock.unlock();
