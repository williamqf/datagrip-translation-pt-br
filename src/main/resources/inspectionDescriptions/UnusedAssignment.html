Relata atribuições que não são utilizadas após a atribuição. Se não for utilizada, é recomendável remover a atribuição para encurtar o código e evitar atribuições redundantes.

Os seguintes casos serão relatados:

  • Variáveis que nunca são lidas após a atribuição
  • Variáveis que são sempre sobrescritas por um novo valor antes de serem lidas
  • Inicializadores de variáveis redundantes (por um dos dois motivos acima)

Configuração da inspeção:

Use a opção Relatar inicializadores redundantes para relatar inicializadores redundantes: 


  int getI() {
    int i = 0; // inicialização redundante
    i = 2;
    return i;
  }

Use a opção Relatar expressões de prefixo que podem ser substituídas por expressões binárias para relatar situações em que a expressão ++i pode ser substituída por i + 1: 


  int preInc(int value) {
    int res = value;
    return ++res;
  }

Use a opção Relatar expressões de sufixo onde o valor alterado não é usado para relatar casos de i++ onde o valor de i não é usado posteriormente: 


  int postInc(int value) {
    int res = value;
    return res++;
  }

Use a opção Relatar variáveis de padrão cujos valores nunca são usados para relatar casos em que o valor de uma variável de padrão é sobrescrito antes de ser lido: 


  if (object instanceof String s) {
      s = "hello";
      System.out.println(s);
  }

Use a opção Relatar parâmetros de iteração cujos valores nunca são usados para relatar situações em que o valor do parâmetro de iteração de um comando for aprimorado é sobrescrito antes de ser lido: 


  for (String arg : args) {
      arg = "test";
      System.out.println(arg);
  }
