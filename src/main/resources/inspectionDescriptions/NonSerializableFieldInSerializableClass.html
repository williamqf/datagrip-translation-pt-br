Relata campos não serializáveis em classes que implementam java.io.Serializable. Tais campos causarão uma exceção em tempo de execução se o objeto for serializado.

Campos declarados como transient ou static não são relatados, nem campos de classes que definem o método writeObject.

Esta inspeção assume que campos dos tipos java.util.Collection e java.util.Map são Serializable, a menos que sejam declarados como um tipo não Serializable.

Exemplo:


  class NonSerializableClass {}

  public class SerializableClass implements Serializable {
      NonSerializableClass clazz; // Aviso: campo não serializável 'clazz' em classe Serializable
      static NonSerializableClass staticClazz; // Sem aviso
  }

Configure a inspeção usando as seguintes opções:

  • Listar as classes cujos sucessores não devem ser relatados por esta inspeção. Isso se aplica a classes que herdam Serializable de uma superclasse, mas não se destinam à serialização.
  • Listar as anotações que farão com que a inspeção ignore os campos anotados.
  • Se deve ignorar campos inicializados com classes anônimas.
