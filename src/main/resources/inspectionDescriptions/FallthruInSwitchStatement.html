Relata 'fallthrough' em instruções switch.

O fallthrough ocorre quando uma série de instruções executáveis após um rótulo case não garante a transferência de controle antes do próximo rótulo case. Por exemplo, isso pode acontecer se uma instrução break estiver ausente em um bloco. Nesses casos, o controle passará para as instruções após o rótulo switch seguinte, mesmo que a expressão switch não seja igual ao valor do rótulo onde ocorreu o fallthrough. Embora essa construção seja ocasionalmente intencional, ela é confusa e geralmente é causada por um erro de digitação.

Esta inspeção ignorará qualquer fallthrough comentado com um texto que corresponda ao padrão de expressão regular (?i)falls?\s*thro?u.

É possível corrigir o problema adicionando um break ao bloco que pode passar para o próximo.

Exemplo:


    switch(x) {
      case (4):
          if (condition) {
              System.out.println("3");
              // nenhum break aqui
          } else {
              break;
          }
      case (6):
          System.out.println("4");
    }

Após a aplicação da correção rápida:


    switch(x) {
        case (4):
            if (condition) {
                System.out.println("3");
            } else {
                break;
            }
            break;
        case (6):
            System.out.println("4");
    }
