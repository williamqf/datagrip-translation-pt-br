Relata chamadas de membros super com qualificação de supertipo redundante.

O código em classes derivadas pode usar a palavra-chave super para chamar as implementações de função e acessadores de propriedade de sua superclasse. Para especificar o supertipo do qual a implementação herdada é obtida, o super pode ser qualificado pelo nome do supertipo entre colchetes angulares, por exemplo, super<Base>. Às vezes, essa qualificação é redundante e pode ser omitida. Use a correção rápida "Remover qualificação explícita de supertipo" para limpar o código.

Exemplo:


  open class B {
      open fun foo(){}
  }

  class A : B() {
      override fun foo() {
         super<B>.foo() // <== redundante, pois 'B' é o único supertipo
      }
  }

  interface I {
      fun foo() {}
  }

  class C : B(), I {
      override fun foo() {
          super<B>.foo() // <== aqui, o qualificador <B> é necessário para distinguir 'B.foo()' de 'I.foo()'
      }
  }

Após aplicar a correção rápida:


  open class B {
      open fun foo(){}
  }

  class A : B() {
      override fun foo() {
         super.foo() // <== atualizado
      }
  }

  interface I {
      fun foo() {}
  }

  class C : B(), I {
      override fun foo() {
          super<B>.foo()
      }
  }
