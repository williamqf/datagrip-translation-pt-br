Relata chamadas em receptores não nulos que só fazem sentido em receptores anuláveis.

Várias funções na biblioteca padrão, como orEmpty() ou isNullOrEmpty, só fazem sentido quando chamadas em um receptor de tipo anulável. Em outros casos, elas podem ser omitidas ou simplificadas, pois o resultado será o mesmo.

As correções rápidas Remover chamada redundante e Alterar chamada para ... podem ser usadas para corrigir o código automaticamente.

Exemplo:


  fun test(s: String) {
      val x = s.orEmpty() // A correção rápida simplificará para 's'
      val y = s.isNullOrEmpty() // A correção rápida simplificará para 's.isEmpty()'
  }
