Relata blocos finally vazios.

Blocos finally vazios geralmente indicam um erro de codificação. Eles podem permanecer após a refatoração de código e podem ser removidos com segurança.

Esta inspeção não relata blocos finally vazios encontrados em arquivos JSP.

Exemplo:


  try {
    Files.readString(Paths.get("in.txt"));
  } catch (IOException e) {
    throw new RuntimeException(e);
  } finally {

  }

Após a aplicação da correção rápida: 


  try {
    Files.readString(Paths.get("in.txt"));
  } catch (IOException e) {
    throw new RuntimeException(e);
  }
