Esta inspeção analisa chamadas de API do Android anotadas com várias anotações de suporte (como RequiresPermission ou UiThread) e sinaliza qualquer chamada que não use a API corretamente conforme especificado pelas anotações.

Exemplos de erros sinalizados por esta inspeção:
  • Passar um tipo incorreto de inteiro de recurso (como R.string) para uma API que requer um tipo diferente (como R.dimen).
  • Esquecer de chamar o método sobrescrito (via super) em um método que o exige.
  • Chamar um método que exige uma permissão sem declarar essa permissão no manifesto.
  • Passar uma referência de cor de recurso para um método que exige um valor inteiro RGB.
...e muito mais.

Para mais detalhes, consulte a documentação em http://developer.android.com/tools/debugging/annotations.html
