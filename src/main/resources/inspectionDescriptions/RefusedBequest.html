Relata métodos que sobrescrevem um supermétodo sem chamá-lo. Isso também é conhecido como Refused Bequest (herança recusada). Tais métodos podem indicar uma falha de abstração, resultando em bugs difíceis de rastrear.

A inspeção não relata métodos sobrescritos de java.lang.Object (com exceção de clone()). Por convenção, o método clone() deve chamar seu supermétodo, que retornará um objeto do tipo correto.

Exemplo 1:


  class A {
    @Override
    public Object clone() {
      // Não chama 'super.clone()'
      return new A();
    }
  }

Exemplo 2:


  interface I {
    default void foo() {}
  }

  class A implements I {
    // Avisa sobre o método quando
    // "Ignorar supermétodos 'default'" está desativado
    @Override
    public void foo(){}
  }

Configuração da inspeção:

  • Use a opção Relatar apenas se o supermétodo estiver anotado por para ignorar supermétodos marcados com as anotações da lista fornecida. Você pode adicionar anotações manualmente a esta lista.
  • Use a opção Ignorar supermétodos vazios para ignorar supermétodos vazios ou que apenas lançam uma exceção.
  • Use a opção Ignorar supermétodos 'default' para ignorar supermétodos default em interfaces.
