Possible report of zip operations in Reactive Stream chain that may return Mono with void value.

Reactive Streams zip transformation methods with Mono Void parameters have no effect. The quick fix will offer to replace zip(), zipWhen(), zipWith() with when that returns Mono with Void value.

Example:

Mono.zip(Mono.just(1), Mono.fromRunnable(() -> System.out.println("value")))
                .map(v -> "New value")
                .doFinally(System.out::println);

After applying the quick fix:

Mono.when(Mono.just(1), Mono.fromRunnable(() -> System.out.println("value")))
                .then(Mono.fromCallable(() -> "New value"))
                .doFinally(System.out::println);
