Relata chamadas a wait(), notify() e notifyAll() que não são feitas dentro de uma instrução synchronized ou método synchronized correspondente.

Chamar esses métodos em um objeto sem possuir o bloqueio nele resultará em uma IllegalMonitorStateException. Tal construção não é necessariamente um erro, pois o bloqueio necessário pode ter sido adquirido antes que o método que a contém seja chamado, mas mesmo assim, deve ser examinada com atenção.

Exemplo:


  class Sync {
    private final Object lock = new Object();

    void test() throws InterruptedException {
      synchronized (this) {
        lock.wait(); // 'lock.wait()' não está sincronizado em 'lock'
      }
    }
  }
