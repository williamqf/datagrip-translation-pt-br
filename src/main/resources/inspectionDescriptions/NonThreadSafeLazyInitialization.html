Relata variáveis static inicializadas de forma tardia (lazy) que não são thread-safe.

A inicialização tardia de variáveis static deve ser realizada com construtos de sincronização apropriados para evitar que threads diferentes executem inicializações conflitantes.

Se aplicável, é sugerida uma correção rápida (Quick Fix) que introduz o idioma de classe holder de inicialização tardia. Esse idioma aproveita o fato de que a JVM garante que uma classe não será inicializada até que seja utilizada.

Exemplo:


  class X {
    private static List<String> list;

    public List<String> getList() {
      if (list == null) {
        list = List.of("one", "two", "tree");
      }
      return list;
    }
  }

Após a aplicação da correção rápida:


  class X {
    private static final class ListHolder {
      static final List<String> list = List.of("one", "two", "tree");
    }

    public List<String> getList() {
      return ListHolder.list;
    }
  }
