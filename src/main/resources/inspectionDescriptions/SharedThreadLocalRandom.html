Relata instâncias de java.util.concurrent.ThreadLocalRandom que podem ser compartilhadas entre threads.

ThreadLocalRandom não deve ser compartilhado entre threads, pois não é thread-safe. Esta inspeção relata instâncias atribuídas a campos, instâncias usadas como argumentos de método ou instâncias atribuídas a variáveis locais e usadas em classes anônimas ou aninhadas, pois podem ser compartilhadas entre threads.

O uso de ThreadLocalRandom normalmente deve ser semelhante a ThreadLocalRandom.current().nextInt(...) (ou nextDouble(...), etc.). Quando todo o uso assume essa forma, a instância de ThreadLocalRandom não será usada incorretamente por várias threads.

Exemplo:


    class Main {
      void printRandomNumbersAsync() {
        ThreadLocalRandom random = ThreadLocalRandom.current();
        CompletableFuture.supplyAsync(() -> generateNumbers(random))
          .thenAccept(numbers -> System.out.println(Arrays.toString(numbers)));
      }

      private int[] generateNumbers(Random random) {
        return random.ints(1000, 0, 100).toArray();
      }
    }
