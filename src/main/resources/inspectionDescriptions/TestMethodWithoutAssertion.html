Relata métodos de teste que não contêm nenhuma asserção. Tais métodos podem indicar casos de teste incompletos ou frágeis.

Exemplo:


  public class ExtensiveTest {

    @Test
    public void testAlive() {
      System.out.println("nothing");
    }
  }

Configuração da inspeção:

  • Use a tabela para especificar combinações de nomes de classes totalmente qualificados e expressões regulares de nomes de métodos que devem ser considerados como asserções. O nome da classe também corresponde a subclasses.
  • Use a opção Considerar a palavra-chave 'assert' como uma asserção para especificar se as instruções Java assert que usam a palavra-chave assert devem ser tratadas como asserções.
  • Use a opção Ignorar métodos de teste que declaram exceções para ignorar métodos de teste que declaram exceções. Isso pode ser útil quando seus testes lançam uma exceção em caso de falha e, portanto, não exigem nenhuma asserção.
