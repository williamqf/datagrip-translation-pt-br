Relata argumentos que exigem um cast explícito para serem resolvidos para uma declaração correta em uma chamada de função sobrecarregada. O aviso atual do compilador (disponível a partir do Kotlin 1.6.20) se tornará um erro no Kotlin 1.8.

Tipos de progressão e intervalo (kotlin.ranges) passarão a implementar a interface Collection no Kotlin 1.9 e versões posteriores. Esta atualização causará mudanças na resolução de funções sobrecarregadas. Por exemplo, no exemplo abaixo, a chamada test(1..5) será resolvida como test(t: Any) no Kotlin 1.8 e versões anteriores, e como test(t: Collection<*>) no Kotlin 1.9 e versões posteriores. 


  fun test(t: Any) { }
  fun test(t: Collection<*>) { }
  fun invoke() {
    test(1..5) // No 1.9, IntRange se tornará Collection
  }

O quick fix fornecido captura o comportamento específico do compilador para a versão 1.8 e anteriores:


  fun test(t: Any) { }
  fun test(t: Collection<*>) { }

  fun invoke() {
    test(1..5) // Resolvido como 'test(t: T)' antes do Kotlin 1.9
  }

Após aplicar o quick fix:


  fun test(t: Any) { }
  fun test(t: Collection<*>) { }

  fun invoke() {
    test((1..5) as Iterable<Int>) // Resolvido como 'test(t: T)' no Kotlin 1.9
  }

A inspeção está disponível a partir do nível de linguagem Kotlin 1.6.
