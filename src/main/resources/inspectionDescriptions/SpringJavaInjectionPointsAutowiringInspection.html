Relata problemas de autowiring em pontos de injeção de Spring Beans @Component, @Service e outros.

  • Mais de um bean de tipo 'concrete'
  • Nenhum bean de tipo 'concrete'
  • Nenhum bean com qualificador
  • Uso incorreto de @Autowired em construtores de Spring Beans
  • Campos/métodos injetados ou autowired em classes que não são Spring Beans válidos

Exemplo:


public interface FooInterface {...}
  @Component public class FooBean implements FooInterface {...}
  @Component public class OtherBean implements FooInterface {...}

@Component
public class MyComponent {
	@Autowired
	FooInterface foo;  // Não foi possível realizar o autowire. Existe mais de um bean do tipo 'FooInterface'.
                     // Beans: fooBean(FooBean.java), otherBean(OtherBean.java)"
}

Exemplo:


@Component
public class MyComponent {
	@Autowired
	public MyComponent(BarBean bean) {...} // Relata "Apenas um construtor @Autowired é permitido"

	@Autowired
	public MyComponent(FooBean bean) {...} // Relata "Apenas um construtor @Autowired é permitido"
}

@Component
public class MyFactory {  // Relata "Nenhum construtor @Autowired correspondente encontrado"
	public MyFactory(String str) {...}
	public MyFactory(int count) {...}
}

Exemplo:


public class FooBeanClass {
  @Autowired   // Relata 'Membros autowired devem ser definidos em um Spring Bean válido: @Component, @Service, etc.'
  ServiceBean bean;
  }
