Relata a autoconfiguração (autowiring) incorreta de Spring Beans configurados em um contexto de aplicação XML.

Exemplo:


public interface FooInterface {...}
@Component public class FooBean implements FooInterface {...}
@Component public class OtherBean implements FooInterface {...}

@Component
public class MyComponent {
	 public void setFooInterface(FooInterface foo) {...}
}


<beans>
	<bean class="beans.OtherBean" id="otherBean"/>
	<bean class="beans.FooBean" id="fooBean"/>

	<bean autowire="byType" class="beans.MyComponent" id="foo"/> <!-- Relata "Não foi possível realizar o autowiring. Existe mais de um bean do tipo 'FooInterface'." -->
                                                                   <!--          Beans: otherBean,fooBean. Propriedades: 'fooInterface' " -->
</beans>
