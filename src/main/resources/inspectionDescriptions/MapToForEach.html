Reporta chamadas map cujos resultados são ignorados e que podem ser substituídas por forEach.

Usar map dessa forma é enganoso, pois o map é destinado a transformar coleções e retornar um resultado. Quando seu valor de retorno não é utilizado, a chamada cria alocações desnecessárias e prejudica a legibilidade do código. Nesses casos, deve-se usar o forEach em seu lugar, pois ele expressa claramente uma iteração com efeitos colaterais.

A correção rápida substituirá a chamada map por uma chamada forEach.

Exemplo:


fun foo(list: List<String>) {
 list.map { print(it) } // Reportado

 val x = list.map { it.length } // Não reportado: o resultado é utilizado
}

Após aplicar a correção rápida:


fun foo(list: List<String>) {
 list.forEach { print(it) }

 val x = list.map { it.length } // Não reportado: o resultado é utilizado
}
