Relata chamadas de coleções não modificáveis de java.util.Collections que podem ser convertidas em métodos de fábrica de coleções mais recentes. Por exemplo, estas podem ser substituídas por List.of() ou Set.of() introduzidos no Java 9, ou List.copyOf() introduzido no Java 10.

Observe que, em comparação com os métodos de java.util.Collections, os métodos de fábrica de coleções do Java 9:

  • Não aceitam valores null.
  • Exigem elementos de coleção e chaves de mapa exclusivos.
  • Não aceitam argumentos null para métodos de consulta, como List.contains() ou Map.get(), na coleção retornada.

Quando essas condições são violadas, uma exceção é lançada. Isso pode alterar a semântica do código após a migração.

Exemplo: 


  List<Integer> even = Collections.unmodifiableList(
    Arrays.asList(2, 4, 6, 8, 10, 2));
  List<Integer> evenCopy = Collections.unmodifiableList(
    new ArrayList<>(list1));

Após a aplicação da correção rápida:


  List<Integer> even = List.of(2, 4, 6, 8, 10, 2);
  List<Integer> evenCopy = List.copyOf(list);

Use a opção Não avisar quando o conteúdo não for uma constante para relatar apenas se os argumentos fornecidos forem constantes de tempo de compilação. Isso reduz a chance de mudanças de comportamento, pois nem sempre é possível verificar estaticamente se os elementos originais são exclusivos e não null.

Use a opção Sugerir 'Map.ofEntries' para sugerir a substituição de mapas não modificáveis com mais de 10 entradas por Map.ofEntries().
