Relata o uso de break ou continue dentro de lambdas de funções semelhantes a loops.

As palavras-chave break e continue sempre se aplicam a loops reais (for, while, do-while). break e continue nunca se aplicam a funções; por exemplo, break e continue não se aplicam a forEach, filter ou map.

O uso de break ou continue dentro de funções semelhantes a loops (como forEach) pode causar confusão. Esta inspeção sugere a adição de rótulos para esclarecer a qual instrução o break ou continue se aplica.

Como o Kotlin não possui o conceito de funções semelhantes a loops, esta inspeção utiliza uma heurística. Ela assume que funções que não possuem um dos contratos callsInPlace(EXACTLY_ONCE) ou callsInPlace(AT_LEAST_ONCE) são funções semelhantes a loops.

Exemplo:


    for (file in files) {
        file.readLines().forEach { line ->
            if (line == commentMarkerLine) continue
            println(line)
        }
    }

A correção rápida adiciona rótulos de esclarecimento:


    loop@ for (file in files) {
        file.readLines().forEach { line ->
            if (line == commentMarkerLine) continue@loop
            println(line)
        }
    }
