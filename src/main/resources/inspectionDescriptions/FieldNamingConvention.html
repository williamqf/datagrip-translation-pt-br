Relata campos cujos nomes são muito curtos, muito longos ou não seguem o padrão de expressão regular especificado.

Exemplo: se a inspeção estiver ativada para constantes e o comprimento mínimo especificado para o nome do campo for 5 (padrão), o seguinte campo gerará um aviso, pois o comprimento do nome é 3, que é menor que 5: public static final int MAX = 42;.

A correção rápida para renomear esses campos está disponível apenas no editor.

Configuração da inspeção:

Use a lista na seção Opções para especificar quais campos devem ser verificados. Desmarque a caixa de seleção dos campos para os quais deseja ignorar a verificação.

Para cada tipo de campo, use os campos de entrada fornecidos para especificar o comprimento mínimo, o comprimento máximo e a expressão regular para o nome do campo. Especifique 0 nos campos de comprimento para ignorar a verificação correspondente.

As expressões regulares devem ser especificadas no formato padrão java.util.regex.
