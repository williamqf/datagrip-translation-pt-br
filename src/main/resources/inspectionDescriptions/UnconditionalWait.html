Relata chamadas de wait() que são realizadas incondicionalmente dentro de um contexto sincronizado.

O método wait() é normalmente usado para bloquear uma thread até que uma determinada condição seja verdadeira. Se wait() for chamado incondicionalmente, isso geralmente indica que a condição foi verificada antes da aquisição do bloqueio (lock). Nesses casos, pode ocorrer uma condição de corrida (data race) se a condição se tornar verdadeira entre o momento da verificação e a aquisição do bloqueio.

Embora a construção detectada por esta inspeção não esteja necessariamente incorreta, certamente vale a pena examiná-la.

Exemplo:


  class Bar {
      void foo() throws InterruptedException {
          synchronized (this) {
              wait(); // Alerta
          }
      }
  }
