Relata variáveis locais que possuem o mesmo nome de um campo na classe circundante. Devido a essa nomenclatura, você pode acabar usando a variável erroneamente quando a intenção era usar o campo de mesmo nome.

Uma correção rápida para renomear a variável é sugerida.

Exemplo:


  public class Foo {
    public Object foo;

    void bar() {
      Object o = new Object() {
        void baz() {
          Object foo; // a variável local 'foo' oculta o campo na classe 'Foo'
        }
      };
    }
  }

Você pode configurar as seguintes opções para esta inspeção:

  1. Ignorar campos inacessíveis - Ignora variáveis locais com o mesmo nome de campos de superclasse que não são visíveis (por exemplo, porque os campos são private).
  2. Ignorar variáveis locais em contexto estático que ocultam campos não estáticos - Por exemplo, quando a variável local está dentro de um método estático ou dentro de um método em uma classe interna estática.
