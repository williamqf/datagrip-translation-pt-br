Relata chamadas de método que usam java.nio.file.Files para ler ou gravar uma String como bytes. Essas chamadas podem ser substituídas por chamadas aos métodos Files.readString() ou Files.writeString(), introduzidos no Java 11.

Exemplo:


  String s = "example";
  Files.write(Paths.get("out.txt"), s.getBytes(StandardCharsets.UTF_8), StandardOpenOption.WRITE);
  s = new String(Files.readAllBytes(Paths.get("in.txt")), StandardCharsets.ISO_8859_1);

Após a aplicação da correção rápida:


  String s = "example";
  Files.writeString(Paths.get("out.txt"), s, StandardOpenOption.WRITE);
  s = Files.readString(Paths.get("in.txt"), StandardCharsets.ISO_8859_1);

Observe que o comportamento de readString() difere do comportamento de new String(bytes, charset) ao lidar com caracteres inválidos (não mapeáveis). Nesses casos, o método readString() lança uma exceção, enquanto o método new String(bytes, charset) substitui silenciosamente caracteres inválidos por um caractere de substituição. Se a substituição silenciosa for desejada, recomenda-se suprimir o aviso da inspeção.

Novidade na versão 2018.3
