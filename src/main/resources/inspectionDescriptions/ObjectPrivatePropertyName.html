Relata propriedades que não seguem as convenções de nomenclatura.

As seguintes propriedades são relatadas:

  • Propriedades private em objetos e companion objects

Você pode especificar o padrão desejado nas opções de inspeção.

Convenções de nomenclatura recomendadas: devem começar com um sublinhado ou letra maiúscula, usando camelCase.

Exemplo:


  class Person {
    companion object {
      // Propriedade dentro de um companion object
      private val NO_NAME = Person()
    }
  }
