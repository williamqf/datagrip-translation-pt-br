Relata métodos com uma cláusula de parâmetro vazia que sobrepõem métodos sem cláusulas de parâmetro.

A convenção recomendada é utilizar um método sem cláusulas de parâmetro quando não houver parâmetros e o método não tiver efeitos colaterais.

Esta convenção promove o princípio de acesso uniforme, que diz que o código do cliente não deve ser afetado por uma decisão de implementar um atributo como um campo ou método.

De acordo com o princípio de substituição de Liskov, sempre que um método sobreposto não tiver uma cláusula de parâmetro, o método de sobreposição também deve ser declarado como um método sem efeitos colaterais.

A correção rápida remove a cláusula de parâmetro vazia.

Exemplo:


  trait Base { def isOk: Boolean }

  class Impl extends Base {
    override def isOk(): Boolean = ???
  }

Após a correção rápida ser aplicada:


  trait Base { def isOk: Boolean }

  class Impl extends Base {
    override def isOk: Boolean = ???
  }
