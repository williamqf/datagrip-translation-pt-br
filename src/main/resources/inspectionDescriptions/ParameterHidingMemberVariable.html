Relata parâmetros de método que possuem o mesmo nome de um campo na classe envolvente. Devido a essa nomenclatura, o parâmetro pode ser usado erroneamente quando a intenção era usar o campo de mesmo nome.

Recomenda-se renomear o parâmetro por meio de uma correção rápida (Quick Fix).

Exemplo:


  class Main {
    private String value;

    public Main(String value) {
      value = value.toUpperCase();
    }
  }

Você pode configurar esta inspeção com as seguintes opções:

  1. Ignorar setters de propriedade - Ignora parâmetros de setters simples.
  2. Ignorar campos da superclasse invisíveis para a subclasse - Ignora campos private na superclasse que são invisíveis para este método.
  3. Ignorar para construtores - Ignora parâmetros de construtores.
  4. Ignorar para métodos abstratos - Ignora parâmetros de métodos abstratos.
  5. Ignorar parâmetros de métodos static que ocultam campos de instância - Ignora parâmetros de métodos static que ocultam campos de instância e ignora parâmetros de métodos de instância em classes internas estáticas que ocultam campos de instância da classe externa. Embora não seja estritamente uma ocultação, tais parâmetros ainda podem ser confusos.
