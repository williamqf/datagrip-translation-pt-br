Relata padrões de uso comuns de java.util.Map e sugere substituí-los por: getOrDefault(), computeIfAbsent(), putIfAbsent(), merge() ou replaceAll().

Exemplo:


  map.containsKey(key) ? map.get(key) : "default";

Após a aplicação da correção rápida:


  map.getOrDefault(key, "default");

Exemplo:


  List<String> list = map.get(key);
  if (list == null) {
    list = new ArrayList<>();
    map.put(key, list);
  }

Após a aplicação da correção rápida:


  map.computeIfAbsent(key, localKey -> new ArrayList<>());

Exemplo:


  Integer val = map.get(key);
  if (val == null) map.put(key, 1);
  else map.put(key, val + 1);

Após a aplicação da correção rápida:


  map.merge(key, 1, (localKey, localValue) -> localValue + 1);

Exemplo:


  for (Map.Entry<String, String> entry : map.entrySet()) {
    map.put(entry.getKey(), transform(entry.getValue()));
  }

Após a aplicação da correção rápida:


  map.replaceAll((localKey, localValue) -> transform(localValue));

Observe que, se o código extraído em uma expressão lambda modificar o mesmo Map, a substituição por computeIfAbsent() ou merge() pode não funcionar corretamente para algumas implementações de Map. Por padrão, se este código tiver efeitos colaterais, nenhum aviso será exibido. Se necessário, ative a opção Sugerir substituição mesmo que a lambda possa ter efeitos colaterais para exibir sempre o aviso.

Além disso, a semântica pode mudar devido ao tratamento diferente de valores null em métodos antigos como put() e métodos novos como computeIfAbsent() ou merge(); você deve levar isso em consideração se for importante armazenar valores null no Map fornecido. A inspeção não sugerirá a substituição quando o valor for conhecido estaticamente como nulo, mas para valores onde a nulidade é desconhecida, a substituição ainda será sugerida. Nesses casos, recomendamos suprimir o aviso e adicionar um comentário explicativo.
