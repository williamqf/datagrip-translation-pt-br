Relata o uso de anotações de parâmetro de valor sem um alvo explícito, onde o alvo padrão pode se tornar inesperado e indesejado.

Anotações de parâmetro de valor sem um alvo explícito antes do Kotlin 2.1.20 são aplicadas apenas ao próprio parâmetro. Para anotações aplicáveis tanto a parâmetros quanto a campos ou propriedades, esse comportamento pode ser inesperado quando a anotação é usada em um parâmetro de construtor val ou var.

Exemplo: 


class Example(
    @Param val foo: String,
    @ParamField val bar: String, // o campo não está anotado, apenas o parâmetro está
    @ParamProperty val baz: String, // a propriedade não está anotada, apenas o parâmetro está
    @ParamProperty boo: String,
)

@Target(AnnotationTarget.VALUE_PARAMETER)
annotation class Param
@Target(AnnotationTarget.VALUE_PARAMETER, AnnotationTarget.FIELD)
annotation class ParamField
@Target(AnnotationTarget.VALUE_PARAMETER, AnnotationTarget.PROPERTY)
annotation class ParamProperty

Após a aplicação da correção rápida:


class Example(
    @Param val foo: String,
    @field:ParamField val bar: String,
    @property:ParamProperty val baz: String,
    @ParamProperty boo: String,
)

@Target(AnnotationTarget.VALUE_PARAMETER)
annotation class Param
@Target(AnnotationTarget.VALUE_PARAMETER, AnnotationTarget.FIELD)
annotation class ParamField
@Target(AnnotationTarget.VALUE_PARAMETER, AnnotationTarget.PROPERTY)
annotation class ParamProperty
