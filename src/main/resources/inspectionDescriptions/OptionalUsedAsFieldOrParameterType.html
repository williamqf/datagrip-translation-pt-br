Relata quaisquer instâncias de java.util.Optional<T>, java.util.OptionalDouble, java.util.OptionalInt, java.util.OptionalLong ou com.google.common.base.Optional sendo usados como tipos de campos ou parâmetros.

O Optional foi projetado para fornecer um mecanismo limitado para tipos de retorno de métodos de biblioteca onde há a necessidade de uma forma clara de representar "nenhum resultado".

Usar campos com o tipo java.util.Optional também é problemático se a classe precisar ser Serializable, pois o java.util.Optional não é serializável.

Exemplo:


  class MyClass {
    Optional<String> name; // campo Optional

    // parâmetro Optional
    void setName(Optional<String> name) {
      this.name = name;
    }
  }
